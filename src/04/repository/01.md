# Repositori XBPS untuk T4n OS (Void Linux Based)

## 📦 Pendahuluan

**Repositori** adalah inti dari sistem paket XBPS. Repositori dapat bersifat **lokal** (di komputer Anda sendiri) atau **remote** (di server internet). Sebuah repositori berisi:

| Komponen | Penjelasan |
|----------|------------|
| **File paket biner (.xbps)** | File paket yang sudah dikompilasi dan siap diinstall |
| **Tanda tangan (signature)** | File `.sig` untuk memverifikasi keaslian paket |
| **File repodata** | File bernama `$ARCH-repodata` (contoh: `x86_64-repodata`) yang berisi metadata semua paket dalam repositori |

> **Penting:** Repositori lokal **tidak wajib** memiliki tanda tangan, tetapi repositori remote **WAJIB** memiliki tanda tangan untuk alasan keamanan .

---

## 🗺️ Struktur Repositori Utama Void Linux

Lokasi repositori utama Void Linux relatif terhadap base mirror URL:

| Arsitektur & Libc | Path Repositori |
|-------------------|-----------------|
| **glibc** (x86_64, i686) | `/current` |
| **musl** | `/current/musl` |
| **aarch64** (glibc) | `/current/aarch64` |
| **aarch64-musl** | `/current/aarch64` (dibedakan dari konfigurasi repositori) |

### Contoh URL lengkap

Untuk T4n OS dengan arsitektur x86_64 glibc (yang paling umum), mirror URL biasanya:
```
https://mirror.example.com/voidlinux/current
```

File `x86_64-repodata` akan berada di URL tersebut.

## 📂 Subrepositori (Repositori Tambahan)

Selain repositori utama yang sudah aktif secara default saat instalasi, Void menyediakan beberapa repositori resmi lainnya yang **tidak aktif secara default** :

| Subrepositori | Isi | Cara Aktivasi |
|---------------|-----|---------------|
| **nonfree** | Paket dengan lisensi tidak bebas (non-free) | `sudo xbps-install -S void-repo-nonfree` |
| **multilib** | Library 32-bit untuk sistem 64-bit (hanya glibc) | `sudo xbps-install -S void-repo-multilib` |
| **multilib/nonfree** | Paket nonfree versi 32-bit | `sudo xbps-install -S void-repo-multilib-nonfree` |
| **debug** | Simbol debugging untuk paket | `sudo xbps-install -S void-repo-debug` |

Semua paket aktivasi repositori di atas **hanya menginstall file konfigurasi** di `/usr/share/xbps.d` dan tidak menginstall paket aplikasi apapun.

---

## 🔓 Repositori nonfree

Repositori `nonfree` berisi paket yang memiliki lisensi **tidak bebas** .

### Cara Mengaktifkan

```bash
sudo xbps-install -S void-repo-nonfree
```

### Mengapa Paket Masuk ke nonfree?

Beberapa alasan mengapa suatu paket ditempatkan di repositori nonfree:

| Alasan | Contoh |
|--------|--------|
| **Lisensi non-free meskipun source code tersedia** | Beberapa software dengan lisensi proprietary tapi source bisa dilihat |
| **Hanya dirilis sebagai biner (tanpa source)** | Driver NVIDIA, beberapa firmware |
| **Mengandung teknologi yang dipatenkan** | Codec multimedia tertentu (misal: H.264, MP3 pada beberapa yurisdiksi) |

### Contoh Paket di nonfree

```bash
# Setelah mengaktifkan nonfree, cari paketnya
xbps-query -Rs nvidia
```

## 📚 Repositori multilib

Repositori `multilib` menyediakan paket **32-bit** untuk sistem 64-bit sebagai lapisan kompatibilitas.

### Prasyarat

| Kondisi | Keterangan |
|---------|------------|
| **Arsitektur** | Hanya untuk `x86_64` (64-bit) |
| **C library** | Hanya untuk **glibc** (tidak untuk musl)  |

### Cara Mengaktifkan

```bash
sudo xbps-install -S void-repo-multilib
```

### Contoh Penggunaan

Setelah mengaktifkan multilib, Anda bisa menginstal package 32-bit, misalnya:

```bash
# Library 32-bit untuk Steam atau game
sudo xbps-install -S libGL-32bit
```

Biasanya paket 32-bit memiliki akhiran `-32bit` pada namanya.

---

## 🔀 Repositori multilib/nonfree

Repositori ini adalah **gabungan** dari `multilib` dan `nonfree` - berisi paket 32-bit dengan lisensi tidak bebas.

### Cara Mengaktifkan

```bash
sudo xbps-install -S void-repo-multilib-nonfree
```

### Contoh Paket

Paket seperti `nvidia-32bit` (driver NVIDIA 32-bit) berada di repositori ini.

---

## 🐞 Repositori debug

Paket Void Linux dikirim **tanpa simbol debugging** untuk menghemat ruang. Jika Anda perlu debugging atau menganalisis core dump, Anda memerlukan simbol debugging.

### Cara Mengaktifkan

```bash
sudo xbps-install -S void-repo-debug
```

### Menginstall Simbol Debugging

Setelah repositori debug diaktifkan, simbol untuk suatu paket `<package>` dapat diinstall dengan paket `<package>-dbg`.

**Contoh untuk bash:**
```bash
sudo xbps-install -S bash-dbg
```

---

## 🔍 Mencari Dependensi Debug dengan `xdbg`

Package `xtools` menyediakan utilitas `xdbg` yang sangat berguna untuk mendapatkan daftar paket debug yang diperlukan (termasuk dependensinya) untuk suatu paket.

### Instalasi xtools

```bash
sudo xbps-install -S xtools
```

### Penggunaan xdbg

```bash
# Mendapatkan daftar debug packages untuk bash
xdbg bash
```

**Output:**
```
bash-dbg
glibc-dbg
```

### Menginstall Semua Debug Packages Sekaligus

```bash
# Install semua debug packages untuk bash beserta dependensinya
sudo xbps-install -S $(xdbg bash)
```

### Contoh Lain

```bash
# Untuk firefox
xdbg firefox

# Untuk kernel
xdbg linux6.6
```

## 📝 Ringkasan Perintah Aktivasi Repositori

| Repositori | Perintah Aktivasi |
|------------|-------------------|
| **nonfree** | `sudo xbps-install -S void-repo-nonfree` |
| **multilib** | `sudo xbps-install -S void-repo-multilib` |
| **multilib/nonfree** | `sudo xbps-install -S void-repo-multilib-nonfree` |
| **debug** | `sudo xbps-install -S void-repo-debug` |

> **Catatan:** Tanda `-S` pada perintah di atas penting karena kita perlu menyinkronkan indeks repositori setelah menambahkan repositori baru.

## ⚙️ Lokasi File Konfigurasi Repositori

Semua file konfigurasi repositori disimpan di:

| Lokasi | Penjelasan |
|--------|------------|
| `/usr/share/xbps.d` | Konfigurasi default (dari package void-repo-*) |
| `/etc/xbps.d` | Konfigurasi lokal pengguna (override) |

### Melihat Repositori yang Aktif

```bash
xbps-query -L
```

Output akan menampilkan semua repositori yang terdaftar beserta alamatnya.

### Menonaktifkan Repositori (tanpa menghapus package)

Buat file di `/etc/xbps.d` dengan nama `10-disable-something.conf`:

```
repository=/usr/share/xbps.d/void-repo-nonfree
```

Atau lebih sederhana, cukup hapus/mindahkan file konfigurasi dari `/usr/share/xbps.d`.

---

## ⚠️ Catatan Penting untuk T4n OS

1. **Untuk sistem musl** (jika T4n OS menggunakan musl libc), repositori `multilib` **tidak tersedia** karena multilib hanya mendukung glibc .

2. **Selalu jalankan `xbps-install -S` setelah mengaktifkan repositori baru** untuk menyegarkan indeks.

3. **Debug symbols memakan ruang besar** - Paket `-dbg` biasanya berukuran beberapa kali lipat dari paket aslinya. Hapus setelah selesai debugging dengan `sudo xbps-remove -R <package>-dbg`.

4. **Urutan prioritas repositori** - Void akan mengambil paket dari repositori yang disebutkan pertama kali di `xbps-query -L`. Jika perlu override, buat file di `/etc/xbps.d`.

---

**Dokumentasi ini disusun untuk T4n OS berbasis Void Linux.** Pastikan sistem Anda sudah terupdate sebelum mengaktifkan repositori tambahan.
