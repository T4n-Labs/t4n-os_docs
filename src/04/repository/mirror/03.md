# Menggunakan Tor Mirrors di T4n OS

## 🧅 Pengenalan Tor

**Tor** (The Onion Router) adalah perangkat lunak anonimisasi yang mengenkripsi dan memantulkan lalu lintas jaringan melalui serangkaian komputer di seluruh dunia. Tor dapat mengakses:
- **Situs biasa** di internet (via exit relay)
- **Situs tersembunyi (hidden services)** yang hanya tersedia di jaringan Tor (dengan domain `.onion`)

Menggunakan Tor untuk mengakses repositori Void Linux memberikan lapisan privasi dan keamanan tambahan, terutama berguna di jaringan yang tidak terpercaya atau untuk menjaga anonimitas.

## 🔧 Membuat XBPS Terhubung via Tor

XBPS mendukung variabel lingkungan **`SOCKS_PROXY`**, yang memudahkan routing koneksi melalui Tor.

### Langkah-langkah:

1. **Install Tor**
2. **Jalankan Tor** (sebagai user biasa atau system service)
3. **Setel variabel `SOCKS_PROXY`** di environment
4. **Konfigurasi mirror ke alamat `.onion`** (jika ingin menggunakan hidden service)
5. **Jalankan perintah XBPS** seperti biasa

---

## 📦 Instalasi dan Menjalankan Tor

### Instalasi Paket Tor

```bash
sudo xbps-install -S tor
```

### Menjalankan Tor

**Metode 1: Menjalankan sebagai user biasa (untuk testing)**
```bash
tor
```

**Metode 2: Menjalankan sebagai system service (runit)**
```bash
sudo ln -s /etc/sv/tor /var/service/
# Cek status
sudo sv status tor
```

Secara default, Tor akan:
- Bertindak sebagai **klien (client)**
- Membuka **proxy SOCKS5** di `localhost` pada **port 9050**

---

## 🔌 Membuat XBPS Terhubung via Proxy SOCKS

XBPS membaca variabel lingkungan `SOCKS_PROXY` dan akan menggunakan proxy yang ditentukan untuk semua koneksi jaringan.

### Contoh: Upgrade sistem melalui Tor

```bash
# Setel proxy SOCKS ke Tor default
export SOCKS_PROXY="socks5://127.0.0.1:9050"

# Jalankan upgrade sistem (koneksi akan melewati Tor)
sudo -E xbps-install -Su
```

> **`-E` pada sudo** diperlukan untuk mempertahankan variabel lingkungan `SOCKS_PROXY`.

### Alternatif: Satu baris perintah

```bash
sudo env SOCKS_PROXY="socks5://127.0.0.1:9050" xbps-install -Su
```

## 🧅 Menggunakan Hidden Service Mirror (`.onion`)

Untuk anonimitas maksimal, Anda dapat menggunakan mirror `.onion` (hidden service) sebagai ganti mirror biasa via exit relay.

### Langkah 1: Override Konfigurasi Repositori

Salin file konfigurasi repositori dari `/usr/share/xbps.d` ke `/etc/xbps.d` dan ganti URL dengan alamat `.onion`.

```bash
# Buat direktori konfigurasi jika belum ada
sudo mkdir -p /etc/xbps.d

# Copy semua file konfigurasi repositori
sudo cp /usr/share/xbps.d/*-repository-*.conf /etc/xbps.d/

# Ganti URL default dengan mirror .onion (contoh menggunakan Lysator)
sudo sed -i 's|https://repo-default.voidlinux.org|http://lysator7eknrfl47rlyxvgeamrv7ucefgrrlhk7rouv3sna25asetwid.onion/pub/voidlinux|g' /etc/xbps.d/*-repository-*.conf
```

> **Catatan:** Gunakan `http://` (bukan https://) untuk mirror `.onion` karena Tor sudah menyediakan enkripsi end-to-end.

### Langkah 2: Setel SOCKS_PROXY dan Sinkronkan

```bash
export SOCKS_PROXY="socks5://127.0.0.1:9050"
sudo -E xbps-install -S
```

### Langkah 3: Verifikasi

Anda akan melihat repositori disinkronkan dari alamat `.onion` yang telah ditentukan:

```
[*] Updating `http://lysator7eknrfl47rlyxvgeamrv7ucefgrrlhk7rouv3sna25asetwid.onion/pub/voidlinux/current/x86_64/x86_64-repodata' ...
x86_64-repodata: 1441KB [avg rate: 773KB/s]
```

## 🛡️ Pertimbangan Keamanan

### ⚠️ Risiko kebocoran DNS

Jika Anda menggunakan mirror `.onion` tetapi **lupa** menyetel `SOCKS_PROXY`, maka:
- Query DNS untuk nama `.onion` akan **bocor** ke DNS server yang dikonfigurasi
- Ini dapat membahayakan anonimitas Anda

### ✅ Solusi: Setel SOCKS_PROXY secara otomatis

Agar `SOCKS_PROXY` selalu tersedia di environment, buat file inisialisasi di `/etc/profile.d/`:

```bash
# Buat file script
sudo nano /etc/profile.d/socksproxy.sh
```

Isi dengan:
```bash
#!/bin/sh
export SOCKS_PROXY="socks5://127.0.0.1:9050"
```

Buat file tersebut executable:
```bash
sudo chmod +x /etc/profile.d/socksproxy.sh
```

Setelah logout dan login kembali, variabel `SOCKS_PROXY` akan otomatis tersedia di semua sesi shell.

| Tujuan | Perintah |
|--------|----------|
| **Install Tor** | `sudo xbps-install -S tor` |
| **Jalankan Tor (user)** | `tor` |
| **Jalankan Tor (service)** | `sudo ln -s /etc/sv/tor /var/service/` |
| **Set proxy sementara** | `export SOCKS_PROXY="socks5://127.0.0.1:9050"` |

| **Auto-set SOCKS_PROXY** | Buat file di `/etc/profile.d/socksproxy.sh` |

## 🔄 Contoh Alur Kerja Lengkap

Berikut alur kerja lengkap menggunakan mirror `.onion` Lysator dengan Tor:

```bash
# 1. Install dan jalankan Tor
sudo xbps-install -S tor
sudo ln -s /etc/sv/tor /var/service/
sudo sv status tor

# 2. Override konfigurasi repositori ke mirror .onion
sudo mkdir -p /etc/xbps.d
sudo cp /usr/share/xbps.d/*-repository-*.conf /etc/xbps.d/
sudo sed -i 's|https://repo-default.voidlinux.org|http://lysator7eknrfl47rlyxvgeamrv7ucefgrrlhk7rouv3sna25asetwid.onion/pub/voidlinux|g' /etc/xbps.d/*-repository-*.conf

# 3. Setel SOCKS_PROXY otomatis (opsional, untuk keamanan)
echo 'export SOCKS_PROXY="socks5://127.0.0.1:9050"' | sudo tee /etc/profile.d/socksproxy.sh
sudo chmod +x /etc/profile.d/socksproxy.sh

# 4. Sinkronkan dan upgrade melalui Tor
export SOCKS_PROXY="socks5://127.0.0.1:9050"
sudo -E xbps-install -S
sudo -E xbps-install -Su
```

## 💡 Kesimpulan

| Skenario | Rekomendasi |
|----------|-------------|
| **Pengguna biasa, hanya ingin privasi ringan** | Gunakan Tor + mirror biasa (via exit relay) |
| **Pengguna yang membutuhkan anonimitas tinggi** | Gunakan Tor + mirror `.onion` + DNS leak protection |
| **Hanya untuk kecepatan (bukan privasi)** | Jangan gunakan Tor, pilih mirror geografis biasa |

**Menggunakan Tor mirror di T4n OS memberikan:**
- ✅ Privasi dan anonimitas saat mengunduh paket
- ✅ Akses ke repositori di wilayah yang mungkin diblokir
- ✅ Enkripsi end-to-end (untuk mirror `.onion`)

**Namun perlu diingat:**
- ⚠️ Kecepatan unduh akan lebih lambat karena routing melalui beberapa hop
- ⚠️ Pastikan `SOCKS_PROXY` selalu terset saat menggunakan mirror `.onion`

---

**Dokumentasi ini disusun untuk T4n OS berbasis Void Linux.** Gunakan Tor mirror dengan bijak dan pahami implikasi privasi serta kecepatan sebelum menggunakannya.
