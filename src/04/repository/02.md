# Repositori Kustom untuk T4n OS

## 📦 Pendahuluan

Void Linux (dan T4n OS sebagai turunannya) mendukung pembuatan **repositori kustom**, baik secara **lokal** (di komputer sendiri) maupun **remote** (di server internet).

Fitur ini **hanya direkomendasikan** untuk:
- Menyajikan paket kustom yang Anda buat sendiri
- Paket dari sumber tepercaya lainnya (seperti hasil build internal perusahaan/komunitas)

---

## ⚠️ Peringatan Penting

> **Proyek Void Linux tidak mendukung repositori pihak ketiga manapun.**
>
> Penggunaan repositori paket pihak ketiga memiliki **risiko keamanan yang sangat serius**, termasuk:
> - Potensi **malware** atau **backdoor** pada paket
> - Konflik dependensi yang dapat **merusak sistem**
> - Paket yang **tidak teruji** dengan versi pustaka di T4n OS
> - Risiko **ketidakstabilan sistem** secara keseluruhan

**Hanya tambahkan repositori kustom jika Anda benar-benar memahami risikonya dan percaya sepenuhnya dengan sumbernya.**

---

## 🛠️ Menambahkan Repositori Kustom

### Langkah-langkah Umum

Untuk menambahkan repositori kustom, buat file konfigurasi di direktori `/etc/xbps.d/` dengan isi:

```
repository=<URL>
```

Dimana `<URL>` dapat berupa:
- **Path direktori lokal** (contoh: `file:///path/to/repo` atau `/path/to/repo`)
- **URL repositori remote** (contoh: `http://my.domain.com/repo`)

### Format URL yang Didukung

| Tipe | Format URL | Contoh |
|------|------------|--------|
| **Direktori lokal** | `/absolute/path` atau `file:///absolute/path` | `repository=/home/user/myrepo` |
| **HTTP/HTTPS** | `http://` atau `https://` | `repository=http://mirror.example.com/repo` |
| **FTP** | `ftp://` | `repository=ftp://ftp.example.com/repo` |

---

## 📂 Contoh Konfigurasi

### 1. Repositori Remote (via HTTP)

```bash
# Buat file konfigurasi
echo 'repository=http://my.domain.com/repo' | sudo tee /etc/xbps.d/my-remote-repo.conf
```

> **Penting untuk repositori remote:** Repositori remote **WAJIB memiliki tanda tangan (signature)** . Perintah `xbps-install` akan **menolak** menginstall paket dari repositori remote yang tidak ditandatangani.

### 2. Repositori Lokal

```bash
# Buat file konfigurasi
echo 'repository=/home/user/myrepo' | sudo tee /etc/xbps.d/my-local-repo.conf
```

Atau dengan format `file://`:
```bash
echo 'repository=file:///home/user/myrepo' | sudo tee /etc/xbps.d/my-local-repo.conf
```

### 3. Menambahkan Beberapa Repositori Sekaligus

Buat file konfigurasi terpisah untuk setiap repositori (atau gabungkan dalam satu file):

```bash
# File: /etc/xbps.d/99-custom-repos.conf
repository=/home/user/local-repo
repository=http://internal.company.com/void-repo
repository=https://backup.example.com/repo
```

---

## 🔍 Prioritas Repositori

Urutan repositori dalam file konfigurasi menentukan **prioritas** saat XBPS mencari paket:

1. Repositori yang disebut **pertama** akan dicari terlebih dahulu
2. Jika paket ditemukan di repositori pertama, repositori berikutnya tidak akan dicari untuk paket yang sama

Untuk mengatur prioritas, gunakan **prefix angka** pada nama file:

| Prioritas | File | Keterangan |
|-----------|------|------------|
| **Tertinggi** | `00-local.conf` | Dicari pertama |
| **Sedang** | `10-custom.conf` | Dicari setelah 00-* |
| **Default** | `20-void-repo.conf` | Repositori resmi Void (prioritas lebih rendah) |

Contoh:
```bash
# Prioritas tertinggi - repositori lokal
echo 'repository=/opt/local-repo' | sudo tee /etc/xbps.d/00-local.conf

# Prioritas sedang - repositori kustom remote
echo 'repository=https://myrepo.example.com' | sudo tee /etc/xbps.d/10-custom.conf
```

## 🔐 Membuat Repositori Remote yang Ditandatangani (Signed)

Karena repositori remote **wajib** memiliki signature, berikut langkah-langkah membuatnya:

### 1. Generate Kunci RSA

```bash
ssh-keygen -t rsa -b 4096 -m PEM -f private.pem
```

Perintah ini menghasilkan:
- `private.pem` → kunci privat (RAHASIA, jangan disebarkan)
- `private.pem.pub` → kunci publik (disebarkan ke pengguna)

### 2. Siapkan Repositori

```bash
mkdir -p ~/myrepo
# Copy paket .xbps ke direktori repositori
cp /path/to/*.xbps ~/myrepo/
```

### 3. Tandatangani Repositori dan Paket

```bash
# Inisialisasi repositori dengan tanda tangan
xbps-rindex --privkey private.pem --sign --signedby "Your Name <email@example.com>" ~/myrepo

# Tandatangani setiap paket
xbps-rindex --privkey private.pem --sign-pkg ~/myrepo/*.xbps
```

### 4. Konfigurasi di PC Pengguna

Di PC pengguna yang akan menggunakan repositori:

1. **Tambahkan repositori:**
   ```bash
   echo 'repository=https://server.com/myrepo' | sudo tee /etc/xbps.d/10-myrepo.conf
   ```

2. **Tambahkan kunci publik** (jika diperlukan verifikasi tambahan), atau biarkan XBPS memverifikasi dengan kunci dari repositori itu sendiri.

## 🔄 Manajemen Repositori Kustom

### Melihat Daftar Repositori Aktif

```bash
xbps-query -L
```

Output akan menampilkan semua repositori yang terdaftar beserta alamatnya.

### Menonaktifkan Repositori Kustom

Cukup hapus atau pindahkan file `.conf`-nya:

```bash
sudo mv /etc/xbps.d/my-custom-repo.conf /etc/xbps.d/my-custom-repo.conf.disabled
```

### Memperbarui Indeks Repositori

Setelah menambah atau mengubah repositori, selalu sinkronkan indeks:

```bash
sudo xbps-install -S
```

## 📝 Template Konfigurasi Lengkap

### File: `/etc/xbps.d/99-custom-repo.conf`

```bash
# Repositori lokal untuk development
repository=/home/developer/void-packages/hostdir/binpkgs

# Repositori internal perusahaan
repository=https://repo.internal.company.com/void/current

# Repositori backup (prioritas terendah karena disebut terakhir)
repository=http://backup-repo.example.com/void
```

## ⚠️ Risiko dan Pertimbangan Keamanan

| Risiko | Penjelasan |
|--------|------------|
| **Malware** | Repositori tidak resmi bisa menyisipkan kode berbahaya |
| **Konflik dependensi** | Paket dari sumber berbeda mungkin tidak kompatibel |
| **Ketidakstabilan** | Paket yang tidak teruji bisa merusak sistem |
| **Update bermasalah** | Versi baru dari repositori kustom bisa merusak dependensi |
| **Void tidak memberikan dukungan** | Jika sistem rusak karena repositori kustom, komunitas Void tidak akan membantu |

### Praktik Terbaik untuk Keamanan

1. **Hanya gunakan repositori dari sumber yang sangat tepercaya**
2. **Verifikasi tanda tangan** pada repositori remote
3. **Gunakan dalam lingkungan terisolasi** (container/VM) untuk pengujian
4. **Backup sistem** sebelum menambahkan repositori kustom
5. **Gunakan versi spesifik** (bukan `latest`) jika memungkinkan
6. **Pantau perubahan** pada repositori kustom secara berkala

---

## 🧹 Menghapus Repositori Kustom

1. **Hapus file konfigurasi:**
   ```bash
   sudo rm /etc/xbps.d/my-custom-repo.conf
   ```

2. **Sinkronkan ulang indeks:**
   ```bash
   sudo xbps-install -S
   ```

3. **Opsional - Hapus paket yang hanya berasal dari repositori tersebut:**
   ```bash
   xbps-query -O  # Cek orphans
   sudo xbps-remove -O  # Hapus orphans
   ```

## 🔗 Kesimpulan

Repositori kustom di T4n OS memberi fleksibilitas luar biasa untuk:
- Mendistribusikan paket internal perusahaan
- Menguji paket sebelum dikirim ke repositori resmi
- Menggunakan versi modifikasi dari paket yang ada

**Namun**, dengan fleksibilitas ini datang tanggung jawab besar. **Void Linux tidak bertanggung jawab** atas kerusakan yang disebabkan oleh repositori pihak ketiga. Gunakan dengan bijak, hanya dari sumber yang benar-benar Anda percayai.

---

**Dokumentasi ini disusun untuk T4n OS berbasis Void Linux.** Selalu prioritaskan keamanan dan stabilitas sistem Anda di atas kenyamanan menggunakan repositori tambahan.
