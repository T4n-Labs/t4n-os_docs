# Menandatangani Repositori (Signing Repositories) di T4n OS

## 📜 Aturan Dasar

| Jenis Repositori | Wajib Ditandatangani? |
|------------------|----------------------|
| **Repositori Remote** (server internet) | ✅ **WAJIB** |
| **Repositori Lokal** (di komputer sendiri) | ❌ Tidak wajib |

> **Mengapa repositori remote wajib ditandatangani?**  
> Tanda tangan (signature) memastikan bahwa paket yang Anda download benar-benar berasal dari sumber yang terpercaya dan belum dimanipulasi selama perjalanan di jaringan. Tanpa tanda tangan, `xbps-install` akan **menolak** menginstall paket dari repositori remote.

---

## 🔑 Membuat Kunci RSA untuk Menandatangani

`xbps-rindex` membutuhkan kunci RSA dalam format **PEM** untuk menandatangani repositori. Anda dapat membuatnya dengan dua cara:

### Metode 1: Menggunakan `ssh-keygen` (Direkomendasikan)

```bash
ssh-keygen -t rsa -b 4096 -m PEM -f private.pem
```

**Parameter:**
| Opsi | Keterangan |
|------|-------------|
| `-t rsa` | Tipe kunci RSA |
| `-b 4096` | Panjang kunci 4096 bit (sangat aman) |
| `-m PEM` | Format output PEM (wajib untuk xbps-rindex) |
| `-f private.pem` | Nama file output |

**Output:**
- `private.pem` → Kunci **PRIVAT** (RAHASIA, jangan disebarkan)
- `private.pem.pub` → Kunci **publik** (dapat disebarkan ke pengguna)

### Metode 2: Menggunakan `openssl`

```bash
openssl genrsa -des3 -out private.pem 4096
```

**Parameter:**
| Opsi | Keterangan |
|------|-------------|
| `-des3` | Enkripsi kunci dengan triple DES (akan meminta password) |
| `-out private.pem` | Nama file output |
| `4096` | Panjang kunci |

> **Catatan:** Metode `openssl` akan meminta Anda memasukkan password untuk mengenkripsi kunci privat. Setiap kali menggunakan kunci ini, Anda harus memasukkan password tersebut.

## 📝 Menandatangani Repositori (Langkah Awal - Satu Kali)

Langkah ini **hanya perlu dilakukan satu kali** saat pertama kali membuat repositori. Perintah ini akan menambahkan metadata tanda tangan ke dalam repositori.

```bash
xbps-rindex --privkey private.pem --sign --signedby "I'm Groot" /path/to/repository/dir
```

### Parameter yang Digunakan

| Parameter | Fungsi | Contoh |
|-----------|--------|--------|
| `--privkey private.pem` | Menentukan file kunci privat untuk menandatangani | `--privkey ~/myrepo/private.pem` |
| `--sign` | Mengaktifkan mode penandatanganan repositori | - |
| `--signedby "string"` | Identitas penanda tangan (bebas, tapi informatif) | `--signedby "T4n OS Team <t4n@example.com>"` |
| `/path/to/repo` | Path direktori repositori yang akan ditandatangani | `/home/user/myrepo` |

### Contoh Lengkap

```bash
# Generate kunci (jika belum)
ssh-keygen -t rsa -b 4096 -m PEM -f ~/myrepo/private.pem

# Tandatangani repositori
xbps-rindex --privkey ~/myrepo/private.pem --sign --signedby "T4n Labs <admin@t4n.org>" ~/myrepo
```

**Setelah perintah ini, di dalam direktori repositori akan muncul:**
- File `repodata` yang sudah ditandatangani
- Metadata tanda tangan yang tersimpan

## 🖍️ Menandatangani Paket Individu

Setelah repositori ditandatangani, Anda perlu **menandatangani setiap paket** `.xbps` yang ada di dalamnya.

```bash
xbps-rindex --privkey private.pem --sign-pkg /path/to/repository/dir/*.xbps
```

### Contoh

```bash
xbps-rindex --privkey ~/myrepo/private.pem --sign-pkg ~/myrepo/*.xbps
```

Perintah ini akan menambahkan file tanda tangan (`.sig`) untuk setiap paket `.xbps`.

---

## ⚠️ Peringatan Penting

> **Paket yang ditambahkan di MASA DEPAN tidak akan ditandatangani secara otomatis!**

Setiap kali Anda menambahkan paket baru ke repositori, Anda **harus menjalankan ulang** perintah `--sign-pkg`:

```bash
# Setelah menambahkan paket baru ke repositori
cp paket-baru-1.0_1.x86_64.xbps ~/myrepo/

# Tandatangani paket baru tersebut
xbps-rindex --privkey private.pem --sign-pkg ~/myrepo/paket-baru-1.0_1.x86_64.xbps

# Atau tandatangani semua paket (yang belum ditandatangani akan ditandatangani ulang)
xbps-rindex --privkey private.pem --sign-pkg ~/myrepo/*.xbps
```

## 🔄 Alur Kerja Lengkap Membuat Repositori Signed

Berikut adalah alur kerja lengkap dari awal hingga repositori siap digunakan:

```bash
# 1. Buat direktori repositori
mkdir -p ~/signed-repo

# 2. Generate kunci RSA
ssh-keygen -t rsa -b 4096 -m PEM -f ~/signed-repo/private.pem

# 3. Copy paket .xbps ke direktori repositori
cp /path/to/*.xbps ~/signed-repo/

# 4. Tandatangani repositori (sekali saja)
xbps-rindex --privkey ~/signed-repo/private.pem --sign --signedby "T4n OS Repository" ~/signed-repo

# 5. Tandatangani semua paket
xbps-rindex --privkey ~/signed-repo/private.pem --sign-pkg ~/signed-repo/*.xbps

# 6. Buat indeks repositori (jika belum)
xbps-rindex -a ~/signed-repo/*.xbps

# 7. Selesai! Repositori siap dijadikan remote repository
```

## 🔌 Menggunakan Repositori Signed dari Sisi Pengguna

Di sisi client (pengguna yang akan menginstall paket dari repositori signed Anda):

**1. Tambahkan konfigurasi repositori:**
```bash
echo 'repository=https://serveranda.com/signed-repo' | sudo tee /etc/xbps.d/10-t4n-repo.conf
```

**2. Sinkronkan indeks:**
```bash
sudo xbps-install -S
```

**3. XBPS akan secara otomatis memverifikasi tanda tangan. Jika valid, instalasi berjalan normal.**

Jika tanda tangan tidak valid atau tidak ada, Anda akan mendapatkan pesan error seperti:
```
ERROR: repository `https://serveranda.com/signed-repo' is not signed, refusing to continue!
```

---

## 📂 Distribusi Kunci Publik (Opsional)

Jika Anda ingin pengguna dapat memverifikasi kunci secara manual (sebagai lapisan keamanan tambahan), distribusikan file kunci publik:

```bash
# Kunci publik ada di private.pem.pub (dari ssh-keygen)
cat ~/signed-repo/private.pem.pub
```

Namun, secara default **XBPS tidak memerlukan kunci publik terpisah** karena metadata tanda tangan sudah disematkan di repositori. Pengguna tetap bisa menginstall tanpa file kunci publik terpisah.

---

## 🛠️ Troubleshooting

### Error: `repository is not signed, refusing to continue`

**Penyebab:** Repositori remote tidak memiliki tanda tangan.

**Solusi:** Pastikan Anda sudah menjalankan `xbps-rindex --sign` dan `--sign-pkg` pada repositori.

### Error: `invalid signature`

**Penyebab:** Paket atau repodata rusak atau dimodifikasi setelah penandatanganan.

**Solusi:** Tandatangani ulang repositori dan paket.

### Lupa Password (jika menggunakan openssl -des3)

**Solusi:** Tidak ada cara memulihkan password. Generate kunci baru.

## 🔗 Ringkasan Perintah

| Tugas | Perintah |
|-------|----------|
| **Generate kunci (ssh-keygen)** | `ssh-keygen -t rsa -b 4096 -m PEM -f private.pem` |
| **Generate kunci (openssl)** | `openssl genrsa -des3 -out private.pem 4096` |
| **Tandatangani repositori** | `xbps-rindex --privkey private.pem --sign --signedby "Nama" /path/to/repo` |
| **Tandatangani paket** | `xbps-rindex --privkey private.pem --sign-pkg /path/to/repo/*.xbps` |
| **Tandatangani ulang setelah tambah paket** | `xbps-rindex --privkey private.pem --sign-pkg /path/to/repo/paket-baru.xbps` |

## 💡 Kesimpulan

| Poin Utama | Kesimpulan |
|------------|------------|
| **Repositori remote WAJIB ditandatangani** | Keamanan adalah prioritas |
| **Repositori lokal bebas tanda tangan** | Untuk testing dan penggunaan pribadi |
| **Satu kali inisialisasi tanda tangan** | `xbps-rindex --sign` cukup dijalankan sekali |
| **Paket baru harus ditandatangani ulang** | Tidak otomatis! Jalankan `--sign-pkg` setiap kali menambah paket |
| **Gunakan ssh-keygen untuk buat kunci** | Lebih sederhana daripada openssl |

Dengan mengikuti panduan ini, Anda dapat membuat repositori remote yang aman dan terverifikasi untuk T4n OS, memungkinkan pengguna lain menginstall paket dari repositori Anda dengan percaya diri.

---

**Dokumentasi ini disusun untuk T4n OS berbasis Void Linux.** Jaga kerahasiaan kunci privat Anda - jangan pernah mengunggahnya ke repository publik atau membagikannya kepada pihak yang tidak berwenang.
