# Paket Terbatas (Restricted Packages) di T4n OS

## 📦 Pengertian Paket Terbatas

Void Linux (dan T4n OS sebagai turunannya) menyediakan beberapa paket yang **dikelola secara resmi namun tidak didistribusikan** dalam bentuk biner. Paket-paket ini ditandai sebagai **restricted** dan **wajib dibangun sendiri** dari template `void-packages` secara lokal.

### Mengapa Sebuah Paket Menjadi Restricted?

| Alasan | Penjelasan |
|--------|------------|
| **Ukuran besar** | Alasan paling umum - paket terlalu besar untuk didistribusikan melalui repositori utama |
| **Lisensi restriktif** | Lisensi yang melarang redistribusi source code atau paket biner oleh pihak ketiga |
| **Kebijakan upstream** | Pembuat software tidak mengizinkan distribusi ulang |
| **Kebijakan Void** | Void berhak membatasi distribusi paket apapun untuk alasan apapun |

> Void **menyediakan template build** untuk paket-paket ini, tetapi tidak menyediakan file `.xbps`-nya. Anda harus membangunnya sendiri.

## 🛠️ Membangun Paket Restricted Secara Manual

### Prasyarat

1. **Repository `void-packages`** sudah di-clone
2. **Aktivasi build restricted** harus dinyalakan secara eksplisit

### Langkah-langkah

**1. Clone void-packages (jika belum):**
```bash
git clone https://github.com/void-linux/void-packages.git
cd void-packages
```

**2. Aktifkan build restricted:**

Edit file `etc/conf` di dalam direktori `void-packages`:

```bash
echo 'XBPS_ALLOW_RESTRICTED=yes' >> etc/conf
```

Atau jika file belum ada, buat dengan isi:
```
XBPS_ALLOW_RESTRICTED=yes
```

**3. Build paket yang diinginkan:**
```bash
./xbps-src pkg <nama-paket-restricted>
```

Contoh untuk paket `virtualbox-ose`:
```bash
./xbps-src pkg virtualbox-ose
```

**4. Install hasil build:**
```bash
sudo xbps-install --repository=hostdir/binpkgs <nama-paket>
```

## 🤖 Build Otomatis dengan `xbps-mini-builder`

Untuk mengotomatiskan proses build paket restricted, tersedia tool **`xbps-mini-builder`**.

### Fitur Utama

| Fitur | Penjelasan |
|-------|------------|
| **Build terjadwal** | Dapat dijalankan secara periodik (misal via cron) |
| **Build hanya jika ada perubahan** | Memeriksa apakah template berubah sebelum membangun ulang |
| **Daftar paket** | Membangun sekelompok paket yang telah ditentukan sebelumnya |

### Contoh Penggunaan Sederhana

```bash
# Instal xbps-mini-builder (jika tersedia)
sudo xbps-install -S xbps-mini-builder

# Bangun daftar paket dari file
xbps-mini-builder --file /path/to/package-list.txt
```

> **Catatan:** Dokumentasi lengkap `xbps-mini-builder` dapat ditemukan di repositori resmi Void Linux.

## 🌐 VUR - Void User Repository (Proyek T4n Labs)

**VUR (Void User Repository)** adalah repository berbasis komunitas untuk Void Linux yang menyediakan **template paket tambahan** di luar repository resmi.

### Karakteristik VUR

| Aspek | Keterangan |
|-------|------------|
| **Berbasis** | Template `xbps-src` |
| **Pengelola** | Komunitas (T4n Labs) |
| **Status** | Tidak didukung secara resmi oleh Void Linux |
| **Keamanan** | Perlu audit sebelum build/install |

### Tujuan VUR

- Memperluas ekosistem Void Linux
- Menyediakan software *niche* atau eksperimental
- Mendukung kolaborasi komunitas

### Sumber Daya VUR

| Sumber Daya | URL |
|-------------|-----|
| **Repositori VUR** | https://github.com/T4n-Labs/vur |
| **Website VUR** | https://t4n-labs.github.io/vur-web |
| **Kategori Paket** | `core`, `extra`, `multilib` |

### Struktur VUR

```
vur/
├── core/          # Paket inti (stabil)
├── extra/         # Paket tambahan (eksperimental)
├── multilib/      # Paket 32-bit untuk sistem 64-bit
├── vtools/        # Tools pendukung
└── packages.json  # Indeks semua paket
```

## 🚀 Let-X: VUR Helper untuk T4n OS

**Let-X** adalah CLI tool yang memudahkan pencarian, pengelolaan, dan pengunduhan template package dari VUR. Fungsinya mirip dengan `yay` atau `paru` di Arch Linux, tetapi untuk ekosistem VUR.

### Fitur Utama Let-X

| Fitur | Keterangan |
|-------|------------|
| `search` | Mencari package di VUR (nama atau deskripsi) |
| `info` | Menampilkan detail package + status lokal |
| `list` | Melihat daftar package per kategori |
| `get` | Mengunduh template package ke lokal |
| `update` | Memperbarui cache index VUR |

### Instalasi Let-X

**Metode 1: Script Otomatis (Direkomendasikan)**
```bash
git clone https://github.com/T4n-Labs/Let-X
cd Let-X
sudo ./install.sh
```

**Metode 2: via xbps-src (untuk yang sudah setup void-packages)**
```bash
cp -r xbps-template/letx ~/void-packages/srcpkgs/letx
cd ~/void-packages
./xbps-src pkg letx
sudo xbps-install --repository=hostdir/binpkgs letx
```

### Contoh Penggunaan Let-X

```bash
# Update index VUR
letx update

# Cari package
letx search discord
letx search "web browser" -c extra

# Lihat detail
letx info zen-browser

# Unduh template
letx get discord

# Lihat statistik repository
letx list -p

# Cari template lokal yang sudah diunduh
letx search -t discord
```

### Struktur Direktori Let-X

Setelah menggunakan Let-X, struktur direktori yang dibuat:

```
~/.config/letx/
├── core/              # Template dari kategori core
│   └── <package>/
│       ├── template
│       ├── files/
│       └── patches/
├── extra/             # Template dari kategori extra
└── multilib/          # Template dari kategori multilib

~/.cache/letx/
└── packages.json      # Cache index VUR
```

## 📊 Perbandingan Metode Mendapatkan Paket Restricted

| Metode | Kelebihan | Kekurangan | Cocok untuk |
|--------|-----------|------------|--------------|
| **Build manual via xbps-src** | Kontrol penuh, stabil | Ribet, perlu setup void-packages | Developer, maintainer |
| **xbps-mini-builder** | Otomatis, bisa dijadwalkan | Perlu konfigurasi awal | Sysadmin, build server |
| **VUR + Let-X** | Mudah, banyak pilihan paket | Tidak resmi, perlu review template | Pengguna harian, power user |

## ⚠️ Peringatan Keamanan

| Risiko | Penjelasan |
|--------|------------|
| **Template tidak teraudit** | Paket dari VUR belum tentu melalui proses review ketat seperti repo resmi |
| **Malware potensial** | Selalu review `template` dan file `patches/` sebelum build |
| **Konflik dependensi** | Paket dari VUR mungkin konflik dengan paket resmi |
| **Tidak ada dukungan resmi** | Void Linux tidak mendukung masalah yang disebabkan oleh VUR |

### Praktik Terbaik

1. **Review template sebelum build:**
   ```bash
   cat ~/.config/letx/extra/<package>/template
   ```

2. **Gunakan dalam container/VM untuk pengujian pertama**

3. **Backup sistem** sebelum menginstall paket dari VUR

4. **Hanya install dari maintainer tepercaya** (cek field `maintainer` di template)

## 🔄 Alur Kerja Tipikal dengan VUR + Let-X

```bash
# 1. Update index VUR
letx update

# 2. Cari package yang diinginkan
letx search "game engine"

# 3. Lihat detail dan maintainer
letx info godot

# 4. (Opsional) Review template sebelum build
cat ~/.config/letx/extra/godot/template

# 5. Unduh template
letx get godot

# 6. Jika sudah setup void-packages, build dan install
cd ~/void-packages
ln -s ~/.config/letx/extra/godot srcpkgs/godot
./xbps-src pkg godot
sudo xbps-install --repository=hostdir/binpkgs godot
```

## 💡 Kesimpulan

**Paket restricted** di T4n OS adalah paket yang tersedia templatenya tetapi tidak didistribusikan dalam bentuk biner. Anda harus membangunnya sendiri menggunakan `xbps-src`.

**VUR (Void User Repository)** memperluas ekosistem dengan menyediakan template paket tambahan dari komunitas T4n Labs.

**Let-X** adalah alat yang mempermudah pencarian, pengunduhan, dan pengelolaan template dari VUR - mirip seperti AUR helper di Arch Linux.

> **Ingat:** Selalu audit template dari VUR sebelum membangun, dan prioritaskan paket dari repositori resmi Void untuk kebutuhan sehari-hari. Gunakan VUR dan Let-X dengan bijak, hanya dari sumber yang Anda percayai.
