# Penggunaan Lanjutan XBPS untuk T4n OS

Dokumentasi ini membahas fitur-fitur lanjutan dari package manager XBPS yang berguna untuk situasi-situasi khusus seperti downgrade paket, mengunci paket, hingga membuat paket virtual.

---

## 📉 Downgrade Paket (Menurunkan Versi)

XBPS memungkinkan Anda untuk menurunkan versi paket ke versi tertentu yang lebih lama. Ini berguna ketika versi terbaru suatu aplikasi bermasalah di sistem Anda.

### Metode 1: Menggunakan `xdowngrade` (Termudah)

Cara termudah untuk melakukan downgrade adalah dengan menggunakan `xdowngrade` dari paket `xtools`.

**Instalasi xtools (jika belum):**
```bash
sudo xbps-install -S xtools
```

**Melakukan downgrade:**
```bash
sudo xdowngrade /var/cache/xbps/nama-paket-1.0_1.xbps
```

Contoh:
```bash
sudo xdowngrade /var/cache/xbps/firefox-115.0_1.x86_64.xbps
```

### Metode 2: Menggunakan XBPS Secara Langsung

Jika paket versi lama **pernah terinstall sebelumnya**, file `.xbps`-nya biasanya masih tersimpan di `/var/cache/xbps/`. Jika tidak pernah terinstall, Anda harus mendapatkan file `.xbps` tersebut dari sumber lain (misalnya dari repositori archive atau build sendiri).

**Langkah-langkah downgrade dengan XBPS:**

1. **Tambahkan paket versi lama ke repositori lokal:**
   ```bash
   sudo xbps-rindex -a /var/cache/xbps/nama-paket-1.0_1.xbps
   ```

2. **Lakukan downgrade dengan `xbps-install`:**
   ```bash
   sudo xbps-install -R /var/cache/xbps/ -f nama-paket-1.0_1
   ```

> **Flag `-f` (force) wajib digunakan** untuk memaksa downgrade atau instalasi ulang paket yang sudah terinstall.

**Contoh lengkap downgrade Firefox:**
```bash
# Asumsikan file firefox-115.0_1.x86_64.xbps ada di cache
sudo xbps-rindex -a /var/cache/xbps/firefox-115.0_1.x86_64.xbps
sudo xbps-install -R /var/cache/xbps/ -f firefox-115.0_1
```

## 🔒 Menahan Paket (Holding Packages)

Fitur **hold** mencegah suatu paket dari pembaruan saat Anda menjalankan `xbps-install -Su` (upgrade sistem). Berguna jika Anda ingin tetap menggunakan versi tertentu dari suatu aplikasi.

### Mengunci Paket (Hold)

```bash
sudo xbps-pkgdb -m hold <nama-paket>
```

Contoh:
```bash
sudo xbps-pkgdb -m hold firefox
# Firefox tidak akan ikut terupdate saat upgrade sistem
```

### Menghapus Kunci (Unhold)

```bash
sudo xbps-pkgdb -m unhold <nama-paket>
```

Contoh:
```bash
sudo xbps-pkgdb -m unhold firefox
# Firefox sekarang dapat diupdate kembali
```

### Melihat Daftar Paket yang Di-hold

```bash
xbps-query -H
```

## 🔐 Penguncian Repositori (Repository-locking / Repolock)

Jika Anda membangun paket sendiri menggunakan `xbps-src` (misalnya dengan opsi build kustom atau template yang dimodifikasi), Anda mungkin ingin mencegah pembaruan sistem mengganti paket kustom Anda dengan versi standar dari repositori.

**Repolock** memastikan bahwa suatu paket **hanya akan diperbarui dari repositori yang sama** dengan saat pertama kali diinstall.

### Mengunci Paket ke Repositori Tertentu

```bash
sudo xbps-pkgdb -m repolock <nama-paket>
```

Contoh:
```bash
# Setelah menginstall Firefox build sendiri dari repositori lokal
sudo xbps-pkgdb -m repolock firefox
# Firefox sekarang hanya akan update dari repositori yang sama
```

### Menghapus Penguncian Repositori

```bash
sudo xbps-pkgdb -m repounlock <nama-paket>
```

### Melihat Daftar Paket yang Direpolock

```bash
xbps-query --list-repolock-pkgs
```

## 🙈 Mengabaikan Paket (Ignoring Packages)

Terkadang Anda ingin menghapus suatu paket karena fungsinya sudah disediakan oleh paket lain, tetapi tidak bisa karena masalah dependensi.

**Contoh kasus:** Anda ingin menggunakan `doas` sebagai pengganti `sudo`, tetapi paket `sudo` adalah dependensi dari `base-system` sehingga tidak bisa dihapus.

### Solusi: Abaikan (Ignore) Paket Tersebut

1. **Buat file konfigurasi di `/etc/xbps.d/`** (misalnya `10-ignore-sudo.conf`):
   ```bash
   sudo nano /etc/xbps.d/10-ignore-sudo.conf
   ```

2. **Isi dengan entri `ignorepkg`:**
   ```
   ignorepkg=sudo
   ```

3. **Sekarang hapus paket yang diabaikan:**
   ```bash
   sudo xbps-remove sudo
   ```

Setelah diabaikan, XBPS akan menganggap paket tersebut tidak ada saat melakukan resolusi dependensi.

### Contoh Kasus Lain

Mengabaikan `elogind` karena menggunakan `runit` sebagai init system:
```
ignorepkg=elogind
```

> **Peringatan:** Mengabaikan paket yang merupakan dependensi penting dapat merusak sistem. Pastikan Anda benar-benar memahami konsekuensinya.

## 📦 Paket Virtual (Virtual Packages)

**Paket virtual** adalah entri yang memetakan permintaan suatu paket ke paket lain yang sebenarnya. Fitur ini berguna ketika:
- Nama paket berubah di repositori
- Anda ingin mengganti satu implementasi dengan implementasi lain
- Menyediakan "alias" untuk paket

### Cara Membuat Paket Virtual

Buat file konfigurasi di `/etc/xbps.d/` (misalnya `99-virtual-linux.conf`):

```
virtualpkg=linux:linux6.6
```

Artinya: setiap kali ada permintaan paket bernama `linux`, XBPS akan mengarahkannya ke paket `linux6.6`.

### Contoh Penggunaan

**Skenario:** Paket `wireguard-tools` berubah nama menjadi `wireguard-tools-lite`. Anda bisa membuat paket virtual:

1. **Buat file `/etc/xbps.d/99-virtual-wireguard.conf`:**
   ```
   virtualpkg=wireguard-tools:wireguard-tools-lite
   ```

2. **Hasilnya:** Perintah seperti `xbps-install wireguard-tools` akan menginstall `wireguard-tools-lite`.

### Virtual Paket untuk Multiple Kernel

```
virtualpkg=linux:linux6.6
virtualpkg=linux-headers:linux6.6-headers
```

### Menampilkan Paket Virtual yang Aktif

```bash
xbps-query -L
xbps-query -Rs nama-paket
```
Output akan menunjukkan bahwa paket virtual diredirect ke paket sebenarnya.

## 📊 Ringkasan Perintah

| Fungsi | Perintah |
|--------|----------|
| **Downgrade (mudah)** | `sudo xdowngrade /path/to/pkg-1.0_1.xbps` |
| **Downgrade (manual)** | `sudo xbps-rindex -a /path/to/pkg-1.0_1.xbps` <br> `sudo xbps-install -R /path/to/repo -f pkg-1.0_1` |
| **Hold paket** | `sudo xbps-pkgdb -m hold <pkg>` |
| **Unhold paket** | `sudo xbps-pkgdb -m unhold <pkg>` |
| **Repolock paket** | `sudo xbps-pkgdb -m repolock <pkg>` |
| **Repounlock paket** | `sudo xbps-pkgdb -m repounlock <pkg>` |
| **Ignore paket** | Tambahkan `ignorepkg=nama-paket` di `/etc/xbps.d/*.conf` |
| **Virtual paket** | Tambahkan `virtualpkg=virtual:real` di `/etc/xbps.d/*.conf` |

---

## ⚠️ Catatan Penting untuk T4n OS

| Fitur | Peringatan |
|-------|------------|
| **Downgrade** | Pastikan versi lama kompatibel dengan dependensi sistem saat ini |
| **Hold** | Jangan hold paket sistem kritis (seperti `glibc`, `xbps`, `runit`) |
| **Repolock** | Repositori lokal yang digunakan harus tetap ada saat ingin update |
| **Ignore** | Hanya gunakan jika benar-benar paham konsekuensinya; bisa membuat sistem tidak stabil |
| **Virtual** | Jangan membuat virtual paket yang konflik dengan paket asli yang ada |
