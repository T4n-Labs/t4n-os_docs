# 6. Fitur Lanjutan xbps-src

Setelah menguasai dasar, Anda bisa memanfaatkan fitur lanjutan untuk pembuatan paket yang lebih kompleks.

---

## 📦 Split Packages (Subpaket)

Anda dapat memecah satu source menjadi beberapa paket biner (misal: library terpisah dari binary utama).

### Contoh template dengan subpaket:

```bash
pkgname=myapp
version=1.0
revision=1
...

# Subpaket - library
libmyapp_package() {
    short_desc+=" - shared library"
    depends="${pkgname}>=${version}_${revision}"
    pkg_install() {
        vbin ${wrksrc}/lib/libmyapp.so /usr/lib
    }
}

# Subpaket - development files
libmyapp-devel_package() {
    short_desc+=" - development files"
    depends="libmyapp>=${version}_${revision}"
    pkg_install() {
        vinstall ${wrksrc}/include/myapp.h 644 usr/include
    }
}
```

### Aturan subpaket:
- Setiap fungsi `_package` harus diakhiri dengan `_package()`
- Didalamnya, gunakan `pkg_install()` untuk menyalin file
- `short_desc` akan otomatis ditambahkan dari short_desc utama

## 🏷️ Virtual Package dan Konflik

### Virtual package (provides)

```bash
provides="mail-agent=1.0"
```

### Konflik

```bash
conflicts="otherpkg<=2.0"
replaces="oldpkg<1.5"
```

Ini berguna ketika paket menggantikan paket lain (misal ketika package rename).

## 🔐 Restricted Packages (Build Only)

Untuk paket yang tidak boleh didistribusikan (karena lisensi atau ukuran), set:

```bash
restricted=yes
```

Maka paket hanya bisa dibangun lokal jika `XBPS_ALLOW_RESTRICTED=yes` di `etc/conf`.

---

## 🌍 Cross Compilation

`xbps-src` mendukung cross compilation. Gunakan:

```bash
./xbps-src -a aarch64 pkg myapp
```

Arsitektur yang didukung: `x86_64`, `i686`, `aarch64`, `armv7l`, `armv6l`, `ppc64le`, `ppc64`, `ppc`, `mips`, `mipsel`

### Template harus mendukung cross:

- Jika package menggunakan `hostmakedepends` dengan benar, otomatis berfungsi
- Untuk kasus khusus, gunakan `XBPS_TARGET_MACHINE` di template.

---

## 📂 Bekerja dengan Multiple Distfiles

Jika source terdiri dari beberapa tarball:

```bash
distfiles="https://example.com/pkg-${version}.tar.gz
 https://example.com/extra-${version}.tar.gz"
checksum="sha256_1
 sha256_2"
```

Akses masing-masing di `wrksrc` dan `wrksrc_extra`.

---

## 🧪 Testing Package

Jika paket menyediakan test suite, jalankan:

```bash
./xbps-src check mypkg
```

Test akan dijalankan setelah build, sebelum install. Dependensi untuk testing didefinisikan di `checkdepends`.

---

## 🔌 Integrasi dengan VUR (Void User Repository)

Untuk menggunakan template dari VUR (T4n Labs), Anda bisa:

1. Clone VUR:
   ```bash
   git clone https://github.com/T4n-Labs/vur ~/vur
   ```

2. Copy template ke `void-packages/srcpkgs/`:
   ```bash
   cp -r ~/vur/extra/discord ~/void-packages/srcpkgs/
   ```

3. Build seperti biasa.

Atau gunakan Let-X (VUR Helper) untuk otomatis mengunduh dan menautkan.

```bash
letx get discord
cd ~/void-packages
ln -s ~/.config/letx/extra/discord srcpkgs/discord
./xbps-src pkg discord
```

## 🐳 Menggunakan Container (podman/docker)

Untuk isolasi total, Anda bisa menjalankan `xbps-src` dalam container:

```bash
podman run -it -v $PWD:/work --workdir /work voidlinux/void-linux-full ./xbps-src pkg myapp
```

## ⚡ Tips Performa

| Masalah | Solusi |
|---------|--------|
| Build lambat | Set `XBPS_MAKEJOBS=$(nproc)` di `etc/conf` |
| Kehabisan RAM | Build satu per satu, atau kurangi job |
| Ruang disk habis | Hapus `masterdir` dan `hostdir` secara periodik |
| Build gagal karena cache | `./xbps-src clean <pkg>` |
