# 2. Struktur Template xbps-src

Template adalah file teks yang memberi tahu `xbps-src` bagaimana cara membangun sebuah paket. Setiap paket memiliki direktori sendiri di `srcpkgs/`.

---

## 📁 Layout Direktori Paket

```
srcpkgs/
└── <nama-paket>/
    ├── template          # WAJIB - file utama
    ├── files/            # Opsional - file tambahan (konfigurasi, script)
    ├── patches/          # Opsional - patch untuk source code
    └── INSTALL           # Opsional - script pasca-instalasi
```

Contoh untuk paket `myapp`:
```
srcpkgs/myapp/
├── template
├── files/
│   ├── myapp.conf
│   └── myapp.desktop
├── patches/
│   └── fix-build.patch
└── INSTALL
```

---

## 📝 Variabel Wajib dalam Template

| Variabel | Contoh | Keterangan |
|----------|--------|------------|
| `pkgname` | `myapp` | Nama paket (hurut kecil, tanpa underscore) |
| `version` | `1.2.3` | Versi upstream (tanpa revisi) |
| `revision` | `1` | Revisi paket (naikkan jika template berubah, bukan versi upstream) |
| `build_style` | `gnu-configure` | Metode build (lihat daftar di cheatsheet) |
| `short_desc` | `"A simple application"` | Deskripsi singkat (<80 karakter, pakai tanda kutip) |
| `maintainer` | `"Your Name <email>"` | Email maintainer |
| `license` | `GPL-3.0-or-later` | Lisensi (gunakan SPDX) |
| `homepage` | `https://example.com` | URL proyek |
| `distfiles` | `"https://example.com/${pkgname}-${version}.tar.gz"` | Source tarball |
| `checksum` | `sha256...` | Hash SHA256 (gunakan `xbps-uhelper fetch` untuk dapatkan) |

---

## 🔧 Variabel Opsional

| Variabel | Contoh | Fungsi |
|----------|--------|--------|
| `wrksrc` | `"${pkgname}-${version}"` | Nama direktori source setelah ekstrak (jika berbeda dari default) |
| `hostmakedepends` | `"pkg-config glib-devel"` | Tools yang berjalan di host (dibutuhkan saat build) |
| `makedepends` | `"gtk+3-devel"` | Library yang dibutuhkan untuk compile (hanya saat build) |
| `depends` | `"desktop-file-utils"` | Runtime dependencies (diinstall bersama paket) |
| `conflicts` | `"otherpkg"` | Paket yang konflik dengan ini |
| `replaces` | `"oldpkg<=1.0"` | Paket yang digantikan |
| `provides` | `"virtual-pkg=1.0"` | Virtual package yang disediakan |
| `nopie` | `yes` | Nonaktifkan PIE (untuk paket tertentu) |
| `nostrip` | `yes` | Jangan strip binary (untuk debug) |
| `make_build_args` | `"V=1"` | Argumen tambahan untuk `make build` |
| `make_install_args` | `"DESTDIR=${DESTDIR}"` | Argumen untuk `make install` |

## 📄 Contoh Template Lengkap

### Template untuk aplikasi sederhana (build style = gnu-configure)

```bash
# Template file for myapp
pkgname=myapp
version=1.2.3
revision=1
build_style=gnu-configure
hostmakedepends="pkg-config"
makedepends="gtk+3-devel"
depends="desktop-file-utils"
short_desc="A simple example application"
maintainer="T4n User <user@t4n.org>"
license="MIT"
homepage="https://example.com/myapp"
distfiles="https://example.com/downloads/${pkgname}-${version}.tar.gz"
checksum=1234567890abcdef...

# Opsi configure kustom
configure_args="--enable-feature --disable-other"

post_install() {
    # Install file konfigurasi tambahan
    vinstall ${FILESDIR}/myapp.conf 644 etc/myapp/
    vinstall ${FILESDIR}/myapp.desktop 644 usr/share/applications/
}
```

### Metapackage (tidak build, hanya dependensi)

```bash
pkgname=my-meta
version=1.0
revision=1
build_style=meta
short_desc="Meta package that pulls dependencies"
maintainer="T4n User <user@t4n.org>"
license="Public Domain"
depends="app1 app2 app3"
```

## 🛠️ Fungsi-fungsi Khusus

| Fungsi | Kapan dijalankan | Kegunaan |
|--------|------------------|----------|
| `pre_fetch()` | Sebelum download distfiles | Modifikasi URL atau variabel |
| `post_fetch()` | Setelah download | Verifikasi tambahan |
| `pre_extract()` | Sebelum ekstrak | Persiapan |
| `post_extract()` | Setelah ekstrak | Pindahkan file, hapus file tidak perlu |
| `pre_configure()` | Sebelum configure | Patch atau modifikasi source |
| `post_configure()` | Setelah configure | - |
| `pre_build()` | Sebelum build | - |
| `post_build()` | Setelah build | - |
| `pre_install()` | Sebelum install ke DESTDIR | - |
| `post_install()` | Setelah install ke DESTDIR | Install file tambahan, perbaikan |
| `do_configure()` | Override configure step | Untuk build system non-standar |
| `do_build()` | Override build step | - |
| `do_install()` | Override install step | - |

### Contoh penggunaan `post_install`

```bash
post_install() {
    # Install file dari direktori files/ (FILESDIR adalah path ke files/)
    vinstall ${FILESDIR}/app.conf 644 etc/app/
    
    # Buat symlink
    vlink usr/bin/app usr/bin/app-bin
    
    # Hapus file tidak perlu
    rm ${DESTDIR}/usr/share/doc/README
    
    # Set permission khusus
    chmod 755 ${DESTDIR}/usr/libexec/app-helper
}
```

## ✅ Validasi Template

Sebelum build, Anda bisa memeriksa sintaks template:

```bash
./xbps-src show-pkg myapp
```

Ini akan menampilkan semua variabel yang dievaluasi.

---

## 📦 Membuat Template Baru

1. Buat direktori untuk paket Anda:
   ```bash
   mkdir -p srcpkgs/myapp
   ```

2. Buat file `template` di dalamnya dengan editor.

3. Isi variabel wajib minimal: `pkgname`, `version`, `revision`, `build_style`, `short_desc`, `maintainer`, `license`, `homepage`, `distfiles`, `checksum`.

4. (Opsional) Tambahkan file di `files/` jika perlu.

5. Test build:
   ```bash
   ./xbps-src pkg myapp
   ```
