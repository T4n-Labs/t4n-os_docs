# 5. Patch pada xbps-src

Patch digunakan untuk memodifikasi source code asli sebelum dibangun. Ini diperlukan ketika:
- Ada perbaikan yang belum masuk ke upstream
- Menyesuaikan dengan Void Linux (misalnya path, compiler flags)
- Menghapus fitur yang tidak diinginkan

---

## 📁 Struktur Patch

```
srcpkgs/mypackage/
├── patches/
│   ├── fix-build.patch
│   ├── fix-path.patch
│   └── series (opsional)
└── template
```

`xbps-src` secara otomatis akan **menerapkan semua file `.patch`** yang ditemukan di `patches/` setelah ekstrak source.

---

## 🛠️ Membuat Patch

### Metode 1: Dengan diff (direkomendasikan)

1. Ekstrak source normal:
   ```bash
   cd void-packages
   ./xbps-src extract mypkg
   ```

2. Masuk ke direktori source:
   ```bash
   cd masterdir/builddir/mypkg-version
   ```

3. Buat salinan file asli, lalu edit file yang perlu diubah:
   ```bash
   cp file.c file.c.orig
   nano file.c
   ```

4. Buat patch:
   ```bash
   diff -Naur file.c.orig file.c > ~/fix-something.patch
   ```

5. Pindahkan ke `srcpkgs/mypkg/patches/`:
   ```bash
   cp ~/fix-something.patch ../../../../../srcpkgs/mypkg/patches/
   ```

### Metode 2: Simpan di `files/` lalu generate patch

```bash
# Simpan file asli di files/file.c.orig
# Edit file setelah extract
# Gunakan diff -Naur
```

---

## 📝 Format Patch (unified diff)

Patch yang baik memiliki format standar:

```diff
--- a/src/main.c.orig
+++ b/src/main.c
@@ -10,7 +10,7 @@
 
 int main() {
-    printf("Hello World\n");
+    printf("Hello T4n OS!\n");
     return 0;
 }
```

---

## 🔢 Urutan Penerapan Patch

Patch diterapkan **sesuai urutan abjad** nama file `.patch`. Untuk mengontrol urutan, beri prefix numerik:

```
patches/
├── 01-fix-build.patch
├── 02-fix-path.patch
├── 10-security.patch
```

Atau gunakan file `series` (opsional) untuk menentukan urutan eksplisit:

```
# file patches/series
fix-build.patch -p1
fix-path.patch -p1
```

---

## 🧪 Menguji Patch Sebelum Build

Gunakan `./xbps-src extract` lalu manual:

```bash
./xbps-src extract mypkg
cd masterdir/builddir/mypkg-version
patch -p1 < /path/to/patch.patch
# Cek apakah tanpa error
```

Jika berhasil, patch akan otomatis diterapkan saat `./xbps-src pkg`.

---

## 🚫 Menonaktifkan Patch

Jika patch menyebabkan masalah, Anda bisa sementara menonaktifkan dengan:

```bash
./xbps-src -N pkg mypkg   # -N = no patches
```

Atau rename file patch menjadi `.patch.disabled`.

---

## 🔍 Patch untuk File Biner (Jarang)

Untuk file biner (misal firmware), gunakan `bsdiff` dan simpan file `.bsdiff` di `patches/`. `xbps-src` akan mendeteksi ekstensi `.bsdiff` dan menggunakan `bspatch`.

---

## 🧹 Membersihkan Patch yang Tidak Terpakai

Setelah template diperbarui ke versi upstream baru, beberapa patch mungkin tidak relevan. Cek dengan:

```bash
./xbps-src extract mypkg
cd masterdir/builddir/mypkg-version
for patch in ../../../../srcpkgs/mypkg/patches/*.patch; do
    echo "Testing $patch"
    patch -p1 --dry-run < $patch || echo "FAIL"
done
```

Hapus patch yang gagal (sudah diterapkan upstream atau tidak kompatibel).

---

## ✅ Praktik Terbaik

1. **Beri nama patch deskriptif**: `fix-build-with-gcc14.patch` bukan `patch1.patch`.
2. **Sertakan penjelasan di template** (opsional, dengan komentar).
3. **Jangan buat patch untuk hal yang bisa diselesaikan dengan `configure_args`**.
4. **Kirim patch ke upstream** jika perbaikannya bersifat universal.
5. **Gunakan `-p1`** secara default saat membuat patch dari root source.
