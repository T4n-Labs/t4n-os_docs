# 4. Manajemen Dependensi di xbps-src

Mengelola dependensi adalah aspek krusial dalam membuat template yang baik. Void membedakan tiga jenis dependensi.

---

## 📚 Tiga Jenis Dependensi

| Variabel | Kapan dibutuhkan | Contoh |
|----------|------------------|--------|
| `hostmakedepends` | Alat yang dijalankan **di host** (mesin build) | `pkg-config`, `cmake`, `python3` |
| `makedepends` | Library yang dibutuhkan **saat compile** (hanya untuk build) | `gtk+3-devel`, `openssl-devel` |
| `depends` | Runtime dependencies (wajib ada saat paket dijalankan) | `desktop-file-utils`, `perl` |

---

## 🖥️ hostmakedepends

Tools yang berjalan di sistem **host** (bukan di dalam chroot). Biasanya:

| Kategori | Contoh |
|----------|--------|
| Compiler/Interpreter | `python3`, `gcc`, `rust` |
| Build system | `cmake`, `meson`, `ninja` |
| Code generator | `flex`, `bison`, `glib-devel` |
| Documentation | `doxygen`, `sphinx` |
| Package helpers | `pkg-config`, `install` |

### Contoh:

```bash
hostmakedepends="pkg-config cmake python3 glib-devel"
```

> `pkg-config` hampir selalu diperlukan untuk proyek C/C++ agar bisa menemukan library.

---

## 🔧 makedepends

Library dan header yang diperlukan **saat compile**. Tidak perlu diinstall di sistem target (namun biasanya tetap dibutuhkan saat runtime juga, terpisah).

### Contoh:

Untuk proyek GTK:
```bash
makedepends="gtk+3-devel glib-devel cairo-devel"
```

---

## 🏃 depends

Paket yang harus ada agar aplikasi dapat berjalan dengan benar **setelah diinstall**. Ini adalah runtime dependencies.

### Membedakan makedepends vs depends:

| Jika... | Maka masukkan ke... |
|---------|---------------------|
| Hanya dibutuhkan saat compile, tidak perlu saat jalan | `makedepends` |
| Dibutuhkan saat aplikasi dijalankan | `depends` |
| Alat pembantu di host (bukan target) | `hostmakedepends` |

### Contoh runtime dependencies:

```bash
depends="desktop-file-utils gtk+3 glibc"
```

---

## 🔄 Dependensi Otomatis

`xbps-src` secara otomatis mendeteksi shared library yang diperlukan dari binary yang dihasilkan. Biasanya kita tidak perlu mencantumkan library dasar seperti `glibc` di `depends`.

Namun untuk beberapa kasus (Python modules, Perl modules, atau file konfigurasi), otomatisasi mungkin tidak sempurna.

### Memeriksa dependensi yang terdeteksi:

```bash
./xbps-src show-build-deps myapp
```

---

## 🧪 checkdepends (Testing)

Untuk paket yang memiliki test suite, Anda dapat menambahkan `checkdepends`:

```bash
checkdepends="python3-pytest"
```

Dependensi ini hanya diperlukan saat menjalankan `./xbps-src check <pkg>`.

---

## 📝 Sintaks Penulisan Dependensi

Dependensi ditulis sebagai string dengan spasi sebagai pemisah:

```bash
depends="libfoo>=1.0 libbar-2.0_1 libbaz"
```

Operator versi:
| Operator | Contoh | Arti |
|----------|--------|------|
| `>=` | `foo>=1.2` | versi 1.2 atau lebih baru |
| `<=` | `foo<=2.0` | versi 2.0 atau lebih lama |
| `=`  | `foo=1.0` | versi exact |
|`-`   | `foo-1.0_1` | versi exact dengan revisi |

---

## 🔍 Mencari Nama Paket Dependensi

Untuk package development (`-devel`), biasanya pola:

- **Debian/Ubuntu**: `libfoo-dev` → di Void: `libfoo-devel`
- **Fedora**: `libfoo-devel` → sama

Cari di repositori:

```bash
xbps-query -Rs keyword-devel
```

Contoh:
```bash
xbps-query -Rs gtk+3-devel
```

---

## 🚫 Menghindari Dependensi Melingkar

`xbps-src` akan error jika ada dependensi melingkar (A butuh B, B butuh A). Hindari dengan:
- Memisahkan paket menjadi sub-paket (split packages) - jika perlu.
- Menggunakan `hostmakedepends` untuk alat yang tersedia di host.

---

## ✅ Praktik Terbaik

1. **Jangan masukkan `-devel` packages ke `depends`** (kecuali benar-benar diperlukan saat runtime).
2. **Gunakan `xbps-src show-build-deps` untuk memeriksa apakah dependensi sudah lengkap**.
3. **Cantumkan runtime dependencies minimum** - jangan terlalu banyak, jangan terlalu sedikit.
4. **Gunakan operator versi jika diperlukan** (misal karena API break).

---

## 📋 Ringkasan

| Jenis | Variabel | Contoh isi |
|-------|----------|------------|
| Alat build di host | `hostmakedepends` | `pkg-config cmake` |
| Library compile | `makedepends` | `gtk+3-devel` |
| Runtime | `depends` | `gtk+3` |
| Testing | `checkdepends` | `python3-pytest` |
