# 3. Opsi Build pada xbps-src

`build_style` adalah variabel paling penting dalam template karena menentukan **cara** paket akan dibangun. Void Linux menyediakan puluhan `build_style` untuk berbagai sistem build.

---

## ЁЯПЧя╕П Daftar Build Style Populer

| build_style | Digunakan untuk proyek dengan... |
|-------------|----------------------------------|
| `gnu-configure` | `./configure` (Autotools) |
| `cmake` | CMake (CMakeLists.txt) |
| `meson` | Meson (meson.build) |
| `python3` | Python package (setup.py / pyproject.toml) |
| `perl` | Perl module (Makefile.PL) |
| `go` | Go packages |
| `rust` | Cargo projects |
| `meta` | Metapackage (hanya dependensi, tidak build) |
| `fetch` | Hanya download file (tanpa build) |
| `waf` | Waf build system (waf configure) |
| `gnu-makefile` | Makefile biasa (tanpa configure) |

## ЁЯФз gnu-configure (Paling Umum)

Untuk proyek yang menggunakan `./configure && make && make install`.

### Variabel pendukung:

| Variabel | Contoh | Fungsi |
|----------|--------|--------|
| `configure_args` | `"--enable-shared --disable-static"` | Argumen tambahan untuk `./configure` |
| `make_build_args` | `"V=1"` | Argumen untuk `make` saat build |
| `make_install_args` | `"DESTDIR=${DESTDIR}"` | Argumen untuk `make install` |
| `make_check_args` | `"-k"` | Argumen untuk `make check` |

### Contoh template:

```bash
build_style=gnu-configure
configure_args="--prefix=/usr --sysconfdir=/etc --disable-static"
```

---

## ЁЯУж cmake

Untuk proyek dengan `CMakeLists.txt`.

### Variabel pendukung:

| Variabel | Contoh | Fungsi |
|----------|--------|--------|
| `configure_args` | `"-DCMAKE_BUILD_TYPE=Release"` | Argumen untuk cmake |
| `make_build_args` | `"--parallel ${XBPS_MAKEJOBS}"` | Argumen build |
| `cmake_builddir` | `"build"` | Subdirektori untuk build (default: `build`) |

### Contoh:

```bash
build_style=cmake
configure_args="-DUSE_SSL=ON -DBUILD_TESTING=OFF"
make_build_args="--verbose"
```

---

## ЁЯРН python3

Untuk package Python yang menggunakan `setup.py` atau `pyproject.toml`.

### Variabel pendukung:

| Variabel | Contoh | Fungsi |
|----------|--------|--------|
| `make_build_args` | `"--skip-deps-check"` | Argumen untuk `setup.py build` |
| `make_install_args` | `"--optimize=1"` | Argumen untuk `setup.py install` |
| `pycompile_module` | `"myapp"` | Modul yang akan dikompilasi (dihasilkan .pyc) |
| `python3_version` | `3.11` | Versi Python target (jika spesifik) |

### Contoh:

```bash
build_style=python3
hostmakedepends="python3-setuptools"
depends="python3-requests"
```

## ЁЯжА rust / go

Untuk proyek Rust (Cargo) atau Go.

### Rust:

```bash
build_style=cargo
configure_args="--features=ssl"
make_build_args="--release"
```

### Go:

```bash
build_style=go
go_import_path="github.com/user/project"
```

## тЪЩя╕П Opsi Kustom (vopt)

`vopt` (void options) memungkinkan pengguna mengaktifkan/menonaktifkan fitur saat build.

### Mendefinisikan opsi:

```bash
# Di dalam template
build_options="alsa pulseaudio wayland"
build_options_default="alsa"   # yang aktif secara default

# Konflik antar opsi (misal: hanya satu dari dua)
vopt_conflict alsa pulseaudio
```

### Menggunakan opsi dalam template:

```bash
configure_args=$(vopt_enable alsa) $(vopt_enable pulseaudio) $(vopt_enable wayland)

if [ "$build_option_extra" ]; then
    configure_args+=" --enable-extra-feature"
fi
```

### Memanggil build dengan opsi:

```bash
./xbps-src -o alsa,wayland pkg myapp
```

## ЁЯОЫя╕П Override Build Step (do_* functions)

Jika `build_style` tidak cukup, Anda bisa override fungsi build dengan menulisnya di template.

### Contoh: Build manual dengan `do_build()`

```bash
build_style=gnu-configure  # tetap bisa dipakai, tapi kita override

do_build() {
    # Perintah build kustom
    cd ${wrksrc}
    gcc -o myapp main.c -lssl
}

do_install() {
    # Perintah install kustom
    vbin myapp /usr/bin
    vdoc README
}
```

## ЁЯЫая╕П Tips Memilih Build Style

1. **Lihat dari source proyek**:
   - Ada `configure`? тЖТ `gnu-configure`
   - Ada `CMakeLists.txt`? тЖТ `cmake`
   - Ada `meson.build`? тЖТ `meson`
   - Hanya Makefile? тЖТ `gnu-makefile`

2. **Lihat template serupa**:
   ```bash
   ./xbps-src search-pkg keyword
   cat srcpkgs/similar-package/template
   ```

3. **Fallback**:
   - Jika tak ada system build, gunakan `do_build()` manual.
