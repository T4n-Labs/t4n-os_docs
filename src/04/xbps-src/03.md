# 1. Instalasi dan Setup xbps-src

Dokumen ini akan memandu Anda menyiapkan `xbps-src` di T4n OS.

---

## 📦 Prasyarat

| Prasyarat | Minimal |
|-----------|---------|
| Ruang disk | 10 GB (lebih untuk build berat) |
| RAM | 4 GB (8 GB direkomendasikan) |
| Koneksi internet | Untuk download source dan bootstrap |
| Git | `sudo xbps-install -S git` |
| `xtools` (opsional tapi berguna) | `sudo xbps-install -S xtools` |

---

## 🔧 Langkah Instalasi

### 1. Clone repositori void-packages

```bash
git clone https://github.com/void-linux/void-packages.git
cd void-packages
```

Repositori ini berisi semua template build untuk paket Void Linux. Ukuran clone sekitar 200 MB.

### 2. Bootstrap

**Untuk x86_64 (glibc) - mayoritas pengguna T4n OS:**
```bash
./xbps-src binary-bootstrap
```

**Untuk arsitektur lain:**
```bash
./xbps-src binary-bootstrap <arch>
# Contoh: ./xbps-src binary-bootstrap aarch64
```

Proses bootstrap akan:
- Mengunduh toolchain (gcc, binutils, dll)
- Membuat `masterdir` (environment chroot)
- Menginstall dependensi dasar build

> ⏱️ Bootstrap memakan waktu 5-15 menit tergantung koneksi.

### 3. Verifikasi

```bash
./xbps-src version
# Output: xbps-src: <versi>
```

### 4. (Opsional) Konfigurasi

Buat file `etc/conf` untuk pengaturan kustom:

```bash
echo 'XBPS_ALLOW_RESTRICTED=yes' >> etc/conf       # Izinkan build paket restricted
echo 'XBPS_MAKEJOBS=4' >> etc/conf                # Jumlah job parallel (sesuai CPU)
echo 'XBPS_CHROOT_CMD=uchroot' >> etc/conf        # Metode chroot (default: uunshare)
```

---

## 🏗️ Memahami Struktur Direktori

Setelah bootstrap, direktori `void-packages` akan memiliki:

| Direktori/File | Fungsi |
|----------------|--------|
| `srcpkgs/` | Tempat semua template paket (subdirektori per paket) |
| `masterdir/` | Environment chroot untuk build (bisa dihapus aman) |
| `hostdir/` | Hasil build dan logs |
| `hostdir/binpkgs/` | Paket `.xbps` yang sudah jadi |
| `hostdir/logs/` | Log build |
| `common/` | Script dan fungsi bersama |
| `etc/conf` | Konfigurasi lokal (buat sendiri) |

---

## 🔄 Update xbps-src

Karena `void-packages` adalah repositori git, update dilakukan dengan:

```bash
git pull --rebase
./xbps-src bootstrap-update
```

> Selalu lakukan `bootstrap-update` setelah `git pull` untuk menyinkronkan perubahan environment.

---

## 🗑️ Menghapus xbps-src (Reset Total)

```bash
cd void-packages
rm -rf masterdir hostdir
git clean -fdx
git reset --hard origin/master
```

Ini akan menghapus semua hasil build dan mengembalikan ke kondisi bersih.

---

## ❓ Masalah Umum Saat Instalasi

### `ERROR: xbps-uchroot: Operation not permitted`

**Solusi:** Pastikan user Anda termasuk grup `xbuilder`:
```bash
sudo usermod -a -G xbuilder $USER
# Logout dan login kembali
```

### `bootstrap: failed to install base-chroot`

**Solusi:** Cek koneksi internet, lalu ulangi bootstrap:
```bash
./xbps-src bootstrap-update
```

### `masterdir already exists`

**Solusi:** Hapus manual atau gunakan flag `-f`:
```bash
rm -rf masterdir
./xbps-src binary-bootstrap
```
