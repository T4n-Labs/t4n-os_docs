## `xbps-create` pada Void Linux

`xbps-create` adalah utilitas dalam package manager XBPS yang berfungsi untuk **membuat paket biner XBPS (.xbps)** dari file-file yang disimpan dalam suatu direktori .


## 🎯 Fungsi Utama `xbps-create`

Utilitas ini mengambil file-file yang sudah disusun dalam struktur direktori tertentu, lalu mengemasnya menjadi satu file paket `.xbps` yang siap diinstal dengan `xbps-install` .

Proses kerjanya sederhana: Anda menyiapkan direktori (biasanya disebut `destdir`) yang berisi file-file dengan struktur seperti di sistem tujuan (misalnya `usr/bin/`, `etc/`, `usr/share/`), lalu `xbps-create` akan mengubah seluruh isi direktori tersebut menjadi paket biner .


## 🔧 Parameter Penting

`xbps-create` memiliki banyak opsi untuk mendefinisikan metadata paket. Berikut parameter yang **wajib** dan **penting**:

### Parameter Wajib

| Parameter | Fungsi | Contoh |
|-----------|--------|--------|
| `-n, --pkgver` | Nama paket dan versi (format: `nama-version_revision`) | `myapp-1.0_1`  |
| `-A, --architecture` | Arsitektur target | `x86_64`, `i686`, `noarch`  |
| `-s, --desc` | Deskripsi singkat (satu baris, <80 karakter) | `"A simple text editor"`  |

### Parameter Penting Lainnya

| Parameter | Fungsi | Contoh |
|-----------|--------|--------|
| `-D, --dependencies` | Daftar paket yang dibutuhkan | `'libfoo>=1.0 libbar-2.0_1'`  |
| `-H, --homepage` | URL homepage proyek | `https://example.com`  |
| `-l, --license` | Lisensi paket | `GPL-3.0-or-later`  |
| `-m, --maintainer` | Maintainer paket | `John Doe <john@example.com>`  |
| `-S, --long-desc` | Deskripsi panjang  |
| `-C, --conflicts` | Paket yang konflik dengan paket ini  |
| `-R, --replaces` | Paket yang digantikan oleh paket ini  |
| `--shlib-provides` | Shared library yang disediakan  |
| `--shlib-requires` | Shared library yang dibutuhkan  |
| `--alternatives` | Konfigurasi `xbps-alternatives`  |
| `--compression` | Format kompresi | `zstd` (default), `gzip`, `xz`, `lz4`  |


## 💡 Siapa yang Membutuhkan `xbps-create`?

| Pengguna | Kegunaan |
|----------|----------|
| **Pengembang/maintainer paket Void** | Membuat paket resmi Void Linux (biasanya melalui `xbps-src`)  |
| **Pengguna yang mem-build software sendiri** | Membungkus hasil kompilasi menjadi paket `.xbps` untuk instalasi yang rapi |
| **Konversi dari format lain** | Tools seperti `deb2xbps` menggunakan `xbps-create` untuk mengkonversi paket Debian (.deb) ke format XBPS  |

Untuk konteks sehari-hari sebagai pengguna biasa, Anda **tidak perlu** menggunakan `xbps-create` secara langsung. Ini adalah alat untuk pembuat paket (package creator).


## 📋 Contoh Penggunaan Sederhana

Misalkan Anda ingin membuat paket untuk program sederhana:

```bash
# 1. Siapkan struktur direktori destdir
mkdir -p myapp-1.0_1/usr/bin
mkdir -p myapp-1.0_1/usr/share/myapp

# 2. Copy file program ke destdir
cp myprogram myapp-1.0_1/usr/bin/
cp data/* myapp-1.0_1/usr/share/myapp/

# 3. Buat paket .xbps
xbps-create \
    -n myapp-1.0_1 \
    -A x86_64 \
    -s "My awesome application" \
    -S "My application does amazing things for you." \
    -H "https://example.com/myapp" \
    -l "MIT" \
    -m "Your Name <you@example.com>" \
    -D 'libcares>=1.0' \
    myapp-1.0_1
```

Perintah di atas akan menghasilkan file `myapp-1.0_1.x86_64.xbps` .


## 📂 Hubungan dengan Tools XBPS Lainnya

Setelah paket biner dibuat, Anda bisa:
- **Mengindeks**: `xbps-rindex -a myapp-1.0_1.x86_64.xbps` untuk menambahkannya ke repositori lokal 
- **Menginstal**: `xbps-install --repository=/path/to/repo myapp` 
- **Memeriksa isi**: `xbps-query -f myapp`


## ⚠️ Catatan Penting

1. **Hak akses file** - File dalam `destdir` harus memiliki permission yang benar, karena akan digunakan langsung saat instalasi 
2. **Tidak boleh ada hardlink di root** - Hati-hati membuat paket yang mencakup direktori seperti `/bin`, `/lib`, `/sbin` karena bisa mengganggu symbolic link sistem 
3. **File yang tidak umum** - `xbps-create` akan menolak file selain regular file, symlink, dan direktori (misalnya socket, fifo) 
4. **Format paket** - Secara default, paket dikompres dengan `zstd` 


## 🔗 Kesimpulan

`xbps-create` adalah fondasi pembuatan paket di ekosistem XBPS - alat yang memungkinkan seluruh ekosistem paket Void Linux ada. Meskipun pengguna akhir jarang menggunakannya langsung (karena `xbps-src` menangani kompleksitas building), `xbps-create` tetap menjadi komponen penting di balik layar yang memungkinkan distribusi software terstruktur dan mudah dikelola di Void Linux.
