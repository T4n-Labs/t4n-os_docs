## `xbps-fetch` pada Void Linux

`xbps-fetch` adalah utilitas dalam XBPS yang berfungsi untuk **mengunduh file dari URL** . Ini adalah alat baris perintah sederhana untuk mengambil file dari internet, mirip dengan `curl` atau `wget`, tetapi terintegrasi langsung dengan ekosistem XBPS dan menggunakan library `libfetch` milik NetBSD .


## 🎯 Fungsi Utama

`xbps-fetch` dapat digunakan secara mandiri oleh pengguna untuk mengunduh file, tetapi lebih sering digunakan **secara internal oleh XBPS** saat:
- Mengunduh paket biner dari repositori remote
- Mengunduh source code (distfiles) untuk proses build

Dari perspektif teknis (library API), fungsi ini memiliki kemampuan untuk:
- Mendownload file ke direktori kerja saat ini 
- Menyimpan file dengan nama tertentu (`xbps_fetch_file_dest`) 
- Menghitung SHA256 hash dari file yang didownload 
- Mendeteksi apakah file lokal identik dengan remote (tidak perlu download ulang) 


## 🔧 Protokol yang Didukung

| Protokol | Dukungan |
|----------|----------|
| `http://` | ✅ Ya |
| `https://` | ✅ Ya |
| `ftp://` | ✅ Ya |
| `socks5://` | ✅ Ya  |


## 📋 Parameter Penting

| Parameter | Fungsi |
|-----------|--------|
| `-o <output>` | Menyimpan file dengan nama tertentu (rename) |
| `-v` | Mode verbose (menampilkan informasi lebih detail) |
| `-s` | Menampilkan SHA256 sum dari file yang diunduh  |
| `-d` | Mode debug (menampilkan pesan debug ke stderr) |
| `-h` | Menampilkan pesan bantuan |
| `-V` | Menampilkan informasi versi |

### Contoh Penggunaan

```bash
# Download file ke direktori saat ini
xbps-fetch https://example.com/file.tar.gz

# Download dengan rename output
xbps-fetch -o myfile.tar.gz https://example.com/file.tar.gz

# Download dengan verbose dan tampilkan SHA256
xbps-fetch -v -s https://example.com/file.tar.gz
```


## 🔧 Konfigurasi via Environment Variables

`xbps-fetch` mendukung banyak environment variables untuk konfigurasi proxy, autentikasi, dan SSL/TLS :

| Variable | Fungsi |
|----------|--------|
| `HTTP_PROXY` | Proxy untuk koneksi HTTP |
| `HTTPS_PROXY` | Proxy untuk koneksi HTTPS |
| `FTP_PROXY` | Proxy untuk koneksi FTP |
| `SOCKS_PROXY` | Proxy untuk koneksi SOCKS |
| `HTTP_AUTH` | Autentikasi ke HTTP server (`user:pass`) |
| `HTTP_USER_AGENT` | Mengganti User-Agent default |
| `SSL_CA_CERT_FILE` | File CA certificates (default: `/etc/ssl/certs/ca-certificates.crt`) |
| `SSL_NO_VERIFY_PEER` | Menonaktifkan verifikasi SSL peer |
| `CONNECTION_TIMEOUT` | Timeout koneksi dalam milidetik (default: 5 menit) |

### Contoh Setting Proxy
```bash
export HTTP_PROXY=http://user:pass@proxy.example.com:3128
export HTTPS_PROXY=http://user:pass@proxy.example.com:3128
xbps-fetch https://example.com/file
```


## ⚙️ Cara Kerja Internal

Ketika `xbps-fetch` mengunduh file, ada beberapa skenario hasil :

| Kode Hasil | Makna |
|------------|-------|
| `1` | File berhasil diunduh |
| `0` | File lokal identik dengan remote (tidak perlu diunduh) |
| `-1` | Terjadi error |

Perilaku ini berguna untuk proses build dan update karena mencegah download ulang file yang sudah ada dan masih valid .


## 🔗 Sejarah dan Posisi di XBPS

`xbps-fetch` adalah utilitas yang **menggantikan `xbps-uhelper fetch`** . Kini `xbps-fetch` menjadi utility mandiri yang terpisah, memberikan fungsionalitas download yang lebih baik dan terintegrasi dengan ekosistem XBPS.


## ⚠️ Catatan Penting

1. **Perilaku URL encoding** - `xbps-fetch` meng-encode karakter tertentu (seperti `~` menjadi `%7e`) yang kadang menyebabkan kegagalan download pada server tertentu yang sensitif terhadap encoding 

2. **Integrasi sistem** - Utility ini menggunakan file CA certificates default di `/etc/ssl/certs` untuk verifikasi SSL/TLS 

3. **Exit status** - Mengembalikan `0` jika sukses, `>0` jika terjadi error 


## 🔗 Kesimpulan

`xbps-fetch` adalah utility download bawaan XBPS yang mendukung berbagai protokol (HTTP, HTTPS, FTP, SOCKS5), dapat dikonfigurasi via environment variables untuk proxy dan autentikasi, serta memiliki kemampuan deteksi file identik untuk menghindari download ulang. Utilitas ini menggantikan `xbps-uhelper fetch` dan menjadi komponen penting dalam proses instalasi paket dan build di Void Linux.
