## `xbps-uunshare` pada Void Linux

`xbps-uunshare` adalah utilitas dalam XBPS yang berfungsi untuk melakukan **chroot dan bind mount** ke dalam direktori target dengan memanfaatkan **Linux user namespaces** . Keunggulan utama dari pendekatan ini adalah pengguna **tidak memerlukan hak akses root** untuk menjalankannya .


## ЁЯОп Fungsi Utama

Utilitas ini memungkinkan Anda untuk:

| Fungsi | Penjelasan |
|--------|------------|
| **Chroot ke direktori target** | Berpindah ke lingkungan root yang terisolasi |
| **Mount pseudo-filesystem otomatis** | Secara otomatis mount `/dev`, `/proc`, dan `/sys` ke dalam chroot  |
| **Bind mount direktori kustom** | Memasang direktori dari host ke dalam chroot dengan opsi `-b`  |
| **Isolasi dengan user namespaces** | Memisahkan IPC, PIDs, dan mount points tanpa perlu hak istimewa  |

`xbps-uunshare` adalah **metode default** yang digunakan oleh `xbps-src` (sistem build paket Void Linux) untuk membuat lingkungan build yang terisolasi . Pengguna tidak perlu menjadi `root` untuk menggunakan `xbps-src` berkat utilitas ini.


## ЁЯФз Parameter Penting

| Parameter | Fungsi |
|-----------|--------|
| `-b src:dest` | Bind mount `src` dari host ke `CHROOTDIR/dest`. `src` dan `dest` harus path absolut dan sudah ada  |
| `--` | Menghentikan interpretasi argumen berikutnya sebagai opsi. Berguna jika argumen `COMMAND` mengandung opsi sendiri  |

### Contoh Penggunaan

```bash
# Chroot sederhana dengan mount otomatis /proc, /sys, /dev
xbps-uunshare /mnt/void /bin/bash

# Bind mount direktori host ke dalam chroot
xbps-uunshare -b /home/user:/home/user /mnt/void /bin/bash

# Menjalankan command tertentu di dalam chroot
xbps-uunshare /mnt/void /usr/bin/ls -la
```


## ЁЯФР Keamanan: User Namespaces

Fitur pembeda utama `xbps-uunshare` adalah penggunaan **user namespaces** :

| Aspek | Penjelasan |
|-------|------------|
| **Hak akses** | Tidak perlu `root` atau `sudo`  |
| **Isolasi** | Memisahkan IPC, PID, dan mount points dari proses pemanggil  |
| **Container ringan** | Dapat membuat container terisolasi tanpa hak istimewa  |


## тЪЩя╕П Prasyarat Kernel

`xbps-uunshare` memerlukan opsi kernel Linux berikut :

| Opsi Kernel | Fungsi |
|-------------|--------|
| `CONFIG_NAMESPACES` | Dukungan namespace secara umum |
| `CONFIG_IPC_NS` | Isolasi IPC (Inter-Process Communication) |
| `CONFIG_UTS_NS` | Isolasi hostname/domainname |
| `CONFIG_USER_NS` | Dukungan user namespaces (kunci agar tidak perlu root) |

Kernel Void Linux default sudah mengaktifkan semua opsi ini. Jika sistem tidak mendukung opsi yang diperlukan, `xbps-uunshare` akan gagal dengan error `EINVAL (Invalid argument)` .


## ЁЯФД Perbandingan: `xbps-uunshare` vs `xbps-uchroot`

`xbps-src` mendukung beberapa utilitas chroot, dengan `xbps-uunshare` sebagai default :

| Aspek | `xbps-uunshare` | `xbps-uchroot` |
|-------|-----------------|-----------------|
| **Hak akses** | Tidak perlu root (user namespace) | Perlu setgid dan grup khusus (`xbuilder`)  |
| **Kernel options** | `CONFIG_USER_NS` diperlukan | `CONFIG_PID_NS` diperlukan (bukan `CONFIG_USER_NS`)  |
| **Default di xbps-src** | тЬЕ Ya | тЭМ Tidak |
| **Portabilitas** | Linux-specific (tidak portable ke OS lain)  | Linux-specific |

### Mengganti metode chroot di `xbps-src`

Jika ingin beralih ke `xbps-uchroot` (misalnya karena alasan performa atau kompatibilitas), Anda cukup mengubah konfigurasi:

```bash
cd void-packages
echo XBPS_CHROOT_CMD=uchroot >> etc/conf
```


## ЁЯТб Siapa yang Membutuhkan `xbps-uunshare`?

| Pengguna | Kebutuhan |
|----------|-----------|
| **Pengembang paket Void** | Membangun paket dengan `xbps-src` tanpa perlu `root`  |
| **Contributor void-packages** | Testing paket dalam lingkungan build yang aman |
| **Power user** | Membangun sendiri paket dari source tanpa hak istimewa |

Untuk penggunaan sehari-hari (instalasi, update, hapus paket), Anda **tidak perlu** menggunakan `xbps-uunshare` secara langsung. Utilitas ini bekerja di belakang layar saat Anda menjalankan `xbps-src`.


## тЪая╕П Troubleshooting

Jika muncul error `Invalid argument (EINVAL)`, kemungkinan kernel tidak mendukung opsi yang diperlukan. Untuk mengecek dukungan user namespaces:

```bash
# Cek apakah user namespace tersedia
cat /proc/sys/kernel/unprivileged_userns_clone
```

Jika nilainya `0`, Anda mungkin perlu mengaktifkannya (atau beralih ke `xbps-uchroot` sebagai alternatif).


## ЁЯФЧ Kesimpulan

`xbps-uunshare` adalah komponen penting dalam ekosistem pembangunan paket Void Linux yang memungkinkan:

| Kemampuan | Manfaat |
|-----------|---------|
| **Chroot tanpa root** | Pengguna biasa dapat membangun paket dengan aman |
| **Isolasi container** | Lingkungan build yang bersih dan terpisah dari host |
| **Integrasi dengan xbps-src** | Metode default untuk membangun paket Void Linux |

Sederhananya: **Jika Anda ingin membangun paket dari source di Void Linux menggunakan `xbps-src`, `xbps-uunshare` adalah alat yang memungkinkan Anda melakukannya tanpa perlu hak akses root.** Ini adalah fondasi keamanan dan kemudahan dalam ekosistem pembangunan paket Void Linux.
