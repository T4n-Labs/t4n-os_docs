## `xbps-reconfigure` pada Void Linux

`xbps-reconfigure` adalah utilitas dalam XBPS yang berfungsi untuk **menjalankan ulang skrip konfigurasi paket yang sudah terinstall** di sistem . Ini adalah alat penting ketika paket mengalami masalah konfigurasi atau ketika Anda perlu menerapkan ulang perubahan konfigurasi .

---

## 🎯 Fungsi Utama

Ketika sebuah paket diinstall di Void Linux, paket tersebut memiliki **skrip INSTALL** yang berisi langkah-langkah konfigurasi pasca-instalasi . `xbps-reconfigure` menjalankan ulang bagian `post` dari skrip tersebut.

Proses konfigurasi terdiri dari dua langkah :

| Langkah | Deskripsi |
|---------|-----------|
| **1. Eksekusi skrip post-installation** | Menjalankan aksi `post` pada skrip INSTALL paket (isi skrip dapat dilihat dengan `xbps-query --property=install-script <paket>`) |
| **2. Update status paket** | Mengubah status paket menjadi `XBPS_PKG_STATE_INSTALLED` jika langkah pertama berhasil |

Secara default, **hanya paket yang belum dikonfigurasi** yang akan diproses .

---

## 📋 Kapan Anda Membutuhkan `xbps-reconfigure`?

| Skenario | Contoh |
|----------|--------|
| **Memperbaiki paket dengan konfigurasi bermasalah** | Setelah instalasi gagal atau terputus di tengah jalan |
| **Menerapkan perubahan konfigurasi manual** | Setelah mengedit file konfigurasi paket secara langsung |
| **Membangun ulang modul kernel (DKMS)** | Setelah menginstall kernel baru atau header kernel |
| **Membangun ulang initramfs** | Setelah mengganti generator initramfs (dracut → mkinitcpio) |
| **Menerapkan perubahan font** | Setelah menambah atau mengubah konfigurasi font |

---

## 🔧 Parameter Penting

| Parameter | Fungsi | Contoh |
|-----------|--------|--------|
| `-a, --all` | Mengonfigurasi **semua** paket yang terinstall | `sudo xbps-reconfigure -a` |
| `-f, --force` | Memaksa konfigurasi ulang meskipun paket sudah dikonfigurasi sebelumnya  | `sudo xbps-reconfigure -f firefox` |
| `-x, --deps` | Mengonfigurasi paket beserta **dependensi langsungnya** | `sudo xbps-reconfigure -x firefox` |
| `--fulldeptree` | Mengonfigurasi **seluruh pohon dependensi** (harus dengan `-x`) | `sudo xbps-reconfigure -x --fulldeptree firefox` |
| `-i, --ignore <pkg>` | Mengabaikan paket tertentu saat konfigurasi massal (`-a`)  | `sudo xbps-reconfigure -a -i firefox -i vim` |
| `-r, --rootdir <dir>` | Menentukan direktori root alternatif (untuk chroot/container) | `sudo xbps-reconfigure -r /mnt/void linux` |
| `-v, --verbose` | Menampilkan output lebih detail | `sudo xbps-reconfigure -v dbus` |
| `-d, --debug` | Mode debugging untuk troubleshooting | `sudo xbps-reconfigure -d pulseaudio` |

---

## 📝 Contoh Penggunaan

### 1. Mengonfigurasi Ulang Satu Paket (Force)

```bash
sudo xbps-reconfigure -f fontconfig
```
Perintah ini memaksa fontconfig untuk menjalankan ulang skrip konfigurasinya, berguna setelah menambah font baru agar terdeteksi sistem .

### 2. Mengonfigurasi Semua Paket

```bash
sudo xbps-reconfigure -a
```
Menjalankan konfigurasi untuk semua paket yang terinstall. Berguna setelah recovery system.

### 3. Membangun Ulang Initramfs untuk Kernel Tertentu

```bash
sudo xbps-reconfigure -f linux6.6
```
Perintah ini **sangat umum** digunakan di Void Linux untuk membangun ulang initramfs kernel . Anda bisa melihat versi kernel yang terinstall dengan:
```bash
xbps-query -Rs '^linux[0-9]'
```

### 4. Membangun Ulang Modul DKMS

Ketika Anda menginstall kernel baru atau `linux-headers`, jalankan:
```bash
sudo xbps-reconfigure -f linux6.6
```
Skrip konfigurasi akan otomatis membangun ulang semua modul DKMS (seperti driver NVIDIA, VirtualBox, dll) untuk kernel tersebut .

### 5. Mengganti Initramfs Generator

Jika Anda beralih dari `dracut` ke `mkinitcpio`:
```bash
# Install mkinitcpio
sudo xbps-install -S mkinitcpio

# Ganti alternatif initramfs
sudo xbps-alternatives -s mkinitcpio -g initramfs

# Bangun ulang initramfs untuk semua kernel
sudo xbps-reconfigure -f linux6.6
```

### 6. Mengonfigurasi Paket dengan Dependensinya

```bash
sudo xbps-reconfigure -x --fulldeptree dbus
```
Ini akan mengonfigurasi `dbus` dan semua paket yang bergantung padanya.

### 7. Melihat Isi Skrip INSTALL Suatu Paket

```bash
xbps-query --property=install-script xbps
```
Perintah ini menampilkan skrip konfigurasi yang akan dijalankan oleh `xbps-reconfigure` .

---

## 🔄 Perbedaan dengan `xbps-install`

| Aspek | `xbps-install` | `xbps-reconfigure` |
|-------|----------------|---------------------|
| **Fungsi utama** | Menginstall/mengupdate paket | Menjalankan ulang konfigurasi paket |
| **Mengubah file paket** | ✅ Ya (download, ekstrak) | ❌ Tidak (hanya eksekusi skrip) |
| **Mengubah konfigurasi** | ⚠️ Hanya saat instalasi pertama | ✅ Bisa setelah file konfigurasi diubah manual |
| **Membangun initramfs** | ❌ Tidak otomatis | ✅ Ya (untuk paket kernel) |
| **Memerlukan download** | ✅ Ya | ❌ Tidak |

---

## ⚠️ Catatan Penting

1. **Hak akses root** - Sebagian besar penggunaan `xbps-reconfigure` memerlukan `sudo` karena mengubah file sistem .

2. **Tanpa `-f`, hanya paket belum dikonfigurasi** - Jika paket sudah memiliki status `INSTALLED`, perintah tanpa `-f` tidak akan melakukan apa-apa .

3. **Beberapa paket bisa menimpa perubahan manual** - Saat direkonfigurasi, skrip INSTALL mungkin mengembalikan file konfigurasi ke default .

4. **Kernel rekonfigurasi** - Membangun initramfs akan memakan waktu dan memerlukan ruang di `/boot` .

5. **Integrasi dengan DKMS** - `xbps-reconfigure` adalah satu-satunya cara resmi untuk membangun ulang modul kernel setelah update header .

---

## 🔗 Kesimpulan

`xbps-reconfigure` adalah alat **maintenance** di ekosistem XBPS yang tidak digunakan sehari-hari, namun sangat kritis saat diperlukan . Anda akan menggunakannya ketika:

| Situasi | Perintah |
|---------|----------|
| **Memperbaiki paket yang konfigurasinya rusak** | `sudo xbps-reconfigure -f <paket>` |
| **Membangun initramfs setelah update kernel** | `sudo xbps-reconfigure -f linux<x.y>` |
| **Membangun ulang modul DKMS** | `sudo xbps-reconfigure -f linux<x.y> --force` |
| **Menerapkan perubahan font** | `sudo xbps-reconfigure -f fontconfig` |
| **Recovery sistem penuh** | `sudo xbps-reconfigure -a` |

Sederhananya: **Jika paket sudah terinstall tetapi tidak berfungsi dengan benar, atau Anda ingin menerapkan perubahan konfigurasi, `xbps-reconfigure` adalah solusinya.**
