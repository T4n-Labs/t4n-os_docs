## `xbps-uhelper` pada Void Linux

`xbps-uhelper` adalah utilitas dalam XBPS yang menyediakan berbagai **fungsi pembantu (helper)** untuk mengolah string dan informasi terkait paket XBPS . Secara sederhana, ini adalah "pisau Swiss army" untuk memanipulasi nama paket, versi, pola dependensi, dan informasi lainnya.


## đźŽŻ Fungsi Utama

`xbps-uhelper` menggabungkan berbagai utilitas kecil ke dalam satu perintah . Fungsinya terbagi dalam beberapa kategori:

| Kategori | Fungsi | Kegunaan |
|----------|--------|----------|
| **Ekstraksi Informasi** | `getpkgname`, `getpkgversion`, `getpkgrevision` | Memisahkan nama, versi, atau revisi dari string paket |
| **Manipulasi Dependensi** | `getpkgdepname`, `getpkgdepversion` | Mengurai pola dependensi (contoh: `foo>=1.0` â†’ nama `foo`) |
| **Perbandingan** | `cmpver`, `pkgmatch` | Membandingkan versi atau mencocokkan paket dengan pola |
| **Informasi Sistem** | `arch`, `getsystemdir` | Mendapatkan arsitektur sistem atau direktori konfigurasi |
| **Query Paket Terinstall** | `version`, `real-version` | Mendapatkan versi paket yang sudah terinstall |

Berdasarkan manual page dan source code XBPS , berikut semua actions yang tersedia:

| Action | Argumen | Fungsi |

| `getpkgdepname` | `<dep...>` | Mencetak nama dari pola dependensi |
| `getpkgdepversion` | `<dep...>` | Mencetak constraint versi dari pola dependensi |

## đź“ť Contoh Penggunaan

### 1. Memisahkan Nama, Versi, dan Revisi Paket

```bash
# Dari string paket "firefox-120.0_1"
$ xbps-uhelper getpkgname firefox-120.0_1
firefox

$ xbps-uhelper getpkgversion firefox-120.0_1
120.0

$ xbps-uhelper getpkgrevision firefox-120.0_1
1
```

### 2. Mengurai Pola Dependensi

```bash
# Pola dependensi seperti yang muncul di file template
$ xbps-uhelper getpkgdepname 'gtk+>=3.0'
gtk+

$ xbps-uhelper getpkgdepversion 'gtk+>=3.0'
>=3.0
```

Action `getname` dan `getversion` lebih fleksibel karena bisa menangani baik `pkgver` maupun pola dependensi :
```bash
$ xbps-uhelper getname 'firefox-120.0_1'
firefox
$ xbps-uhelper getname 'gtk+>=3.0'
gtk+
```

### 3. Membandingkan Versi

```bash
# cmpver mengembalikan exit code yang berbeda
$ xbps-uhelper cmpver 'foo-1.0_1' 'foo-2.0_1'
$ echo $?
1    # 1 berarti instver < reqver (versi yang diinstall lebih rendah)

$ xbps-uhelper cmpver 'foo-2.0_1' 'foo-1.0_1'
$ echo $?
255  # 255 berarti instver > reqver

# Dengan --verbose, akan mencetak hasil perbandingan
$ xbps-uhelper -v cmpver 'foo-1.0_1' 'foo-2.0_1'
1.0 < 2.0
```

### 4. Mencocokkan Paket dengan Pola

```bash
# Cek apakah firefox-120.0_1 memenuhi pola firefox>=100.0
$ xbps-uhelper pkgmatch firefox-120.0_1 'firefox>=100.0'
$ echo $?
1    # 1 berarti MATCH (pola terpenuhi)

# verbose mode akan mencetak hasil
$ xbps-uhelper -v pkgmatch firefox-1.0_1 'firefox>=100.0'
firefox-1.0_1 does not match firefox>=100.0
$ echo $?
0    # 0 berarti tidak match
```

**Exit codes untuk `pkgmatch`** :
| Exit Code | Arti |
|-----------|------|
| `1` | `pkgver` **memenuhi** `pkgpattern` |
| `0` | `pkgver` **tidak memenuhi** `pkgpattern` |
| `255` | Terjadi error |

### 5. Informasi Sistem dan Paket

```bash
# Arsitektur sistem
$ xbps-uhelper arch
x86_64

# Direktori konfigurasi sistem
$ xbps-uhelper getsystemdir
/usr/share/xbps.d

# Versi paket yang terinstall
$ xbps-uhelper version firefox xbps
120.0_1
0.68_1

# Versi "real" (untuk paket virtual/metapackage)
$ xbps-uhelper real-version void-repo-base
2025.01_1
```

| Opsi | Fungsi |

| `-V, --version` | Menampilkan versi XBPS |


## đź“ś Sejarah: Penggabungan dan Pemisahan Utilitas

`xbps-uhelper` awalnya adalah tempat berkumpulnya berbagai utilitas kecil XBPS, termasuk:
- `xbps-cmpver` (perbandingan versi)
- `xbps-digest` (menghitung hash)
- `xbps-fetch` (mengunduh file) 

Seiring perkembangan, fungsi-fungsi yang berdiri sendiri dipisahkan menjadi utilitas mandiri:

| Utilitas Mandiri | Menggantikan | Tahun |
|------------------|--------------|-------|
| `xbps-digest` | `xbps-uhelper digest` | 2019  |
| `xbps-fetch` | `xbps-uhelper fetch` | 2019  |

Namun, untuk menjaga kompatibilitas ke belakang (terutama dengan versi lama `xbps-src`), action `digest` dan `fetch` masih tersedia meskipun tidak dicantumkan di `--help` .


## đź’ˇ Siapa yang Membutuhkan `xbps-uhelper`?

| Pengguna | Skenario |
|----------|----------|
| **Pengembang paket** | Membuat template build, mengurai dependensi |
| **Pengguna advanced** | Scripting dan automation (membaca versi paket, dll) |
| **Maintainer paket** | Memeriksa dan memvalidasi dependensi |
| **Contributor void-packages** | Debugging dan testing template |

`xbps-uhelper` lebih ditujukan untuk **scripting dan pengembangan paket** daripada penggunaan sehari-hari oleh pengguna biasa. Jika Anda hanya install, update, dan hapus paket, Anda tidak akan sering menggunakan `xbps-uhelper` .

Namun, jika Anda ingin menulis script yang berinteraksi dengan sistem paket XBPS, `xbps-uhelper` adalah alat yang sangat berguna untuk memanipulasi string nama dan versi paket.


## âš ď¸Ź Catatan Penting

1. **Case-sensitive** - Semua operasi bersifat case-sensitive.

2. **Exit codes khusus** - Untuk `cmpver` dan `pkgmatch`, exit codes memiliki arti spesifik yang berbeda dari perintah lain :
   - Gunakan `$?` di bash untuk menangkap nilai kembalian
   - Mode `--verbose` akan mencetak hasil perbandingan ke stdout

3. **Kompatibilitas** - Action `digest` dan `fetch` masih ada untuk kompatibilitas, tapi disarankan menggunakan `xbps-digest` dan `xbps-fetch` yang lebih baru .


## đź”— Kesimpulan

`xbps-uhelper` adalah **utilitas serbaguna** di ekosistem XBPS yang menyediakan berbagai fungsi pembantu untuk:
- **Mengurai string paket** (memisahkan nama, versi, revisi)
- **Mengurai pola dependensi** (mengambil nama dan constraint versi)
- **Membandingkan versi** (dengan exit codes yang informatif)
- **Mencocokkan paket dengan pola** (untuk validasi dependensi)
- **Mendapatkan informasi sistem** (arsitektur, direktori konfigurasi)

Meskipun beberapa fungsinya telah dipisahkan ke utilitas mandiri (`xbps-digest`, `xbps-fetch`), `xbps-uhelper` tetap menjadi komponen penting untuk scripting dan pengembangan paket di Void Linux .
