## `xbps-uchroot` pada Void Linux

`xbps-uchroot` adalah utilitas dalam XBPS yang berfungsi untuk melakukan **chroot dan bind mount** dengan memanfaatkan **Linux namespaces** untuk isolasi proses .


## 🎯 Fungsi Utama

`xbps-uchroot` memungkinkan pengguna untuk:

| Fungsi | Penjelasan |
|--------|------------|
| **Chroot ke direktori target** | Berpindah ke lingkungan root yang terisolasi |
| **Bind mount pseudo-filesystem** | Secara otomatis mount `/dev`, `/dev/shm`, `/proc`, dan `/sys` ke dalam chroot  |
| **Bind mount direktori kustom** | Memasang direktori dari host ke dalam chroot (opsional) |
| **Isolasi dengan namespaces** | Memisahkan IPC, PID, dan mount points dari proses pemanggil  |

Utilitas ini terutama digunakan oleh **`xbps-src`** (sistem build paket Void Linux) untuk membuat lingkungan build yang terisolasi dan aman .


## 🔧 Parameter Penting

| Parameter | Fungsi |
|-----------|--------|
| `-b src:dest` | Bind mount `src` ke `CHROOTDIR/dest` (read-write)  |
| `-B src:dest` | Bind mount `src` ke `CHROOTDIR/dest` dalam mode **read-only**  |
| `-O` | Membuat direktori temporary dengan overlay layer (menggunakan overlayfs). Perubahan tidak disimpan ke `CHROOTDIR`  |
| `-t` | Mount direktori temporary di **tmpfs** (RAM), hanya berguna bersama `-O`  |
| `-o opts` | Argumen tambahan untuk mount tmpfs (bersama `-O` dan `-t`)  |

### Contoh Penggunaan

```bash
# Chroot sederhana dengan mount otomatis /proc, /sys, /dev
xbps-uchroot /mnt/void /bin/bash

# Bind mount direktori host ke dalam chroot
xbps-uchroot -b /home/user:/home/user /mnt/void /bin/bash

# Overlay sementara (perubahan tidak disimpan)
xbps-uchroot -O /mnt/void /bin/bash
```


## 🔐 Keamanan dan Izin

`xbps-uchroot` memiliki persyaratan keamanan yang ketat:

| Persyaratan | Penjelasan |
|-------------|------------|
| **setgid** | Binary harus memiliki bit setgid untuk berfungsi  |
| **Group khusus** | Hanya anggota group tertentu yang bisa menjalankannya (default: `xbuilder` di Void Linux)  |
| **Izin file** | `chmod 4750` dan kepemilikan `root:<group>`  |

Di **Void Linux**, konfigurasi ini sudah diatur secara default. Pengguna yang menjadi anggota grup `xbuilder` dapat langsung menggunakan `xbps-uchroot` .


## ⚙️ Prasyarat Kernel

`xbps-uchroot` memerlukan opsi kernel Linux berikut :

| Opsi Kernel | Fungsi |
|-------------|--------|
| `CONFIG_NAMESPACES` | Dukungan namespace secara umum |
| `CONFIG_IPC_NS` | Isolasi IPC (Inter-Process Communication) |
| `CONFIG_PID_NS` | Isolasi process ID |
| `CONFIG_UTS_NS` | Isolasi hostname/domainname |
| `CONFIG_OVERLAY_FS` | Diperlukan untuk opsi `-O` (overlay)  |

Kernel Void Linux default sudah mengaktifkan semua opsi ini.


## 🔄 Perbandingan dengan Tools Lain

`xbps-src` mendukung beberapa utilitas chroot, dan `xbps-uchroot` adalah salah satu opsinya :

| Utility | Cara Kerja | Default | catatan |
|---------|------------|---------|---------|
| **`xbps-uunshare`** | user namespaces | ✅ Ya | Tidak perlu setgid, lebih aman untuk user biasa  |
| **`xbps-uchroot`** | namespaces + setgid | ❌ | Memerlukan grup khusus, performa baik |
| **`proot`** | User-space | ❌ | Tidak perlu hak khusus, tapi lebih lambat  |

### Mengaktifkan `xbps-uchroot` untuk `xbps-src`

```bash
cd void-packages
echo XBPS_CHROOT_CMD=uchroot >> etc/conf
```


## 💡 Siapa yang Membutuhkan `xbps-uchroot`?

| Pengguna | Kebutuhan |
|----------|-----------|
| **Pengembang paket Void** | Membangun paket dengan `xbps-src` dalam lingkungan terisolasi |
| **Contributor void-packages** | Testing paket sebelum dikirim ke repositori resmi |
| **Power user** | Membangun sendiri paket dari source |

Untuk penggunaan sehari-hari sebagai pengguna biasa, Anda **tidak perlu** menggunakan `xbps-uchroot` secara langsung. Ini adalah alat internal yang digunakan oleh sistem build Void Linux .


## ⚠️ Troubleshooting Umum

Jika muncul error `ERROR clone (Operation not permitted)` :

1. **Pastikan user berada di grup yang tepat:**
   ```bash
   groups $USER
   # Harus mengandung 'xbuilder'
   ```

2. **Jika tidak, tambahkan user ke grup:**
   ```bash
   sudo usermod -a -G xbuilder $USER
   ```

3. **Logout dan login kembali** agar perubahan grup生效.

4. **Cek izin binary** (idealnya sudah benar di Void):
   ```bash
   ls -la /usr/bin/xbps-uchroot
   # Output: -rwsr-x--- ... root xbuilder ...
   ```


## 🔗 Kesimpulan

`xbps-uchroot` adalah komponen penting di balik layar ekosistem Void Linux yang memungkinkan `xbps-src` membangun paket dalam lingkungan yang bersih dan terisolasi. Meskipun pengguna akhir jarang berinteraksi langsung dengannya, `xbps-uchroot` adalah salah satu fondasi yang membuat sistem build paket Void Linux aman dan dapat dijalankan tanpa hak akses root .

Jika Anda berencana untuk mengembangkan atau mem-build paket untuk T4n OS, `xbps-uchroot` (atau alternatifnya `xbps-uunshare`) akan menjadi alat yang Anda gunakan setiap hari melalui `xbps-src`.
