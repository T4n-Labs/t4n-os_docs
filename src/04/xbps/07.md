## `xbps-fbulk` pada Void Linux

`xbps-fbulk` adalah utilitas dalam XBPS yang berfungsi untuk melakukan **bulk build (build massal) paket dengan cepat** dari repositori `void-packages`. `xbps-fbulk` dapat membangun seluruh pohon dependensi dari sebuah paket atau kumpulan paket secara otomatis dan paralel.


## 🎯 Fungsi Utama

`xbps-fbulk` membaca direktori source package (`srcpkgs`) di repositori `void-packages`, kemudian membangun pohon dependensi secara dinamis menggunakan perintah `xbps-src show-build-deps`. Selama proses berjalan:

1. **Dependensi terminal** dibangun dan dipaketkan terlebih dahulu
2. Setelah build selesai, dependensi berikutnya yang terpenuhi akan ditambahkan ke urutan build
3. Seluruh pohon dependensi akhirnya akan terbangun sepenuhnya

Setiap paket hanya akan dicoba dibangun **satu kali**, tidak peduli berapa banyak paket lain yang bergantung padanya.


## 🔧 Parameter Penting

| Parameter | Fungsi |
|-----------|--------|
| `-j, --jobs X` | Menentukan jumlah build paralel yang berjalan bersamaan (default: 1) |
| `-l, --logdir <dir>` | Menentukan direktori log (default: `fbulk-log.<pid>`) |
| `-s, --system` | Mode sistem - hanya memproses paket yang diinstall secara manual di sistem (lihat `xbps-pkgdb(1)`) |
| `-d, --debug` | Menampilkan debugging tambahan ke stderr |
| `-v, --verbose` | Menampilkan pesan verbose |
| `-h, --help` | Menampilkan bantuan |
| `-V, --version` | Menampilkan versi |


## 📋 Format Log Output

`xbps-fbulk` menyimpan hasil build dalam file-file di direktori log:

| File | Isi |
|------|-----|
| `logdir/run` | Paket yang sedang dalam proses dibangun |
| `logdir/good` | Paket yang berhasil dibangun |
| `logdir/bad` | Paket yang gagal dibangun |
| `logdir/skipped` | Paket yang dilewati (arsitektur tidak didukung, broken, atau restricted) |
| `logdir/deps` | Paket yang tidak dibangun karena dependensi gagal atau hilang |


## 💡 Prasyarat Sistem

Sebelum menggunakan `xbps-fbulk`, pastikan:

1. **Repositori `void-packages`** sudah di-clone dan **masterdir** sudah terisi penuh untuk operasi chroot
2. **Konfigurasi di `etc/conf`**:
   ```
   XBPS_CHROOT_CMD=uchroot
   ```
3. **Kernel mendukung overlay filesystem** (diperkenalkan pada kernel 4.0)
4. **User memiliki hak** untuk menjalankan `xbps-uchroot(1)`


## 📂 Contoh Penggunaan

**Build semua paket di repositori:**
```bash
xbps-fbulk /path/to/void-packages
```

**Build paket tertentu saja:**
```bash
xbps-fbulk /home/user/void-packages firefox chromium
```

**Build dengan 4 proses paralel:**
```bash
xbps-fbulk -j 4 /path/to/void-packages
```

**System mode (hanya paket manual di sistem):**
```bash
xbps-fbulk -s /path/to/void-packages
```


## 👥 Siapa yang Membutuhkan `xbps-fbulk`?

| Pengguna | Kegunaan |
|----------|----------|
| **Maintainer paket Void Linux** | Membangun ulang banyak paket sekaligus setelah perubahan dependensi |
| **Pengembang yang berkontribusi ke void-packages** | Menguji dampak perubahan pada paket yang bergantung |
| **Power user yang mengelola repositori sendiri** | Membangun dan memutakhirkan sistem secara massal |

`xbps-fbulk` bukan untuk penggunaan sehari-hari oleh pengguna biasa. Ini adalah alat untuk **pembangun paket (package builder)** dan **maintainer** yang perlu memproses banyak paket secara efisien.


## 🔗 Sejarah

`xbps-fbulk` awalnya ditulis oleh Matthew Dillon untuk proyek **DragonFlyBSD**, kemudian diadaptasi ke XBPS oleh Juan Romero Pardines dengan tambahan fitur baru.


## ⚠️ Catatan Penting

1. Dibutuhkan **hak akses khusus** (seperti `sudo` atau kemampuan `setuid` pada `xbps-uchroot`)
2. Proses build bisa memakan waktu sangat lama tergantung jumlah paket
3. Pastikan ruang disk cukup karena setiap build menghasilkan paket biner
4. **Build tidak bersifat incremental penuh** - hanya satu percobaan per paket
