## `xbps-install` pada Void Linux

`xbps-install` adalah utilitas utama dalam package manager XBPS (X Binary Package System) untuk **menginstal, memperbarui, dan mengelola paket** di Void Linux .

---

## 🎯 Fungsi Utama

| Fungsi | Penjelasan |
|--------|------------|
| **Instalasi** | Memasang paket baru dari repositori |
| **Update** | Memperbarui paket yang sudah terinstall ke versi terbaru |
| **Upgrade sistem** | Memperbarui seluruh sistem sekaligus |
| **Reinstall** | Memasang ulang paket yang sudah ada |
| **Downgrade** | Menurunkan versi paket (dengan opsi `--force`) |

`xbps-install` secara otomatis menangani **dependency resolution** (menentukan dan menginstal paket-paket dependensi yang dibutuhkan) .

---

## 🔧 Parameter Penting

### Parameter Wajib untuk Sinkronisasi

| Parameter | Fungsi |
|-----------|--------|
| `-S, --sync` | **Sinkronisasi indeks repositori** - perintah ini wajib dijalankan sebelum instalasi atau update agar sistem mengetahui paket terbaru yang tersedia  |

### Parameter untuk Update & Upgrade

| Parameter | Fungsi |
|-----------|--------|
| `-u, --update` | Memperbarui paket yang sudah terinstall. Jika tanpa argumen paket, akan melakukan **upgrade seluruh sistem**  |

### Parameter untuk Kontrol Instalasi

| Parameter | Fungsi |
|-----------|--------|
| `-f, --force` | Memaksa instalasi ulang atau downgrade (jika versi di repositori lebih rendah)  |
| `-n, --dry-run` | **Mode simulasi** - hanya menampilkan apa yang akan dilakukan tanpa mengubah sistem  |
| `-y, --yes` | Menjawab "ya" secara otomatis untuk semua pertanyaan interaktif  |
| `-D, --download-only` | Hanya mengunduh paket ke cache tanpa menginstal  |

### Parameter untuk Manajemen Repositori

| Parameter | Fungsi |
|-----------|--------|
| `-R, --repository <url>` | Menambahkan repositori ke daftar pencarian (bisa dipakai berkali-kali)  |

### Parameter Lainnya

| Parameter | Fungsi |
|-----------|--------|
| `-v, --verbose` | Menampilkan informasi lebih detail  |
| `-d, --debug` | Mode debugging untuk troubleshooting  |
| `-r, --rootdir <dir>` | Menentukan direktori root target (untuk instalasi di lingkungan chroot/container)  |
| `-A, --automatic` | Menandai paket sebagai "automatic" (akan diperlakukan sebagai orphan jika tidak ada yang bergantung padanya)  |

## 📋 Contoh Penggunaan

### 1. **Instalasi Paket Baru**

Sebelum instalasi, selalu sinkronkan indeks repositori terlebih dahulu :

```bash
sudo xbps-install -S firefox
```

### 2. **Update Satu Paket**

```bash
sudo xbps-install -u firefox
```

### 3. **Update Seluruh Sistem (Full System Upgrade)**

Perintah ini akan memperbarui semua paket yang terinstall ke versi terbaru :

```bash
sudo xbps-install -Su
```

> **⚠️ Catatan Penting:** Jika update mencakup paket `xbps` itu sendiri, Anda perlu menjalankan perintah `-Su` **dua kali** - sekali untuk mengupdate XBPS, sekali lagi untuk paket lainnya .

### 4. **Install Multiple Paket Sekaligus**

```bash
sudo xbps-install -S vim curl wget git
```

### 5. **Mode Simulasi (Dry Run)**

```bash
xbps-install -n -S firefox
```
Output akan menunjukkan paket apa yang akan diinstal tanpa benar-benar menginstal .

### 6. **Memaksa Instalasi Ulang**

```bash
sudo xbps-install -f firefox
```

### 7. **Non-Interaktif (untuk Scripting)**

```bash
sudo xbps-install -Sy firefox
```

## 📦 Cara Kerja Internal

Ketika Anda menjalankan `xbps-install -S <paket>`, urutan prosesnya adalah:

```
1. Sinkronisasi indeks repositori (-S)
   ↓
2. Resolve dependencies (cari paket dependensi yang dibutuhkan)
   ↓
3. Download paket ke cache (/var/cache/xbps)
   ↓
4. Verifikasi signature dan checksum
   ↓
5. Ekstrak dan install file ke sistem
   ↓
6. Update database paket lokal (/var/db/xbps/pkgdb-0.38.plist)
   ↓
7. Jalankan post-install scripts (konfigurasi)
```

## 🔗 Package Expression (Cara Menulis Nama Paket)

`xbps-install` mendukung package expression untuk spesifikasi paket yang lebih fleksibel :

| Format | Contoh | Keterangan |
|--------|--------|-------------|
| Nama saja | `firefox` | Menginstal versi terbaru yang tersedia |
| Nama + versi eksak | `firefox-120.0_1` | Menginstal versi spesifik |
| Dengan operator | `firefox>=120.0` | Menginstal versi minimal tertentu |

---

## 📝 Mode Khusus Paket

`xbps-install` mendukung beberapa mode operasi untuk paket yang terinstall :

| Mode | Fungsi |
|------|--------|
| **hold** | Paket tidak akan diupdate kecuali dipaksa dengan `-f` |
| **manual** | Paket tidak akan dianggap sebagai orphan (tidak akan dihapus oleh `xbps-remove -o`) |
| **repolock** | Paket hanya akan menerima update dari repositori yang sama saat instalasi |

Cek status mode paket dengan `xbps-query -H` (hold) atau `xbps-query -m` (manual).

---

## ⚠️ Prasyarat dan Catatan Penting

1. **Hak akses root** - Sebagian besar operasi `xbps-install` memerlukan `sudo` karena mengubah file sistem .

2. **Update XBPS terlebih dahulu** - Void Linux mewajibkan update paket `xbps` sebelum paket lainnya :
   ```bash
   sudo xbps-install -u xbps
   sudo xbps-install -Su
   ```

3. **Sinkronisasi sebelum instalasi** - Jangan lupa `-S` sebelum menginstal atau update .

4. **Transaksi atomic** - Jika satu paket gagal diinstal, seluruh transaksi dibatalkan (tidak ada instalasi parsial).

5. **File konfigurasi** - File konfigurasi yang sudah dimodifikasi pengguna akan tetap terjaga (tidak ditimpa) saat update .

## 🔄 Perbandingan dengan Package Manager Lain

| Perintah | APT (Debian/Ubuntu) | DNF (Fedora) | Pacman (Arch) | XBPS (Void) |
|----------|---------------------|--------------|---------------|-------------|
| Update repo | `apt update` | `dnf check-update` | `pacman -Sy` | `xbps-install -S` |
| Install paket | `apt install nano` | `dnf install nano` | `pacman -S nano` | `xbps-install -S nano` |
| Upgrade sistem | `apt upgrade` | `dnf upgrade` | `pacman -Su` | `xbps-install -Su` |
| Hapus paket | `apt remove nano` | `dnf remove nano` | `pacman -R nano` | `xbps-remove nano` |
| Cari paket | `apt search nano` | `dnf search nano` | `pacman -Ss nano` | `xbps-query -Rs nano` |

---

## 🔗 Kesimpulan

`xbps-install` adalah **command center** untuk manajemen paket di Void Linux. Dengan menguasai perintah ini dan parameter utamanya (`-S` untuk sinkronisasi, `-u` untuk update, `-Su` untuk upgrade sistem), Anda dapat mengelola seluruh software di sistem T4n OS dengan efisien. `xbps-install` menangani dependency resolution secara otomatis, mendukung instalasi massal, dan memiliki mekanisme atomic transaction yang aman.
