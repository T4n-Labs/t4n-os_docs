## `xbps-pkgdb` pada Void Linux

`xbps-pkgdb` adalah utilitas dalam XBPS yang berfungsi untuk **memeriksa, memperbaiki, dan memodifikasi database paket (pkgdb)** di Void Linux .

---

## 🎯 Fungsi Utama

`xbps-pkgdb` memiliki dua peran besar: **maintenance** (memeriksa dan memperbaiki database paket) serta **manajemen status** (mengubah mode paket) .

### 1. Mode Maintenance & Pemeriksaan Database

Perintah ini dapat memeriksa beberapa aspek berikut :

| Pemeriksaan | Fungsi |
|-------------|--------|
| **METADATA CHECK** | Memeriksa apakah file metadata paket telah dimodifikasi dengan membandingkan hash asli saat instalasi dengan yang sekarang. |
| **FILES CHECK** | Memeriksa semua file milik paket, memverifikasi apakah ada file yang hilang atau dimodifikasi. Untuk file biasa, waktu modifikasi dan hash SHA256 dibandingkan; untuk symbolic link, diperiksa targetnya. |
| **DEPENDENCIES CHECK** | Memeriksa apakah semua dependensi yang dibutuhkan suatu paket terpenuhi. |
| **ALTERNATIVES CHECK** | Memeriksa apakah semua symbolic link untuk grup `xbps-alternatives` sudah benar. |
| **OBSOLETE METADATA CHECK** | Membersihkan data usang dari versi XBPS sebelumnya. |
| **FORMAT CONVERSION** | Mengupdate format pkgdb ke versi terbaru (`-u` flag). |

### 2. Mode Manajemen Status Paket

`xbps-pkgdb` dapat mengubah status (mode) suatu paket menggunakan flag `-m` :

| Mode | Fungsi |
|------|--------|
| **auto** | Menandai paket sebagai **automatic** (terinstall otomatis sebagai dependensi). Paket ini akan dianggap sebagai orphan dan dapat dihapus oleh `xbps-remove --remove-orphans` jika tidak dibutuhkan lagi . |
| **manual** | Menandai paket sebagai **manual** (diinstall secara eksplisit oleh pengguna). Paket tidak akan dihapus sebagai orphan . |
| **hold** | Mencegah paket dari update saat upgrade sistem. Berguna untuk paket yang ingin Anda tetap pada versi tertentu . |
| **unhold** | Menghapus status hold. |
| **repolock** | Mengunci paket agar hanya menerima update dari repositori yang sama saat instalasi. Berguna untuk paket yang di-build sendiri agar tidak tertimpa versi dari repositori default . |
| **repounlock** | Menghapus status repolock. |

## 📋 Contoh Penggunaan

### Maintenance

**1. Memeriksa semua paket** 
```bash
xbps-pkgdb -a
```
Menampilkan laporan lengkap semua paket yang terinstall.

**2. Memeriksa paket tertentu saja**
```bash
xbps-pkgdb firefox
```

**3. Menjalankan pemeriksaan spesifik**  (flag `--checks`)
```bash
xbps-pkgdb --checks files,dependencies -a
```
Hanya menjalankan pemeriksaan file dan dependensi.

**4. Update format database** 
```bash
sudo xbps-pkgdb -u
```

### Manajemen Status Paket

**5. Hold paket (mencegah update)** 
```bash
sudo xbps-pkgdb -m hold firefox
```
Firefox tidak akan ikut terupdate saat `xbps-install -Su`.

**6. Unhold paket** 
```bash
sudo xbps-pkgdb -m unhold firefox
```

**7. Repolock paket (kunci ke repositori tertentu)** 
```bash
sudo xbps-pkgdb -m repolock firefox
```

**8. Ubah status dari automatic ke manual** 
```bash
sudo xbps-pkgdb -m manual xfce4
```

**9. Lihat daftar paket yang di-hold**
```bash
xbps-query --list-hold-pkgs
```

**10. Lihat daftar paket manual**
```bash
xbps-query --list-manual-pkgs
```

## 📂 File yang Dikelola

| File | Deskripsi |
|------|-----------|
| `/var/db/xbps/pkgdb-0.38.plist` | Database utama paket (format 0.38) yang mencatat semua paket terinstall dan propertinya . |
| `/var/db/xbps/.<pkgname>-files.plist` | Metadata file per paket . |
| `/var/cache/xbps` | Cache paket biner . |

---

## 🗂️ Parameter Penting Lainnya

| Parameter | Fungsi |
|-----------|--------|
| `-a, --all` | Memproses semua paket terlepas dari statusnya . |
| `--checks checks` | Menjalankan hanya pemeriksaan yang ditentukan (dipisah koma) . |
| `-C, --config dir` | Menentukan direktori konfigurasi XBPS . |
| `-d, --debug` | Menampilkan debugging tambahan . |
| `-r, --rootdir dir` | Menentukan direktori root target . |
| `-v, --verbose` | Mode verbose (informasi lebih detail) . |

---

## ⚠️ Catatan Penting

1. **Perubahan mode paket** akan langsung mempengaruhi perilaku `xbps-install` dan `xbps-remove` .
2. Untuk **repolock** yang efektif, Anda perlu memiliki repositori lokal yang tidak sinkron dengan repositori utama .
3. `xbps-pkgdb` adalah satu-satunya cara resmi untuk mengubah status paket dari automatic ke manual .
4. Beberapa operasi memerlukan hak akses **root** (menggunakan `sudo`).

---

## 🔗 Kesimpulan

`xbps-pkgdb` adalah alat **diagnostik sekaligus manajemen** di ekosistem XBPS. Anda akan menggunakannya ketika:
- Ingin **mencek kesehatan sistem** (file hilang, dependensi rusak)
- Perlu **mencegah paket tertentu dari update** (mode `hold`)
- Ingin **mengunci paket ke repositori tertentu** (mode `repolock`)
- Perlu **membersihkan database** dari data usang

Untuk penggunaan sehari-hari sebagai pengguna biasa (install, update, hapus paket), Anda lebih sering menggunakan `xbps-install` dan `xbps-remove`.
