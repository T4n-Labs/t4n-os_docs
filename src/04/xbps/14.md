## `xbps-rindex` pada Void Linux

`xbps-rindex` adalah utilitas dalam XBPS yang berfungsi untuk **mengelola repositori paket biner lokal** . Dengan kata lain, ini adalah alat yang mengubah folder biasa menjadi repositori XBPS yang dapat dikenali oleh `xbps-install`.

---

## 🎯 Fungsi Utama

| Fungsi | Penjelasan |
|--------|------------|
| **Menambahkan paket ke repositori** | Mendaftarkan paket `.xbps` ke dalam indeks repositori lokal |
| **Membersihkan repositori** | Menghapus entri usang dari indeks dan menghapus file paket yang tidak terdaftar |
| **Menandatangani repositori** | Menambahkan metadata tanda tangan untuk keamanan (diperlukan untuk repositori remote) |
| **Menandatangani paket** | Menandatangani satu atau lebih paket dengan kunci RSA |

Secara sederhana, `xbps-rindex` membuat dan memelihara file `repodata` yang dibutuhkan XBPS untuk memahami isi suatu repositori .

---

## 🔧 Mode Operasi

| Mode | Perintah | Fungsi |
|------|----------|--------|
| **Menambahkan paket** | `xbps-rindex -a /path/to/*.xbps` | Mendaftarkan paket biner ke dalam repositori. Paket hanya akan ditambahkan jika versinya lebih baru dari yang sudah ada; gunakan `-f` untuk memaksa menimpa  |
| **Membersihkan indeks** | `xbps-rindex -c /path/to/repo` | Menghapus entri usang dari file indeks repositori (tanpa menghapus file `.xbps`-nya)  |
| **Menghapus paket usang** | `xbps-rindex -r /path/to/repo` | Menghapus file `.xbps` yang tidak terdaftar lagi di indeks (out of date, corrupt, dll)  |
| **Menandatangani repositori** | `xbps-rindex --sign --signedby "Nama" --privkey key.pem /path/to/repo` | Menginisialisasi repositori dengan metadata tanda tangan  |
| **Menandatangani paket** | `xbps-rindex --sign-pkg --privkey key.pem /path/to/*.xbps` | Menandatangani paket secara individual  |

---

## 📋 Contoh Penggunaan

### 1. **Membuat Repositori Lokal Sederhana**

```bash
# Buat direktori repositori
mkdir ~/myrepo

# Pindahkan atau salin paket .xbps ke direktori tersebut
cp /home/build/myapp-1.0_1.x86_64.xbps ~/myrepo/

# Buat indeks repositori (tanpa tanda tangan)
xbps-rindex -a ~/myrepo/*.xbps
```

Sekarang Anda dapat menginstall paket dari repositori lokal ini:
```bash
sudo xbps-install --repository=~/myrepo myapp
```

### 2. **Downgrade Paket dengan Repositori Lokal**

Salah satu kegunaan praktis `xbps-rindex` adalah untuk **downgrade** paket ke versi lama :

```bash
# Asumsikan paket versi lama ada di /var/cache/xbps/
# Tambahkan ke repositori lokal
sudo xbps-rindex -a /var/cache/xbps/pkg-1.0_1.xbps

# Downgrade menggunakan repositori lokal
sudo xbps-install -R /var/cache/xbps/ -f pkg-1.0_1
```

Flag `-f` diperlukan untuk memaksa downgrade karena versi yang akan diinstall lebih rendah dari yang sudah terinstall .

### 3. **Membuat Repositori Bertanda Tangan (Signed Repository)**

Untuk repositori yang akan dibagikan melalui internet, Anda perlu menandatanganinya :

**Step 1 - Generate kunci RSA:**
```bash
ssh-keygen -t rsa -b 4096 -m PEM -f private.pem
```

**Step 2 - Inisialisasi repositori dengan tanda tangan:**
```bash
xbps-rindex --privkey private.pem --sign --signedby "Your Name <email@example.com>" /path/to/repo
```

**Step 3 - Tandatangani paket-paket di dalamnya:**
```bash
xbps-rindex --privkey private.pem --sign-pkg /path/to/repo/*.xbps
```

> **Catatan:** Paket yang ditambahkan di masa depan harus ditandatangani ulang secara manual; tidak otomatis .

### 4. **Membersihkan Repositori**

```bash
# Hanya membersihkan indeks (entri usang dihapus dari repodata)
xbps-rindex -c ~/myrepo

# Menghapus file .xbps yang tidak terdaftar di indeks
xbps-rindex -r ~/myrepo
```

---

## 🗂️ Struktur Direktori Repositori

Setelah menjalankan `xbps-rindex -a`, di dalam direktori repositori akan muncul:

| File | Deskripsi |
|------|-----------|
| `repodata` | File indeks utama yang berisi metadata semua paket (format `.plist`) |
| `repodata.old` | Backup repodata sebelumnya |
| `*.xbps` | File paket biner itu sendiri |
| `*.xbps.sig` | File tanda tangan (jika repositori ditandatangani) |

---

## 🔐 Kunci RSA untuk Signing

`xbps-rindex` memerlukan kunci RSA dalam format **PEM** untuk menandatangani repositori .

**Generate dengan ssh-keygen:**
```bash
ssh-keygen -t rsa -b 4096 -m PEM -f private.pem
```

**Generate dengan openssl:**
```bash
openssl genrsa -out private.pem 4096
```

File kunci publiknya (`private.pem.pub`) perlu didistribusikan ke pengguna agar mereka dapat memverifikasi tanda tangan repositori.

---

## 🔄 Perbedaan Repositori Signed vs Unsigned

| Aspek | Unsigned | Signed |
|-------|----------|--------|
| **Keamanan** | Tidak ada verifikasi keaslian | Terverifikasi dengan kriptografi |
| **Penggunaan** | Repositori lokal, testing | Repositori remote/produksi |
| **Persyaratan** | Minimal | Memerlukan kunci RSA |
| **XBPS behavior** | `xbps-install` akan memperingatkan (tapi tetap bisa) | Wajib diverifikasi |

> **Peringatan:** Void Linux mengharuskan repositori remote untuk ditandatangani. Repositori lokal tidak wajib ditandatangani, sehingga cocok untuk keperluan testing dan development .

---

## ⚙️ Opsi Tambahan

| Opsi | Fungsi |
|------|--------|
| `--compression zstd` | Menentukan format kompresi repodata (default: `zstd`)  |
| `-f, --force` | Memaksa menimpa entri yang sudah ada saat menambahkan paket  |
| `-C, --hashcheck` | Saat membersihkan (`-c`), memeriksa hash file (bukan hanya keberadaan file)  |
| `--privkey key.pem` | Menentukan path kunci RSA privat  |
| `--signedby "string"` | Identitas penanda tangan repositori  |

## 💡 Siapa yang Membutuhkan `xbps-rindex`?

| Pengguna | Kebutuhan |
|----------|-----------|
| **Maintainer paket** | Menguji paket sebelum upload ke repositori resmi |
| **Power user** | Membuat repositori pribadi untuk paket custom |
| **Pengembang** | Mendistribusikan software melalui repositori sendiri |
| **Sysadmin** | Mengelola repositori offline/internal perusahaan |
| **Semua pengguna** | Melakukan downgrade paket (via cache)  |

---

## ⚠️ Catatan Penting

1. **`xbps-rindex` tidak memerlukan hak akses root** untuk operasi dasar pada repositori user (kecuali jika repositori di direktori sistem seperti `/var/cache/xbps`).

2. **Paket baru tidak otomatis ditandatangani** - Setiap kali menambahkan paket ke repositori signed, Anda harus menjalankan ulang perintah sign .

3. **Konsistensi shlibs** - `xbps-rindex -a` melakukan pengecekan konsistensi shared library untuk mencegah kegagalan runtime .

4. **Repodata menggunakan format `.plist`** - File indeks ini adalah file properti daftar dalam format biner yang digunakan XBPS untuk membaca metadata paket dengan cepat.

---

## 🔗 Kesimpulan

`xbps-rindex` adalah alat yang membuat ekosistem XBPS **extensible** - Anda tidak terbatas pada repositori resmi Void Linux. Dengan `xbps-rindex`, Anda dapat:

| Tujuan | Perintah |
|--------|----------|
| **Testing paket sebelum rilis** | `xbps-rindex -a ~/testrepo/*.xbps` |
| **Downgrade aplikasi** | Tambahkan ke repositori lokal, lalu `xbps-install -R`  |
| **Distribusi internal** | Buat signed repo dengan `--sign` dan `--sign-pkg` |
| **Bersih-bersih repositori** | `xbps-rindex -c` dan `-r` |

Sederhananya: **Jika Anda memiliki file `.xbps` dan ingin menginstallnya dengan `xbps-install` melalui repositori (bukan langsung), Anda memerlukan `xbps-rindex` untuk mendaftarkannya terlebih dahulu.**
