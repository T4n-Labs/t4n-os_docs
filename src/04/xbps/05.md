## `xbps-dgraph` pada Void Linux

`xbps-dgraph` adalah utilitas dalam package manager XBPS yang berfungsi untuk **menghasilkan grafik dependensi paket** dalam format DOT, yang dapat divisualisasikan menggunakan Graphviz.


## 🎯 Fungsi Utama `xbps-dgraph`

Utilitas ini memungkinkan Anda untuk melihat hubungan antar paket secara visual dengan cara memproses paket target (baik yang sudah terinstall di sistem maupun yang ada di repositori) dan menghasilkan output berupa kode DOT yang menggambarkan:

| Mode | Fungsi |
|------|--------|
| **Dependency Graph** (default) | Menampilkan hubungan dependensi antar paket secara default |
| **Full Dependency Tree** (`-f`) | Menampilkan pohon dependensi lengkap dari paket target |
| **Metadata Graph** (`-m`) | Menampilkan grafik metadata paket |


## 🔧 Parameter Penting

### Mode Operasi (wajib pilih salah satu)

| Parameter | Fungsi |
|-----------|--------|
| `-f, --fulldeptree` | Menghasilkan pohon dependensi lengkap paket target |
| `-m, --metadata` | Menghasilkan grafik metadata paket target |
| `-g, --gen-config` | Menghasilkan file konfigurasi grafik di direktori saat ini |

### Opsi Tambahan

| Parameter | Fungsi |
|-----------|--------|
| `-R, --repository` | Mode repositori - mencari di repositori (bukan di sistem terinstall) |
| `-r, --rootdir <dir>` | Menentukan direktori root target |
| `-c, --graph-config <file>` | Menentukan file konfigurasi grafik |
| `-M, --memory-sync` | Mengambil data repositori langsung ke memori |
| `-d, --debug` | Menampilkan debug info |


## 📋 Cara Penggunaan

Sintaks dasar:
```bash
xbps-dgraph [OPTIONS] MODE PKG
```

Dimana `PKG` adalah **package expression** yang bisa berupa:
- Nama paket saja: `firefox`
- Nama dan versi eksak: `firefox-120.0_1`
- Dengan operator versi: `firefox>=120.0`

### Contoh Penggunaan

**1. Grafik dependensi dari paket yang terinstall**
```bash
xbps-dgraph -f firefox | dot -Tpng -o firefox-deps.png
```
Perintah ini menghasilkan pohon dependensi lengkap Firefox dan mengubahnya ke PNG menggunakan alat `dot` dari Graphviz.

**2. Mode repositori (paket belum terinstall)**
```bash
xbps-dgraph -R -f firefox
```
Untuk mencari paket yang belum terinstall (hanya tersedia di repositori).

**3. Grafik metadata paket**
```bash
xbps-dgraph -m base-system
```


## 💡 Siapa yang Membutuhkan `xbps-dgraph`?

| Pengguna | Kegunaan |
|----------|----------|
| **Power user/advanced user** | Menganalisis dependensi paket sebelum instalasi atau untuk debugging |
| **Maintainer paket Void** | Memeriksa struktur dependensi paket yang dikelola |
| **Pengembang sistem** | Memahami hubungan antar paket dalam sistem |

`xbps-dgraph` adalah alat yang umumnya digunakan oleh pengguna tingkat lanjut, bukan untuk penggunaan sehari-hari oleh pengguna biasa.


## 🔗 Visualisasi dengan Graphviz

Hasil output `xbps-dgraph` adalah kode DOT yang bisa diproses dengan berbagai tools Graphviz:

```bash
# Hasilkan file DOT
xbps-dgraph -f vim > vim-deps.dot

# Konversi ke gambar
dot -Tpng vim-deps.dot -o vim-deps.png

# Atau langsung satukan
xbps-dgraph -f vim | dot -Tpng -o vim-deps.png
```


## 📂 File Terkait

| File | Deskripsi |
|------|-----------|
| `/var/db/xbps/pkgdb-0.38.plist` | Database paket default XBPS |
| `/etc/xbps.d` | Direktori konfigurasi utama |
| `/var/cache/xbps` | Cache paket biner |


## ⚠️ Catatan Penting

1. **Package expression bersifat case-sensitive** - nama paket harus ditulis persis
2. `xbps-dgraph -f` **harus diberikan argumen paket** - jika tidak, perintah akan error
3. **Output adalah DOT** - Anda memerlukan Graphviz (`sudo xbps-install graphviz`) untuk melihatnya sebagai gambar


## 🔗 Tools XBPS Lainnya yang Terkait

| `xbps-install -n` | Simulasi instalasi (dry-run) untuk melihat dependensi yang akan terinstall |
| `xbps-pkgdb` | Memeriksa konsistensi database paket |

**Kesimpulan**: `xbps-dgraph` adalah alat visualisasi dependensi yang berguna untuk memahami hubungan antar paket di Void Linux. Alat ini lebih ditujukan untuk keperluan debugging, analisis sistem, dan maintenance paket oleh pengguna tingkat lanjut.
