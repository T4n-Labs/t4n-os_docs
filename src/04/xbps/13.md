## `xbps-remove` pada Void Linux

`xbps-remove` adalah utilitas dalam package manager XBPS yang berfungsi untuk **menghapus paket yang sudah terinstall** dari sistem Void Linux .

---

## 🎯 Fungsi Utama

| Fungsi | Penjelasan |
|--------|------------|
| **Menghapus paket** | Menghapus satu atau lebih paket dari sistem |
| **Menghapus dependensi tak terpakai** | Membersihkan package orphans (dependensi yang tidak lagi dibutuhkan) |
| **Menghapus cache** | Membersihkan paket biner yang tersimpan di cache |
| **Menghapus konfigurasi** | Opsi untuk menghapus file konfigurasi paket (purge) |

`xbps-remove` selalu memeriksa dependensi sebelum menghapus - jika ada paket lain yang membutuhkan paket yang akan dihapus, operasi akan ditolak (kecuali dipaksa) .

---

## 🔧 Parameter Penting

### Parameter Dasar

| Parameter | Fungsi |
|-----------|--------|
| `-R, --recursive` | **Menghapus paket beserta dependensinya yang tidak dibutuhkan oleh paket lain** . Sangat berguna untuk membersihkan dependensi yang tidak terpakai setelah menghapus paket utama |
| `-f, --force` | **Memaksa penghapusan** meskipun ada paket lain yang bergantung (berbahaya, bisa merusak sistem) |
| `-v, --verbose` | Menampilkan informasi lebih detail selama proses |
| `-d, --debug` | Mode debugging untuk troubleshooting |
| `-n, --dry-run` | **Mode simulasi** - hanya menampilkan apa yang akan dihapus tanpa benar-benar menghapus |

### Parameter Pembersihan

| Parameter | Fungsi |
|-----------|--------|
| `-O, --remove-orphans` | Menghapus **package orphans** - paket yang terinstall sebagai dependensi tapi tidak lagi dibutuhkan oleh paket lain. Gabungkan dengan `-a` untuk menghapus semua orphans sekaligus |
| `-o, --remove-obsoletes` | Menghapus paket usang yang tidak lagi tersedia di repositori. Berguna untuk membersihkan sistem setelah perubahan besar |
| `-C, --remove-cache` | Membersihkan cache paket biner di `/var/cache/xbps` |

### Parameter Lainnya

| Parameter | Fungsi |
|-----------|--------|
| `-r, --rootdir <dir>` | Menentukan direktori root target (untuk chroot/container) |
| `-y, --yes` | Menjawab "ya" secara otomatis untuk semua pertanyaan interaktif (berguna untuk scripting) |

## 📋 Contoh Penggunaan

### 1. Menghapus Paket Sederhana

```bash
sudo xbps-remove firefox
```
Perintah ini akan menghapus Firefox dari sistem, tetapi **tidak** akan menghapus dependensi yang hanya dibutuhkan oleh Firefox (seperti GTK+, libcairo, dll).

### 2. Menghapus Paket Beserta Dependensinya yang Tidak Terpakai

```bash
sudo xbps-remove -R firefox
```
Ini adalah **best practice** untuk penghapusan paket. Firefox akan dihapus, dan semua dependensinya yang tidak lagi dibutuhkan oleh paket lain juga akan dihapus .

### 3. Menghapus Banyak Paket Sekaligus

```bash
sudo xbps-remove firefox vim curl
```

### 4. Menghapus Package Orphans

**Cek orphans terlebih dahulu:**
```bash
xbps-query -O
```

**Hapus semua orphans:**
```bash
sudo xbps-remove -O
```

Atau untuk menghapus semua orphans tanpa konfirmasi per paket:
```bash
sudo xbps-remove -Oy
```

### 5. Mode Simulasi (Dry Run)

```bash
xbps-remove -Rn firefox
```
Perintah ini hanya menunjukkan apa yang akan dihapus tanpa benar-benar menghapus. Sangat berguna untuk mengecek dampak sebelum eksekusi .

### 6. Membersihkan Cache Paket

```bash
sudo xbps-remove -C
```
Menghapus semua paket biner yang tersimpan di `/var/cache/xbps` .

### 7. Menghapus Paket dengan Paksa (Tidak Direkomendasikan)

```bash
sudo xbps-remove -f firefox
```
Memaksa penghapusan meskipun ada paket lain yang bergantung pada Firefox. Ini bisa merusak sistem .

### 8. Menghapus Paket Usang dari Repositori

```bash
sudo xbps-remove -o
```
Menghapus paket yang sudah tidak ada lagi di repositori (misalnya setelah update major version).

## ⚠️ Perilaku Penting

### 1. Konfigurasi yang Tersimpan

Secara default, **file konfigurasi paket TIDAK dihapus** saat menjalankan `xbps-remove`. File-file di `/etc` tetap tersimpan di sistem. Hal ini dilakukan agar konfigurasi Anda tidak hilang jika ingin menginstall ulang paket tersebut nanti .

Untuk menghapus juga file konfigurasi, gunakan:
```bash
sudo xbps-remove -R --configpreserve=false firefox
```

### 2. Pengecekan Dependensi

`xbps-remove` secara otomatis memeriksa dependensi:
- Jika ada paket lain yang membutuhkan paket yang akan dihapus → operasi **ditolak**
- Kecuali Anda menggunakan `-f` (force)

### 3. Perbedaan `-R` vs Tanpa `-R`

| Tanpa `-R` | Dengan `-R` |
|------------|-------------|
| Hanya menghapus paket yang disebutkan | Menghapus paket + semua dependensinya yang tidak terpakai |
| Dependensi tetap terinstall | Dependensi yang tidak dibutuhkan dibersihkan |
| Lebih cepat | Lebih lambat (harus menghitung dependensi) |

## 🗂️ File yang Terpengaruh

| Lokasi | Isi |
|--------|-----|
| `/var/db/xbps/pkgdb-0.38.plist` | Database paket - entri paket dihapus |
| File sistem (usr/bin, usr/lib, etc) | File milik paket dihapus (kecuali konfigurasi jika `--configpreserve=true`) |
| `/var/cache/xbps` | Cache paket (jika dihapus dengan `-C`) |

---

## 📊 Perbandingan dengan Package Manager Lain

| Fungsi | XBPS (Void) | APT (Debian) | Pacman (Arch) | DNF (Fedora) |
|--------|-------------|--------------|---------------|--------------|
| Hapus paket | `xbps-remove nano` | `apt remove nano` | `pacman -R nano` | `dnf remove nano` |
| Hapus + dependensi | `xbps-remove -R nano` | `apt autoremove nano` | `pacman -Rs nano` | `dnf autoremove nano` |
| Hapus paket + konfigurasi | `xbps-remove --configpreserve=false nano` | `apt purge nano` | `pacman -Rn nano` | `dnf remove nano` |
| Hapus orphans | `xbps-remove -O` | `apt autoremove` | `pacman -Rns $(pacman -Qdtq)` | `dnf autoremove` |
| Hapus cache | `xbps-remove -C` | `apt clean` | `pacman -Sc` | `dnf clean all` |

## 💡 Best Practices

### Urutan Pembersihan Rutin

```bash
# 1. Update sistem terlebih dahulu
sudo xbps-install -Su

# 2. Cek orphans sebelum dihapus
xbps-query -O

# 3. Hapus orphans
sudo xbps-remove -O

# 4. Bersihkan cache (opsional)
sudo xbps-remove -C
```

### Hapus Paket dengan Benar

```bash
# Cara yang benar - beserta dependensinya yang tidak terpakai
sudo xbps-remove -R nama_paket
```

### Simulasi Sebelum Eksekusi

```bash
# Lihat dampak terlebih dahulu
xbps-remove -Rn firefox

# Jika sudah yakin...
sudo xbps-remove -R firefox
```

---

## ⚠️ Catatan Penting

1. **Butuh hak akses root** - Semua operasi penghapusan memerlukan `sudo` karena mengubah file sistem .

2. **Jangan asal force** - `-f` (force) dapat merusak sistem jika digunakan tanpa pemahaman yang jelas .

3. **Orphans aman dihapus** - Paket yang muncul di `xbps-query -O` aman untuk dihapus karena tidak dibutuhkan oleh paket lain .

4. **Konfigurasi tersimpan** - Defaultnya konfigurasi tidak dihapus, jadi Anda aman untuk reinstall nanti .

5. **Cek dependensi reverse dulu** - Sebelum menghapus paket besar, lihat dulu paket apa saja yang bergantung padanya:
   ```bash
   xbps-query -X nama_paket
   ```

---

## 🔗 Kesimpulan

`xbps-remove` adalah alat **penghapusan paket** di ekosistem XBPS. Anda akan menggunakannya ketika:

| Situasi | Perintah |
|---------|----------|
| **Menghapus aplikasi yang tidak terpakai** | `sudo xbps-remove -R <paket>` |
| **Membersihkan sistem dari dependensi sampah** | `sudo xbps-remove -O` |
| **Membebaskan ruang disk dari cache** | `sudo xbps-remove -C` |
| **Menghapus paket yang konflik atau rusak** | `sudo xbps-remove -f <paket>` (hati-hati) |

**Ingat:** Gunakan selalu `-R` untuk penghapusan yang bersih, dan jalankan `xbps-remove -O` secara berkala untuk menjaga sistem tetap rapi.
