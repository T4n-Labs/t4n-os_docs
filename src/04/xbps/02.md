## `xbps-alternatives` pada Void Linux

`xbps-alternatives` adalah utilitas dalam package manager XBPS (X Binary Package System) yang berfungsi untuk mengelola sistem **alternatives** di Void Linux .

---

## 🎯 Apa itu Sistem "Alternatives"?

**Sistem alternatives** memungkinkan beberapa package menyediakan fungsionalitas yang sama melalui file-file yang seharusnya saling konflik, dengan cara membuat **symbolic link** dari path umum ke versi spesifik package yang dipilih oleh pengguna .

### Contoh Penggunaan

Bayangkan Anda memiliki beberapa program yang bisa melakukan fungsi yang sama, misalnya:
- **Editor teks**: `nano`, `vim`, `emacs` bisa menyediakan perintah `/usr/bin/editor`
- **Java**: beberapa versi Java bisa menyediakan `/usr/bin/java`
- **Web server**: mungkin ada alternatif konfigurasi default

Tanpa sistem alternatives, Anda harus:
1. Memilih satu package secara manual
2. Menghapus package lain jika ingin beralih
3. Mengelola symbolic link secara manual

Dengan `xbps-alternatives`, Anda bisa dengan mudah "mengganti" provider suatu fungsi tanpa menghapus package lain.

---

## 🔧 Cara Kerja

```
Saat package diinstall:
├── Alternative groups terdaftar di database package (pkgdb)
└── Package menjadi provider untuk group tertentu

Saat package dihapus:
├── Alternative groups dihapus dari database
└── Jika ada provider lain, otomatis diset sebagai default 
```

---

## 📝 Perintah Dasar

### 1. **Melihat daftar alternatives** (`-l` / `--list`)

```bash
# List semua alternative groups yang terinstall
xbps-alternatives -l

# List alternatives dari package tertentu
xbps-alternatives -l <nama_package>

# List group tertentu
xbps-alternatives -l -g <nama_group>
```

### 2. **Mengatur alternatives** (`-s` / `--set`)

```bash
# Set semua alternative groups dari suatu package
xbps-alternatives -s <nama_package>

# Set hanya group tertentu dari suatu package
xbps-alternatives -s <nama_package> -g <nama_group>
```

### 3. **Mode Repository** (`-R`)

```bash
# Melihat alternatives yang tersedia di repositori (belum tentu terinstall)
xbps-alternatives -R -l
```

---

## 📋 Contoh Kasus

### Kasus 1: Editor Teks

Misalkan Anda menginstall `nano` dan `vim`. Keduanya menyediakan `/usr/bin/editor`:

```bash
# Lihat alternatives untuk editor
xbps-alternatives -l -g editor

# Output:
# editor:
#   current: /usr/bin/nano -> /usr/bin/editor
#   available:
#     vim: /usr/bin/vim
#     nano: /usr/bin/nano (current)

# Beralih ke vim
xbps-alternatives -s vim -g editor

# Sekarang /usr/bin/editor mengarah ke vim
```

### Kasus 2: Multiple Java Versions

```bash
# Lihat group java
xbps-alternatives -l -g java

# Set ke openjdk11
xbps-alternatives -s openjdk11 -g java

# Set ke openjdk17
xbps-alternatives -s openjdk17 -g java
```

---

## 🧠 Mekanisme Internal

| Event | Yang Terjadi |
|-------|--------------|
| **Package diinstall** | Alternative groups diregistrasi ke `/var/db/xbps/pkgdb-0.38.plist`  |
| **Package dihapus** | Alternative groups dihapus dari database, provider sebelumnya menjadi default otomatis  |
| **User pilih alternative** | Symlink di system path diperbarui sesuai pilihan |
| **Tidak ada alternative aktif** | System fallback ke default (biasanya package pertama yang diinstall) |

---

## 💡 Perbandingan dengan Sistem Lain

| Distro | Tool | Sintaks |
|--------|------|---------|
| **Void Linux** | `xbps-alternatives` | `xbps-alternatives -s <pkg> -g <group>` |
| **Debian/Ubuntu** | `update-alternatives` | `update-alternatives --config <group>` |
| **Arch Linux** | Tidak ada sistem bawaan (pakai manual atau AUR helpers) | - |

---

## 📂 File Terkait

- **Database package**: `/var/db/xbps/pkgdb-0.38.plist` 
- **Konfigurasi**: `/etc/xbps.d/` dan `/usr/share/xbps.d/` 
- **Manual pages**: `man xbps-alternatives` 

---

## ⚠️ Catatan Penting

1. `xbps-alternatives` hanya bekerja untuk package yang **sudah terinstall** (kecuali dengan flag `-R`) 
2. Package harus dideklarasikan sebagai provider alternatives saat dibuat (oleh maintainer package)
3. Perubahan alternatif bersifat **langsung** - symlink akan terupdate seketika
4. Void Linux secara otomatis mengatur fallback ketika provider aktif dihapus

  current: /usr/bin/vim -> /usr/bin/editor

Dari output di atas, Anda bisa melihat group `java` dan `editor` beserta package mana yang sedang aktif (ditandai `current`).

---

**Kesimpulan**: `xbps-alternatives` adalah fitur elegant di XBPS yang memecahkan masalah konflik file antar package dengan mekanisme symbolic link yang mudah dikelola, mirip dengan `update-alternatives` di Debian/Ubuntu tetapi diintegrasikan langsung ke dalam package manager Void Linux.
