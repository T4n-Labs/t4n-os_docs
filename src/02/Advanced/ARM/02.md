# Apple Silicon (Coming Soon)

Dukungan T4n OS untuk Apple Silicon berbasis **Asahi Linux**. Lihat [website](https://asahilinux.org) dan [dokumentasi](https://asahilinux.org/docs) Asahi Linux untuk informasi lebih lanjut.

---

## Instalasi

### 1. Siapkan UEFI Environment via Asahi Installer

Jalankan dari **macOS** (bukan dari live USB):

```bash
curl https://alx.sh > alx.sh
sh ./alx.sh
```

Pilih opsi instalasi **"UEFI environment only"**. Saat diminta, biarkan Asahi Installer membuat ruang kosong untuk partisi root T4n OS.

> ⚠️ **Jangan** mencoba mengecilkan atau mengubah APFS container secara manual dari live ISO T4n OS. Gunakan fasilitas yang disediakan oleh Asahi Installer.

---

### 2. Buat Live USB T4n OS

Unduh [ISO T4n OS untuk Apple Silicon](#) lalu buat Live USB.

U-Boot yang terinstal oleh Asahi Installer seharusnya mendeteksi USB secara otomatis. Jika tidak muncul, jalankan perintah berikut di prompt U-Boot:

```
setenv boot_targets "usb"
setenv bootmeths "efi"
boot
```

---

### 3. Instalasi via Chroot (Metode XBPS)

Ikuti panduan [Instalasi via Chroot](instalasi-chroot-t4n-os.md) dengan catatan berikut:

- **Jangan ubah** struktur disk selain ruang kosong yang disiapkan Asahi Installer
- **EFI System Partition** yang digunakan adalah partisi yang dibuat oleh Asahi Installer — mount di `/mnt/boot/efi`
- Saat instalasi paket base, instal `base-system` **dan** `asahi-base`:

```bash
xbps-install -S base-system asahi-base
```

- Saat menjalankan `grub-install`, tambahkan flag `--removable`:

```bash
grub-install --removable --target=arm64-efi --efi-directory=/boot/efi
```

---

### Bootloader Alternatif (m1n1)

Jika menggunakan bootloader selain GRUB bersama `m1n1`, pastikan executable EFI bootloader terinstal di:

```
EFI/BOOT/BOOTAA64.EFI
```

dalam EFI system partition.

`m1n1` juga dapat dikonfigurasi untuk booting langsung dari kernel dan initramfs tanpa bootloader. Edit `/etc/default/m1n1-kernel-hook` dan ubah nilai `PAYLOAD` menjadi:

```
PAYLOAD=kernel
```

## Audio

Instal paket `asahi-audio` dan aktifkan layanan yang diperlukan:

```bash
xbps-install -S asahi-audio
ln -s /etc/sv/speakersafetyd /var/service/
```

Lalu konfigurasi **PipeWire** dan **WirePlumber** — lihat [dokumentasi PipeWire](pipewire-t4n-os.md).

---

## Firmware

Perbarui firmware menggunakan `asahi-fwupdate` dari paket `asahi-scripts`:

```bash
xbps-install -S asahi-scripts
asahi-fwupdate
```

> Disarankan untuk menjalankan `asahi-fwupdate` setiap kali paket `asahi-firmware` diperbarui.
