# Locale dan Terjemahan

- **glibc** — mendukung pengaturan locale sistem maupun bahasa aplikasi
- **musl** — hanya mendukung pengaturan bahasa aplikasi (tidak mendukung locale sistem)

---

## Melihat Locale Aktif

Untuk melihat daftar locale yang saat ini aktif:

```bash
locale -a
```

---

## Mengaktifkan Locale

Edit file `/etc/default/libc-locales`, lalu hapus komentar (`#`) pada locale yang diinginkan. Contoh:

```
# en_US.UTF-8 UTF-8
# id_ID.UTF-8 UTF-8
```

Menjadi:

```
en_US.UTF-8 UTF-8
id_ID.UTF-8 UTF-8
```

Setelah itu, regenerasi paket `glibc-locales`:

```bash
sudo xbps-reconfigure -f glibc-locales
```

---

## Mengatur Bahasa Sistem

Set variabel `LANG` di `/etc/locale.conf`:

```bash
LANG=id_ID.UTF-8
```

---

## Locale untuk Aplikasi

Beberapa aplikasi menyimpan terjemahannya di paket terpisah yang harus diinstal secara manual. Cari paket bahasa yang diinginkan di repositori, contoh:

```bash
xbps-query -Rs german
xbps-query -Rs portuguese
```

Instal paket yang relevan dengan aplikasi yang kamu gunakan.

> **Contoh — LibreOffice:**
> Jika menginstal paket individual seperti `libreoffice-writer`, wajib menginstal minimal satu paket `libreoffice-i18n-*` agar terjemahan tersedia.
> Jika menginstal meta-paket `libreoffice`, paket terjemahan umum sudah disertakan secara otomatis.
