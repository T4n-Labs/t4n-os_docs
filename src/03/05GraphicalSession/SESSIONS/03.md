# Panduan Konfigurasi Wayland - T4n OS

**Wayland** adalah protokol modern yang menggantikan Xorg. Berbeda dengan Xorg, Wayland menggabungkan fungsi *display server*, *window manager*, dan *compositor* dalam satu aplikasi tunggal.

## 1. Lingkungan Desktop (DE)
Beberapa lingkungan desktop besar sudah mendukung sesi Wayland secara bawaan:

*   **GNOME:** Menggunakan sesi Wayland secara default. Aplikasi berbasis GTK+ akan otomatis menggunakan backend Wayland.
*   **KDE Plasma:** Mendukung Wayland, namun aplikasi berbasis Qt mungkin memerlukan konfigurasi tambahan jika dijalankan di luar lingkungan KDE.
*   **Enlightenment:** Memiliki dukungan sesi Wayland mandiri.

---

## 2. Standalone Compositor (Mandiri)
T4n OS menyediakan berbagai pilihan *compositor* Wayland mandiri bagi pengguna yang lebih suka membangun sistem secara minimalis:

| Compositor | Deskripsi |
| :--- | :--- |
| **Sway** | Kompositor tiling yang kompatibel dengan konfigurasi i3. |
| **River** | Kompositor tiling dinamis. |
| **Wayfire** | Kompositor 3D dengan efek visual yang kaya. |
| **Labwc** | Kompositor stacking yang terinspirasi oleh Openbox. |
| **Weston** | Kompositor referensi resmi dari proyek Wayland. |
| **Niri** | Kompositor tiling dengan mekanisme *scrolling*. |
| **Qtile** | Tersedia melalui paket `qtile-wayland`. |

> **Penting:** Beberapa *compositor* tidak menyertakan font secara default. Jika aplikasi tidak tampil atau error, pastikan Anda sudah menginstal paket font (seperti `fonts-roboto` atau `ttf-dejavu`).

---

## 3. Driver Grafis & Seat Management

### **Driver Video**
*   **Mesa/Open Source:** Sebagian besar kompositor membutuhkan driver yang mengimplementasikan antarmuka **GBM**. Pastikan paket `mesa-dri` sudah terinstal.
*   **NVIDIA:** GNOME dan KDE Plasma mendukung driver proprietary NVIDIA melalui backend **EGLStreams**. Kompositor lain mungkin memerlukan konfigurasi khusus atau driver versi terbaru yang mendukung GBM.

### **Manajemen Seat**
Wayland memerlukan kontrol atas perangkat input dan tampilan. Di T4n OS, Anda harus mengaktifkan salah satu layanan manajemen seat berikut:
*   `elogind`
*   `seatd`

---

## 4. Konfigurasi Aplikasi (Native Wayland)

Agar aplikasi berjalan secara "native" (bukan melalui emulasi X11), Anda perlu mengatur variabel lingkungan berdasarkan toolkit yang digunakan:

| Toolkit / Aplikasi | Paket yang Diperlukan | Variabel Lingkungan |
| :--- | :--- | :--- |
| **Qt5** | `qt5-wayland` | `QT_QPA_PLATFORM=wayland` |
| **Qt6** | `qt6-wayland` | `QT_QPA_PLATFORM=wayland` |
| **SDL** | - | `SDL_VIDEODRIVER=wayland` |
| **EFL** | - | `ELM_DISPLAY=wl` |
| **GTK+** | - | Otomatis (Default) |

### **Web Browser**
*   **Firefox & Chromium:** Mendukung Wayland secara default pada versi terbaru.
*   **Browser Berbasis Qt (qutebrowser):** Akan berjalan native selama toolkit Qt sudah dikonfigurasi dengan benar.

---

## 5. Menjalankan Aplikasi X11 (XWayland)
Jika sebuah aplikasi belum mendukung Wayland secara native, Anda tetap bisa menjalankannya menggunakan **XWayland**. Ini adalah server X yang berjalan sebagai jembatan di dalam sesi Wayland.
*   **Paket:** `xorg-server-xwayland` (biasanya terinstal otomatis sebagai dependensi kompositor).

---

## 6. Pengaturan Variabel Lingkungan
Wayland membutuhkan direktori runtime agar soket komunikasinya berfungsi. Pastikan variabel berikut terdefinisi dalam sesi Anda (biasanya diatur otomatis oleh pengelola sesi):

1.  **XDG\_RUNTIME\_DIR:** Menentukan lokasi direktori soket Wayland.
2.  **XDG\_SESSION\_TYPE:** Atur ke `wayland` untuk memberitahu aplikasi bahwa Anda berada di sesi Wayland.

```bash
# Contoh pengecekan di terminal
echo $XDG_SESSION_TYPE
```
