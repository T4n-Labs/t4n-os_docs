# Panduan Konfigurasi Xorg - T4n OS

Bagian ini merinci instalasi manual dan konfigurasi **Xorg Display Server** serta layanan terkait lainnya. Jika Anda menginginkan sistem yang langsung siap pakai dengan lingkungan desktop lengkap, disarankan untuk langsung menggunakan **image XFCE** T4n OS.

---

## 1. Instalasi

T4n OS menyediakan dua jalur instalasi utama untuk Xorg:

*   **Paket Lengkap (`xorg`):**
    Ini adalah opsi paling aman. Paket ini menginstal server Xorg, semua driver video gratis, driver input, font, dan aplikasi dasar. Sangat memadai untuk sistem yang tidak memerlukan driver proprietary.
*   **Paket Minimal (`xorg-minimal`):**
    Hanya menginstal server dasar. Jika Anda memilih ini, Anda harus menginstal komponen lain secara manual agar sistem grafis dapat digunakan, seperti:
    *   Font (contoh: `xorg-fonts`)
    *   Terminal Emulator (contoh: `xterm`)
    *   Window Manager (contoh: `i3`, `openbox`)

```bash
# Untuk instalasi lengkap
sudo xbps-install -S xorg
```

---

## 2. Driver Video

T4n OS mendukung driver open-source maupun proprietary (non-free).

### **Driver Open Source**
Terdapat dua kategori driver open source yang bisa digunakan:

1.  **DDX (Device Dependent X):**
    *   Disediakan oleh paket `xf86-video-*`.
    *   Terinstal otomatis jika Anda menggunakan paket `xorg`.
    *   Contoh: `xf86-video-intel`, `xf86-video-amdgpu`.
2.  **Modesetting:**
    *   Memerlukan paket `mesa-dri`.
    *   Tidak butuh paket driver spesifik vendor.
    *   Xorg akan memprioritaskan DDX jika tersedia. Jika ingin menggunakan modesetting, Anda harus memaksanya melalui konfigurasi (lihat bagian bawah).

### **Driver Proprietary**
Tersedia melalui repositori **nonfree**, khususnya untuk pengguna kartu grafis **NVIDIA**.

---

## 3. Driver Input
Driver input menangani perangkat seperti keyboard, mouse, hingga tombol power. Tersedia melalui paket `xf86-input-*`. Jika perangkat input tidak merespons pada instalasi `xorg-minimal`, Anda mungkin perlu menginstal driver spesifik seperti `xf86-input-evdev` atau `xf86-input-libinput`.

---

## 4. Konfigurasi Xorg

Secara umum, Xorg akan mendeteksi perangkat secara otomatis. Namun, konfigurasi manual dapat dilakukan di direktori `/etc/X11/xorg.conf.d/`.

### **Contoh: Konfigurasi Keyboard**
Buat file `/etc/X11/xorg.conf.d/30-keyboard.conf`:
```text
Section "InputClass"
    Identifier "keyboard-all"
    Driver "evdev"
    MatchIsKeyboard "on"
EndSection
```

### **Memaksa Driver Modesetting**
Jika Anda ingin menggunakan *modesetting* alih-alih driver DDX, buat file `/etc/X11/xorg.conf.d/10-modesetting.conf`:
```text
Section "Device"
    Identifier "GPU0"
    Driver "modesetting"
EndSection
```
**Verifikasi:** Setelah restart Xorg, jalankan perintah ini untuk memastikan:
```bash
grep -E -m1 '\(II\) modeset\([0-9]+\):' /var/log/Xorg.0.log
```

---

## 5. Memulai Sesi X (Start X)

### **Menggunakan `startx`**
Paket `xinit` menyediakan skrip `startx` untuk menjalankan sesi X langsung dari konsol (TTY). Konfigurasinya dilakukan melalui file `~/.xinitrc`.

**Contoh isi file `~/.xinitrc`:**
```bash
# Menjalankan aplikasi di latar belakang
xscreensaver &

# Menjalankan Window Manager (Harus baris terakhir dengan 'exec')
exec i3
```
Setelah file disimpan, cukup jalankan perintah `startx` di terminal.

---

## 6. Display Manager (DM)

**Display Manager** menyediakan antarmuka grafis untuk login (pengganti login teks di konsol). T4n OS menyediakan beberapa pilihan populer:

*   **GDM:** Default untuk lingkungan GNOME.
*   **SDDM:** Default untuk KDE Plasma.
*   **LightDM:** Ringan dan fleksibel, sering digunakan untuk XFCE atau Window Manager.

> **Penting:** Pastikan untuk menguji fungsionalitas Xorg Anda dengan `startx` sebelum mengaktifkan layanan Display Manager secara permanen agar tidak terjebak dalam layar hitam saat boot.
