# Panduan XDG Desktop Portals - T4n OS

**XDG Desktop Portals** adalah antarmuka standar yang digunakan oleh aplikasi (terutama **Flatpak**, Snap, dan aplikasi sandboxed lainnya) untuk berinteraksi dengan sistem host. Portal ini menyediakan akses aman ke fitur-fitur seperti dialog buka/simpan file, *clipboard*, berbagi layar (*screencasting*), membuka URL, dan banyak lagi.

## 1. Instalasi

Untuk menggunakan XDG Desktop Portal, sistem Anda harus menjalankan **D-Bus user session bus**. 

Langkah pertama adalah menginstal paket utama:
```bash
sudo xbps-install -S xdg-desktop-portal
```

Selanjutnya, Anda harus menginstal setidaknya satu **backend** yang sesuai dengan lingkungan desktop atau kompositor yang Anda gunakan:

| Backend | Catatan / Kegunaan |
| :--- | :--- |
| **xdg-desktop-portal-gtk** | **Pilihan Default Terbaik.** Menyediakan antarmuka umum (UI berbasis GTK+). |
| **xdg-desktop-portal-gnome** | Antarmuka umum dan fitur spesifik untuk GNOME. |
| **xdg-desktop-portal-kde** | Antarmuka umum dan fitur spesifik untuk KDE (UI berbasis Qt/KF5). |
| **xdg-desktop-portal-wlr** | **Wajib untuk Wayland (wlroots)** seperti Sway/Hyprland untuk fitur screenshot dan berbagi layar. |
| **xdg-desktop-portal-lxqt** | Hanya menyediakan pemilih file (berbasis *libfm-qt*). |
| **io.elementary.files** | Hanya menyediakan pemilih file. |

> **Tips:** Jika Anda ragu, instal `xdg-desktop-portal-gtk` karena kompatibel dengan hampir semua lingkungan desktop.

---

## 2. Konfigurasi

Secara umum, konfigurasi bawaan yang terletak di `/usr/share/xdg-desktop-portal/portals.conf` sudah cukup untuk kebutuhan sebagian besar pengguna.

### **Kustomisasi (Override)**
Jika Anda perlu menyesuaikan portal untuk lingkungan desktop tertentu atau antarmuka portal spesifik, Anda bisa menimpanya dengan membuat file konfigurasi di lokasi berikut:

*   **Level Sistem:** `/etc/xdg-desktop-portal/portals.conf`
*   **Level Pengguna:** `~/.config/xdg-desktop-portal/portals.conf` atau `$XDG_CONFIG_HOME/xdg-desktop-portal/$XDG_CURRENT_DESKTOP-portals.conf`

Format konfigurasi ini memungkinkan Anda menentukan backend mana yang harus menangani layanan tertentu (misalnya: menggunakan backend KDE untuk file chooser, tapi backend GTK untuk notifikasi). Untuk panduan lengkap penulisan file ini, gunakan perintah:
```bash
man 5 portals.conf
```

---

## 3. Verifikasi
Pastikan portal berjalan setelah instalasi dengan memeriksa proses D-Bus atau mencoba membuka file dialog dari aplikasi Flatpak. Jika menggunakan lingkungan minimal (seperti Window Manager), pastikan `dbus-run-session` sudah aktif saat Anda memulai sesi desktop.
