# Dokumentasi T4n OS XFCE-Wayland Edition

**T4n OS XFCE-Wayland** adalah edisi eksperimental yang menawarkan pengalaman desktop XFCE yang ringan namun berjalan di atas protokol **Wayland**. Varian ini memberikan performa visual yang lebih mulus, keamanan yang lebih baik, dan dukungan *tearing-free* secara bawaan.

## 1. Arsitektur Sesi
Dalam edisi ini, server grafis Xorg digantikan oleh arsitektur Wayland:
*   **Compositor:** `labwc` (berbasis wlroots, meniru perilaku Openbox/Xfwm4) atau `xfce4-wayland-session`.
*   **Protokol:** Wayland dengan kompatibilitas **XWayland** untuk aplikasi lama.
*   **Seat Management:** `seatd` atau `elogind`.

---

## 2. Persyaratan Driver
Wayland membutuhkan driver dengan dukungan **GBM (Generic Buffer Management)**.
*   **Intel:** Menggunakan `mesa-dri`.
*   **AMD:** Menggunakan `mesa-dri` dan `xf86-video-amdgpu` (kernel-mode).
*   **NVIDIA:** Membutuhkan driver versi 470+ (disarankan 525+) dengan mode `nvidia-drm.modeset=1` diaktifkan pada kernel cmdline.

---

## 3. Manajemen Sesi & Autostart

### **Memulai Sesi**
T4n OS menggunakan **Greetd** atau **SDDM** yang dikonfigurasi untuk Wayland. Untuk menjalankan sesi secara manual dari TTY:
```bash
start-xfce-wayland
```

### **Variabel Lingkungan (Environment Variables)**
Agar toolkit aplikasi berjalan secara native di Wayland, file `/etc/environment` atau `~/.bash_profile` menyertakan:
```bash
export CLUTTER_BACKEND=wayland
export GDK_BACKEND=wayland,x11
export QT_QPA_PLATFORM=wayland
export MOZ_ENABLE_WAYLAND=1
```

---

## 4. Komponen Desktop di Wayland
Beberapa aplikasi standar XFCE telah diganti dengan alternatif native Wayland untuk stabilitas:

*   **Panel:** `xfce4-panel` (versi GTK3+) dengan dukungan plugin `wayland-proxy`.
*   **Wallpaper & Desktop:** `wpaperd` atau `swaybg`.
*   **App Launcher:** `xfce4-appfinder` atau `wofi`.
*   **Screenshot:** `grim` + `slurp` (menggantikan `xfce4-screenshooter`).
*   **Notifications:** `mako` atau `xfce4-notifyd` (versi terbaru).

---

## 5. Konfigurasi Input & Layar

### **Input (Mouse & Touchpad)**
Pengaturan input tidak lagi dikelola oleh `xf86-input-libinput`, melainkan langsung oleh compositor. Konfigurasi dapat ditemukan di:
`~/.config/labwc/autostart` atau melalui tool `wlopm`.

### **Display (Output)**
Untuk mengatur resolusi dan orientasi monitor, gunakan tool **wlr-randr** atau antarmuka grafis **wdisplays**:
```bash
# Contoh melihat status monitor
wlr-randr
```

---

## 6. XWayland (Kompatibilitas Aplikasi)
Untuk aplikasi yang belum mendukung Wayland secara native (seperti beberapa game lama atau tools tertentu), T4n OS mengaktifkan **XWayland** secara otomatis.
*   **Verifikasi:** Jalankan perintah `xeyes`. Jika mata bergerak saat kursor di atas jendela aplikasi, maka aplikasi tersebut berjalan di bawah XWayland.

---

## 7. Troubleshooting (Pemecahan Masalah)

### **Masalah Layar Berkedip (Flickering)**
Hal ini sering terjadi pada GPU NVIDIA. Pastikan modul kernel sudah dimuat dengan benar:
```bash
# Tambahkan ke kernel parameters
nvidia_drm.modeset=1 nvidia_drm.fbdev=1
```

### **Aplikasi GTK Menampilkan Dekorasi Jendela Ganda**
Gunakan variabel lingkungan berikut untuk memaksa dekorasi sisi klien:
`GTK_CSD=1`

### **Berbagi Layar (Screencasting)**
Agar fitur berbagi layar di Discord atau OBS berfungsi, pastikan layanan berikut aktif:
1. `xdg-desktop-portal`
2. `xdg-desktop-portal-wlr` (atau sesuai compositor)
3. `pipewire` dan `wireplumber`
