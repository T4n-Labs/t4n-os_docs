# Panduan NVIDIA Optimus - T4n OS

**NVIDIA Optimus** adalah teknologi grafis ganda pada laptop yang terdiri dari GPU terintegrasi (**Intel**) dan GPU diskrit (**NVIDIA**). Untuk menggunakan GPU NVIDIA pada konfigurasi ini, Anda harus memilih metode yang sesuai dengan perangkat keras Anda.

## 1. Identifikasi Driver yang Tepat
Dukungan Optimus tidak selalu menjamin versi driver terbaru akan bekerja. Cara terbaik untuk menentukannya adalah dengan mencoba:

1.  Instal paket `nvidia` terbaru dan lakukan *reboot*.
2.  Cek log kernel. Jika kartu grafis Anda tidak didukung oleh driver tersebut, Anda akan melihat pesan kesalahan seperti ini:
    > `NVRM: The NVIDIA GPU xxxx:xx:xx.x ... is not supported by the xxx.xx NVIDIA Linux driver release.`
3.  **Solusi:** Jika pesan tersebut muncul, hapus driver `nvidia` dan instal driver legacy `nvidia390`.

---

## 2. Metode yang Didukung (Pilih Salah Satu)

Berikut adalah ringkasan metode yang tersedia di T4n OS. Metode-metode ini bersifat eksklusif (tidak bisa dijalankan bersamaan):

| Metode | Driver yang Mendukung | Kelebihan | Kekurangan |
| :--- | :--- | :--- | :--- |
| **PRIME Render Offload** | `nvidia`, `nvidia470` | Paling fleksibel, bisa dipilih per aplikasi. | Manajemen daya pada seri lama (pre-Turing) kurang optimal. |
| **RandR 1.4** | `nvidia`, `nvidia470`, `nvidia390` | Bisa mematikan GPU NVIDIA sepenuhnya. | Kurang fleksibel (harus ditentukan di awal sesi X). |
| **Bumblebee** | `nvidia`, `nvidia470`, `nvidia390` | Manajemen daya baik. | Metode tidak resmi, performa cenderung rendah. |
| **Nouveau PRIME** | `nouveau` (Open Source) | Berbasis open-source. | Performa rendah karena hasil *reverse-engineering*. |

---

## 3. Cara Penggunaan & Konfigurasi

### **A. PRIME Render Offload (Direkomendasikan)**
Metode ini memungkinkan Anda menjalankan aplikasi tertentu menggunakan GPU NVIDIA melalui variabel lingkungan. T4n OS menyediakan skrip pembantu `prime-run` yang disertakan dalam paket `nvidia`.

*   **Cara menjalankan aplikasi:**
    ```bash
    prime-run <nama_aplikasi>
    ```
    *Contoh: `prime-run steam`*

### **B. Bumblebee**
Jika Anda memilih metode ini untuk penghematan daya ekstra:
1.  Aktifkan layanan `bumblebeed`.
2.  Masukkan *user* Anda ke dalam grup `bumblebee`:
    
```bash
    sudo usermod -aG bumblebee $USER
 ```
3.  *Log out* dan *log in* kembali.
4.  **Cara menjalankan aplikasi:**
    ```bash
    optirun <nama_aplikasi>
    ```

### **C. Nouveau PRIME**
Jika Anda menggunakan driver *open-source* Nouveau:
1.  Pastikan driver NVIDIA sudah dihapus atau dinonaktifkan.
2.  Gunakan variabel `DRI_PRIME=1`.
3.  **Cara menjalankan aplikasi:**
    ```bash
    DRI_PRIME=1 <nama_aplikasi>
    ```

---

## 4. Cara Verifikasi GPU yang Aktif

Untuk memastikan aplikasi Anda benar-benar berjalan di atas GPU NVIDIA, gunakan metode berikut:

1.  **Cek Renderer (Membutuhkan paket `glxinfo`):**
    ```bash
    glxinfo | grep "OpenGL renderer"
    ```
    Atau jika menggunakan `prime-run`:
    ```bash
    prime-run glxinfo | grep "OpenGL renderer"
    ```

2.  **Cek Status Proses (Khusus driver NVIDIA):**
    Jalankan perintah berikut untuk melihat daftar proses yang sedang menggunakan GPU diskrit:
    
```bash
    nvidia-smi
```
