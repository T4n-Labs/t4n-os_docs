# Panduan Konfigurasi GPU Intel - T4n OS

Dokumentasi ini berisi langkah-langkah instalasi driver, akselerasi video, hingga pemecahan masalah untuk pengguna GPU Intel pada T4n OS.

## 1. Instalasi Firmware Dasar
Untuk dukungan penuh GPU Intel, diperlukan paket firmware yang sesuai.

*   **Pemasangan Otomatis:** Jika Anda menggunakan paket `linux` atau `linux-lts`, paket `linux-firmware-intel` akan terinstal secara otomatis sebagai dependensi.
*   **Pemasangan Manual:** Jika Anda menggunakan kernel versi spesifik (contoh: `linux5.4`), Anda wajib menginstal firmware secara manual:
    ```bash
    sudo xbps-install -S linux-firmware-intel
    ```

## 2. Driver Grafis (OpenGL & Vulkan)

Agar aplikasi 3D dan game berjalan optimal, Anda perlu menginstal driver API grafis berikut:

### **OpenGL**
Membutuhkan paket **Mesa DRI** (`mesa-dri`).
*   Paket ini sudah termasuk dalam meta-paket `xorg`.
*   **Wajib instal manual** jika Anda menggunakan `xorg-minimal` atau menjalankan sesi **Wayland**.

### **Vulkan**
Untuk dukungan API Vulkan, instal pemuat (loader) dan driver spesifik Intel:
*   `vulkan-loader`
*   `mesa-vulkan-intel`

---

## 3. Akselerasi Video (Hardware Acceleration)

Gunakan fitur ini untuk meringankan beban CPU saat memutar video (YouTube, pemutar video, dll).

### **VA-API (Video Acceleration API)**
Instal meta-paket `intel-video-accel` untuk mendapatkan semua driver yang diperlukan. Secara default, sistem akan menggunakan `intel-media-driver`. Jika perlu menggantinya secara manual, gunakan variabel lingkungan `LIBVA_DRIVER_NAME`:

| Driver | Generasi GPU | Variabel Lingkungan |
| :--- | :--- | :--- |
| **libva-intel-driver** | Hingga Coffee Lake | `LIBVA_DRIVER_NAME=i965` |
| **intel-media-driver** | Broadwell dan yang lebih baru | `LIBVA_DRIVER_NAME=iHD` |

### **VDPAU**
Jika memerlukan dukungan VDPAU, instal paket `libvdpau-va-gl` dan atur variabel lingkungan berikut:
`VDPAU_DRIVER=va_gl`

## 4. Pemecahan Masalah (Troubleshooting)

### **Masalah IOMMU (Khusus Broadwell)**
Kernel pada T4n OS dikonfigurasi dengan `CONFIG_INTEL_IOMMU_DEFAULT_ON=y`. Pada GPU generasi **Broadwell**, hal ini sering menyebabkan masalah driver grafis.
*   **Solusi:** Matikan IOMMU untuk GPU internal dengan menambahkan parameter berikut ke baris perintah kernel (*kernel cmdline*):
    `intel_iommu=igfx_off`
*   *Catatan: Jika GPU model lain mengalami masalah serupa dan sembuh dengan opsi ini, harap laporkan sebagai bug.*

### **Gangguan Driver DDX**
Pada chipset Intel yang lebih baru, driver DDX lama terkadang menyebabkan instabilitas atau akselerasi grafis tidak berjalan.
*   **Solusi:** Coba hapus semua paket `xf86-video-*`.
*   **Pengecualian:** Beberapa chipset lama justru **membutuhkan** driver ini. Jika tampilan bermasalah setelah penghapusan, instal kembali paket `xf86-video-intel`.
