## 1. Pendahuluan

**Waydroid** adalah solusi container-based untuk menjalankan Android (LineageOS) secara native di Linux dengan performa tinggi, tanpa emulasi hardware. Berbeda dengan emulator tradisional, Waydroid menggunakan container LXC yang memberikan akses langsung ke hardware host, sehingga hampir tidak ada overhead performa.

**Weston** adalah referensi Wayland compositor yang dapat berjalan sebagai aplikasi di dalam sesi X11, bertindak sebagai "jembatan" agar Waydroid (yang membutuhkan Wayland) dapat berjalan di lingkungan X11.

**Mengapa dokumentasi ini penting untuk T4n OS?** T4n OS berbasis Void Linux, yang menggunakan `xbps` sebagai package manager dan `runit` sebagai init system (bukan systemd). Dokumentasi resmi Waydroid umumnya mengasumsikan systemd, sehingga diperlukan penyesuaian khusus.

### Cara Kerja Arsitektur
```
X11 Session (Desktop Host)
    └── Weston (Wayland Compositor sebagai jendela)
        └── Waydroid Container (LineageOS Android)
            └── Aplikasi Android
```

## 2. Prasyarat Sistem

### Spesifikasi Minimal
- CPU dengan dukungan virtualisasi (Intel VT-x / AMD-V)
- RAM minimal 4 GB (8 GB direkomendasikan)
- Ruang disk minimal 4 GB untuk image Android
- Kernel Linux dengan modul binder (kernel bawaan Void modern sudah mendukung)

### Cek Lingkungan X11
Pastikan Anda sedang berjalan di sesi X11:
```bash
echo $XDG_SESSION_TYPE
# Output harus: x11
```

### Update Sistem
```bash
sudo xbps-install -Su
```

## 3. Instalasi Waydroid dan Weston

### 3.1 Instalasi Weston
Weston adalah Wayland compositor yang akan berjalan di dalam X11 Anda:
```bash
sudo xbps-install weston
```

### 3.2 Instalasi Waydroid
Untuk Void Linux, Waydroid tersedia di repositories resmi:
```bash
sudo xbps-install waydroid
```

> **Catatan untuk T4n OS:** Jika paket waydroid tidak tersedia di repositori default, Anda mungkin perlu mengaktifkan repository `nonfree` atau `multilib/nonfree` terlebih dahulu.

### 3.3 Verifikasi Instalasi
```bash
# Cek versi waydroid
waydroid --version

# Cek ketersediaan modul kernel
lsmod | grep binder
```

Jika modul binder tidak muncul, coba load manually:
```bash
sudo modprobe ashmem_linux
sudo modprobe binder_linux
```

## 4. Konfigurasi Awal Waydroid

### 4.1 Inisialisasi Image Android
Pilih salah satu opsi di bawah ini:

**Opsi A: Minimal (LineageOS saja, tanpa Google Apps):**
```bash
sudo waydroid init
```

**Opsi B: Dengan GAPPS (Google Play Services):**
```bash
sudo waydroid init -s GAPPS
```

Proses ini akan mengunduh image Android (sekitar 2 GB) ke `/var/lib/waydroid/images/`. Waktu unduh tergantung kecepatan internet.

### 4.2 Konfigurasi Network (runit)
Karena Void Linux menggunakan **runit** (bukan systemd), kita perlu mengatur service secara manual:

**Buat service script untuk waydroid-container:**

Buat file `/etc/sv/waydroid-container/run`:

```bash
#!/bin/sh
exec waydroid-container-start 2>&1
```

Buat symlink untuk enable service:
```bash
sudo mkdir -p /etc/sv/waydroid-container
sudo ln -s /etc/sv/waydroid-container /var/service/
```

**Start container:**
```bash
sudo waydroid container start
```

**Cek status container:**
```bash
ps aux | grep waydroid
```

## 5. Menjalankan Waydroid di X11 dengan Weston

### 5.1 Cara Manual (Untuk Testing)

**Step 1: Start Weston di background**
```bash
weston &
```

Anda akan melihat jendela Weston baru terbuka di desktop X11 Anda.

**Step 2: Buka terminal di dalam jendela Weston**
- Klik ikon terminal di pojok kiri atas jendela Weston

**Step 3: Jalankan Waydroid UI dari dalam terminal Weston**
```bash
waydroid show-full-ui
```

Android akan terbuka di dalam jendela Weston.

### 5.2 Cara Otomatis dengan Script

Buat script `~/waydroid-x11.sh`:

```bash
#!/bin/bash

# Script untuk menjalankan Waydroid di X11 menggunakan Weston
# Untuk T4n OS (Void Linux)

echo "Starting Weston..."
weston &

# Tunggu Weston siap
sleep 3

# Cek apakah Weston berjalan
if pgrep -x "weston" > /dev/null; then
    echo "Weston started successfully"
    
    # Jalankan waydroid session start di background
    waydroid session start &
    
    # Tunggu session siap
    sleep 2
    
    # Tampilkan UI
    echo "Starting Waydroid UI..."
    waydroid show-full-ui
    
    echo "Waydroid is running inside Weston window"
else
    echo "Failed to start Weston"
    exit 1
fi
```

Buat script executable:
```bash
chmod +x ~/waydroid-x11.sh
```

Jalankan:
```bash
~/waydroid-x11.sh
```

### 5.3 Alternatif: Menggunakan Cage (Lebih Ringan)

Selain Weston, Anda juga bisa menggunakan **Cage** - Wayland compositor minimal yang lebih ringan:

```bash
sudo xbps-install cage
```

Jalankan dengan:
```bash
cage -- waydroid show-full-ui
```

**Keunggulan Cage:** Tidak ada dekorasi jendela tambahan, lebih ringan dari Weston.

## 6. Integrasi dengan Desktop Environment

### 6.1 Membuat Desktop Entry (.desktop)

Buat file `~/.local/share/applications/waydroid-x11.desktop`:

```ini
[Desktop Entry]
Name=Waydroid (X11)
Comment=Run Android apps on Linux
Exec=/home/username/waydroid-x11.sh
Icon=waydroid
Terminal=false
Type=Application
Categories=System;Emulator;
```

### 6.2 Keyboard Shortcut (Opsional)

Jika menggunakan Window Manager seperti i3, dwm, atau bspwm, tambahkan binding:

**Contoh untuk bspwm** (`~/.config/bspwm/bspwmrc`):
```bash
# Super + W untuk launch Waydroid
super + w
    ~/waydroid-x11.sh
```

## 8. Troubleshooting

### 8.1 "WAYLAND_DISPLAY is not set" Error
**Penyebab:** Waydroid tidak menemukan Wayland socket karena Weston belum berjalan.

**Solusi:** Pastikan Weston berjalan terlebih dahulu sebelum menjalankan `waydroid show-full-ui`.

### 8.2 Waydroid Container STOPPED
**Penyebab:** Container service tidak berjalan.

**Solusi untuk Void Linux (runit):**
```bash
sudo waydroid container start
sudo sv status waydroid-container
```

### 8.3 Modul binder tidak ditemukan
**Penyebab:** Kernel tidak memiliki modul binder.

**Solusi:** Cek dukungan kernel:
```bash
zgrep BINDER /proc/config.gz
```
Jika tidak ada, Anda perlu recompile kernel dengan `CONFIG_ANDROID_BINDER_IPC=y` atau menggunakan kernel yang lebih baru.

### 8.4 Weston tidak menampilkan Waydroid
**Penyebab:** Waydroid session belum dimulai.

**Solusi:**
```bash
# Stop session jika ada
waydroid session stop

# Start session baru
waydroid session start

# Tampilkan UI
waydroid show-full-ui
```

### 8.5 Tidak ada internet di Waydroid
**Penyebab:** Firewall memblokir interface `waydroid0`.

**Solusi:** Tambahkan aturan firewall:
```bash
# Untuk iptables
sudo iptables -A FORWARD -i waydroid0 -j ACCEPT
sudo iptables -A FORWARD -o waydroid0 -j ACCEPT
```

### 8.6 Menghentikan Waydroid (runit)
Void Linux menggunakan `pkill` secara default, bukan `killall`:
```bash
sudo pkill waydroid-container
pkill weston
```

## 9. Performa dan Optimalisasi

### Faktor yang Mempengaruhi Performa
- **CPU:** Container menggunakan CPU host secara langsung, performa hampir native
- **RAM:** Alokasikan minimal 2 GB untuk container
- **GPU:** Akses GPU melalui Wayland → X11 memiliki sedikit overhead

### Tips Optimalisasi

**1. Gunakan Cage sebagai compositor (lebih ringan dari Weston):**
```bash
cage -- waydroid show-full-ui
```

**2. Alokasikan lebih banyak resource ke container:**
Edit `/etc/waydroid/waydroid.cfg`:
```ini
[resources]
memory_mb = 4096
cpu_shares = 1024
```

**3. Nonaktifkan animasi Android:**
- Masuk ke Settings → Developer Options
- Set semua animasi scale ke 0

**4. Untuk NVIDIA GPU:** 
Driver proprietary NVIDIA tidak kompatibel dengan Wayland dengan baik. Gunakan driver open-source `nouveau` atau akselerasi software.

### 10.1 Hapus Waydroid
```bash
sudo xbps-remove waydroid
sudo rm -rf /var/lib/waydroid
```

### 10.2 Hapus Weston
```bash
sudo xbps-remove weston
```

```bash
sudo rm /var/service/waydroid-container
sudo rm -rf /etc/sv/waydroid-container
```

## 📝 Catatan Akhir untuk T4n OS

T4n OS sebagai distribusi berbasis **Void Linux** memiliki karakteristik unik yang perlu diperhatikan:

| Aspek | Implementasi di T4n OS |
|-------|------------------------|
| Package Manager | `xbps` (bukan apt/pacman) |
| Init System | `runit` (bukan systemd) |
| Process Killer | `pkill` (bukan killall) |
| Default Shell | `bash` |

**Testing Status:** Dokumentasi ini telah disusun berdasarkan referensi resmi Waydroid untuk Void Linux dan panduan komunitas untuk menjalankan Waydroid di X11 menggunakan Weston. Untuk hasil terbaik, disarankan menggunakan compositor seperti Weston atau Cage dengan konfigurasi yang telah disesuaikan untuk lingkungan X11.

**Performance Note:** Menjalankan Waydroid melalui Weston di X11 akan memiliki overhead performa tambahan dibandingkan menjalankannya langsung di sesi Wayland native. Namun, ini tetap menjadi solusi terbaik untuk pengguna X11 yang tidak ingin beralih ke Wayland.
