## 1. Pendahuluan

**Waydroid** adalah solusi container berbasis LXC untuk menjalankan sistem Android (LineageOS) secara native di Linux dengan integrasi penuh ke desktop . Waydroid **hanya dapat berjalan di atas sesi Wayland** .

**Mengapa dokumentasi ini penting untuk T4n OS?** T4n OS berbasis Void Linux, yang menggunakan `xbps` sebagai package manager dan `runit` sebagai init system (bukan systemd). Dokumentasi resmi Waydroid umumnya mengasumsikan systemd, sehingga diperlukan penyesuaian khusus untuk Void Linux.

**Cara Kerja Waydroid:**
```
Wayland Compositor (GNOME/KDE/Sway/Wayfire)
    └── Waydroid Container (LineageOS Android)
        └── Aplikasi Android (terintegrasi sebagai aplikasi native)
```

## 2. Prasyarat Sistem

### 2.1 Spesifikasi Minimal
- CPU dengan dukungan virtualisasi (Intel VT-x / AMD-V)
- RAM minimal 4 GB (8 GB direkomendasikan)
- Ruang disk minimal 4 GB untuk image Android
- Kernel Linux dengan modul binder (kernel bawaan Void modern sudah mendukung)

### 2.2 Sesi Wayland (WAJIB)
Sebelum memulai, pastikan Anda sedang berjalan di sesi Wayland. Cek dengan perintah berikut:

```bash
echo $XDG_SESSION_TYPE
```

**Output yang diharapkan:** `wayland`

Jika output menunjukkan `x11`, Anda perlu memilih sesi Wayland dari display manager saat login (misalnya: GNOME on Wayland, Plasma (Wayland), atau compositor standalone seperti Sway/Wayfire).

### 2.3 Driver Grafis dan Dependencies untuk Void Linux

Untuk kelancaran Waydroid dan Wayland, pastikan driver grafis terinstall:

```bash
# Untuk AMD GPU
sudo xbps-install mesa-dri mesa-vaapi mesa-vdpau mesa-vulkan-radeon

# Untuk Intel GPU  
sudo xbps-install mesa-dri intel-video-accel mesa-intel-dri intel-ucode

# Untuk NVIDIA GPU (dengan driver proprietary)
sudo xbps-install nvidia nvidia-libs-32bit
```

> **Catatan untuk NVIDIA:** Waydroid dengan GPU NVIDIA memiliki keterbatasan pada beberapa compositor. Disarankan menggunakan GNOME atau KDE Plasma Wayland untuk kompatibilitas terbaik .

### 2.4 Update Sistem
```bash
sudo xbps-install -Su
```

## 3. Instalasi Waydroid

### 3.1 Install dari Repositori Resmi Void Linux

Waydroid tersedia di repositori resmi Void Linux :

```bash
sudo xbps-install -S waydroid
```

### 3.2 Verifikasi Instalasi

```bash
# Cek versi waydroid
waydroid --version

# Cek ketersediaan modul kernel binder
lsmod | grep binder
```

Jika modul binder tidak muncul, load secara manual:
```bash
sudo modprobe ashmem_linux
sudo modprobe binder_linux
```

Untuk membuat modul tetap termuat setelah reboot, buat file `/etc/modules-load.d/binder.conf`:
```bash
echo "binder_linux" | sudo tee /etc/modules-load.d/binder.conf
echo "ashmem_linux" | sudo tee -a /etc/modules-load.d/binder.conf
```

## 4. Konfigurasi Awal

### 4.1 Inisialisasi Image Android

**Opsi A: Minimal (LineageOS tanpa Google Apps):**
```bash
sudo waydroid init
```

**Opsi B: Dengan GAPPS (Google Play Services):**
```bash
sudo waydroid init -s GAPPS
```

Proses ini akan mengunduh image Android (sekitar 2 GB) ke `/var/lib/waydroid/images/`. Waktu unduh tergantung kecepatan internet.

### 4.2 Parameter Kernel PSI (Pressure Stall Information)

Untuk performa optimal, tambahkan parameter kernel `psi=1` :

```bash
# Backup konfigurasi boot
sudo cp /etc/default/grub /etc/default/grub.bak

# Edit GRUB config
sudo nano /etc/default/grub
```

Cari baris `GRUB_CMDLINE_LINUX_DEFAULT` dan tambahkan `psi=1`:
```
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash psi=1"
```

Update GRUB:
```bash
sudo update-grub
```

> **Catatan:** Jika menggunakan rEFInd atau bootloader lain, sesuaikan dengan caranya masing-masing.

### 4.3 Konfigurasi Container (runit)

Karena Void Linux menggunakan **runit**, kita perlu mengatur container service secara manual.

**Start container service:**
```bash
sudo waydroid container start
```

**Cek status container:**
```bash
ps aux | grep waydroid
```

**Untuk autostart saat boot**, buat file `/etc/sv/waydroid-container/run`:

```bash
#!/bin/sh
exec waydroid-container-start 2>&1
```

Buat symlink untuk enable service:
```bash
sudo mkdir -p /etc/sv/waydroid-container
sudo ln -s /etc/sv/waydroid-container /var/service/
```

## 5. Menjalankan Waydroid

### 5.1 Cara Standar (Untuk Sesi Wayland)

**Step-by-step:**

1. **Start waydroid session**
   Buka terminal dan jalankan:
   ```bash
   waydroid session start &
   ```

2. **Tampilkan UI Android**
   ```bash
   waydroid show-full-ui
   ```

3. **Atur ukuran jendela** (jika perlu)
   Gunakan shortcut compositor Anda untuk mengatur mode jendela.

### 5.2 Multi-window Mode (App Grid)

Untuk melihat semua aplikasi Android dalam grid:
```bash
waydroid app-grid
```

### 5.3 Tips untuk Sesi Wayland

Pastikan environment variable Wayland sudah ter-set dengan benar:
```bash
echo $WAYLAND_DISPLAY
# Output: wayland-0 (atau wayland-1)
```

Jika tidak ter-set, atur secara manual:
```bash
export WAYLAND_DISPLAY=wayland-0
export XDG_RUNTIME_DIR=/run/user/$(id -u)
```

## 6. Integrasi dengan Desktop Environment

### 6.1 GNOME Wayland

Waydroid terintegrasi secara native dengan GNOME Shell:
- Aplikasi Android akan muncul di menu aplikasi
- Notifikasi Android akan muncul sebagai notifikasi GNOME
- Copy-paste antara host dan container berfungsi (jika clipboard portal diaktifkan)

**Package tambahan untuk GNOME:**
```bash
sudo xbps-install gnome-shell-extensions
```

### 6.2 KDE Plasma Wayland

KDE Plasma memiliki integrasi Waydroid yang baik:
- Aplikasi Android muncul di Kickoff/Application Launcher
- Dukungan task manager untuk aplikasi Android

### 6.3 Sway/Wayfire (Standalone Compositor)

Untuk pengguna Sway atau Wayfire, pastikan package berikut terinstall:

```bash
# XDG Portal untuk integrasi desktop
sudo xbps-install xdg-desktop-portal xdg-desktop-portal-wlr

# Clipboard support untuk copy-paste
sudo xbps-install wl-clipboard
```

**Untuk autostart Waydroid di Sway** (tambahkan ke `~/.config/sway/config`):
```
exec waydroid session start
exec waydroid show-full-ui
```

### 6.4 Membuat Desktop Entry (.desktop)

Buat file `~/.local/share/applications/waydroid.desktop`:

```ini
[Desktop Entry]
Name=Waydroid
Comment=Run Android apps on Linux
Exec=waydroid show-full-ui
Icon=waydroid
Terminal=false
Type=Application
Categories=System;Emulator;
```

## 7. Manajemen Aplikasi Android

### 7.1 Install APK

```bash
# Install dari file
waydroid app install /path/to/app.apk

# Install dari URL
waydroid app install https://example.com/app.apk
```

### 7.2 Menampilkan Daftar Aplikasi Terinstall

```bash
waydroid app list
```

### 7.3 Menjalankan Aplikasi Tertentu

```bash
waydroid app launch com.example.appname
```

### 7.4 Menghapus Aplikasi

```bash
# Hapus berdasarkan package name
waydroid app remove com.example.appname

# Hapus data aplikasi
waydroid app clear com.example.appname
```

### 7.5 Mengelola Multi-user

```bash
# Daftar user
waydroid user list

# Pindah ke user lain
waydroid user switch <user_id>

# Buat user baru
waydroid user create
```

### 7.6 Akses Shell Android

Untuk mengakses shell Android di dalam container:
```bash
sudo waydroid shell

# Atau langsung eksekusi perintah
sudo waydroid shell "getprop ro.build.version.release"
```

## 8. Troubleshooting

### 8.1 WAYLAND_DISPLAY is not set Error

**Penyebab:** Session Wayland tidak terdeteksi.

**Solusi:** Pastikan Anda login ke sesi Wayland, bukan X11.

```bash
echo $XDG_SESSION_TYPE
# Jika output 'x11', logout dan pilih sesi Wayland dari Display Manager
```

### 8.2 Waydroid Container STOPPED

**Penyebab:** Container service tidak berjalan.

**Solusi untuk Void Linux (runit):**
```bash
sudo waydroid container start
sudo sv status waydroid-container
```

### 8.3 Layar Hitam / Tidak Ada Output

**Penyebab:** Masalah driver grafis atau compositor.

**Solusi:**
1. Cek apakah compositor Wayland berjalan:
   ```bash
   pgrep -l sway   # untuk Sway
   pgrep -l mutter # untuk GNOME
   pgrep -l kwin   # untuk KDE
   ```

2. Coba restart session:
   ```bash
   waydroid session stop
   waydroid session start &
   waydroid show-full-ui
   ```

### 8.4 Aplikasi Android Force Close (Crash)

**Penyebab:** Inkompatibilitas atau resource terbatas.

**Solusi:**
```bash
# Lihat log Waydroid
sudo waydroid log

# Lihat log aplikasi spesifik
adb logcat | grep -i "error\|fatal"
```

### 8.5 Tidak Ada Internet di Waydroid

**Penyebab:** Firewall memblokir interface `waydroid0`.

**Solusi untuk iptables:**
```bash
sudo iptables -A FORWARD -i waydroid0 -j ACCEPT
sudo iptables -A FORWARD -o waydroid0 -j ACCEPT

# Simpan aturan
sudo iptables-save > /etc/iptables/rules.v4
```

### 8.6 Cara Menghentikan Waydroid

**Untuk Void Linux (menggunakan pkill):**
```bash
# Stop session
waydroid session stop

# Stop container (butuh root)
sudo pkill waydroid-container

# Atau stop all
pkill waydroid
```

> **Catatan:** Void Linux menggunakan `pkill`, bukan `killall` yang umum di distro lain .

### 8.7 NVIDIA GPU Issues

Jika menggunakan NVIDIA GPU dengan driver proprietary:

1. **Pastikan menggunakan compositor yang mendukung EGLStreams:**
   - GNOME Wayland ✅
   - KDE Plasma Wayland ✅
   - Weston ✅ (dengan flag `--use-egl`)
   - Sway ❌ (GBM only)
   - Wayfire ❌ (GBM only)

2. **Atau gunakan driver open-source `nouveau`** (performa lebih rendah tapi kompatibel)

## 9. Performa dan Optimalisasi

### 9.1 Faktor yang Mempengaruhi Performa

- **CPU:** Container menggunakan CPU host secara langsung, performa hampir native
- **RAM:** Alokasikan minimal 2 GB untuk container
- **GPU:** Akses GPU melalui Wayland langsung (tanpa overhead X11)
- **Storage:** Gunakan SSD untuk penyimpanan image

### 9.2 Konfigurasi Resource

Edit `/etc/waydroid/waydroid.cfg`:
```ini
[resources]
memory_mb = 4096
cpu_shares = 1024
```

### 9.3 Nonaktifkan Animasi Android

Masuk ke Settings → Developer Options, set semua animasi scale ke 0.

Untuk mengaktifkan Developer Options:
- Buka Settings → About Phone
- Tap "Build Number" 7 kali

## 10. Uninstall

### 10.1 Hapus Waydroid

```bash
# Hapus package
sudo xbps-remove waydroid

# Hapus data
sudo rm -rf /var/lib/waydroid

# Hapus konfigurasi
sudo rm -rf /etc/waydroid
```

### 10.2 Hapus Service runit

```bash
sudo rm /var/service/waydroid-container
sudo rm -rf /etc/sv/waydroid-container
```

### 10.3 Hapus Modul Kernel (Opsional)

```bash
sudo rm /etc/modules-load.d/binder.conf
```

## 📝 Perbandingan Waydroid: Wayland vs X11

| Aspek | Wayland (Native) | X11 (dengan Weston) |
|-------|------------------|---------------------|
| **Performa** | ⭐⭐⭐⭐⭐ (Native) | ⭐⭐⭐ (Overhead compositor tambahan) |
| **Instalasi** | ⭐⭐⭐⭐⭐ (Simple) | ⭐⭐ (Perlu Weston dan konfigurasi) |
| **Integrasi Desktop** | ⭐⭐⭐⭐⭐ (Aplikasi muncul di menu) | ⭐⭐⭐ (Terisolasi di jendela Weston) |
| **GPU Acceleration** | ✅ Langsung | ⚠️ Lewat Weston (sedikit delay) |
| **Multi-window** | ✅ Ya | ❌ Tidak |
| **Copy-paste** | ✅ Ya (lewat portal) | ⚠️ Terbatas |

**Kesimpulan:** Untuk pengalaman terbaik dan performa optimal, gunakan **sesi Wayland native**. Dokumentasi ini fokus pada konfigurasi Wayland karena itulah cara yang didesain dan didukung secara resmi oleh Waydroid .
