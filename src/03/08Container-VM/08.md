# VirtualBox (Oracle VM VirtualBox)

VirtualBox adalah aplikasi virtualisasi open-source dari Oracle yang memungkinkan menjalankan berbagai sistem operasi sebagai virtual machine (VM). T4n OS dapat digunakan sebagai **host** (menjalankan VirtualBox) maupun sebagai **guest** (diinstal di dalam VirtualBox).

---

## T4n OS sebagai Host

### Instalasi

VirtualBox tersedia di repositori nonfree T4n OS:

```bash
sudo xbps-install -S void-repo-nonfree
sudo xbps-install -Su
sudo xbps-install -S virtualbox-ose
```

Untuk dukungan modul kernel:

```bash
sudo xbps-install -S virtualbox-ose-dkms
```

### Konfigurasi Awal

Tambahkan pengguna ke grup `vboxusers`:

```bash
sudo usermod -aG vboxusers <nama_user>
```

Login ulang agar perubahan grup diterapkan.

### Muat Modul Kernel

```bash
sudo modprobe vboxdrv
sudo modprobe vboxnetflt
sudo modprobe vboxnetadp
```

Untuk memuat otomatis saat boot, buat file `/etc/modules-load.d/virtualbox.conf`:

```
vboxdrv
vboxnetflt
vboxnetadp
```

### Jalankan VirtualBox

```bash
virtualbox
```

Atau via menu aplikasi jika sudah menggunakan desktop environment.

## T4n OS sebagai Guest (di dalam VirtualBox)

### Membuat VM untuk T4n OS

1. Buka VirtualBox → klik **New**
2. Isi konfigurasi:

| Setting        | Nilai yang Disarankan                   |
|----------------|-----------------------------------------|
| Name           | T4n OS (atau sesuai keinginan)          |
| Type           | Linux                                   |
| Version        | Other Linux (64-bit)                    |
| RAM            | Minimal 1 GB, disarankan 2 GB+          |
| Storage        | Minimal 10 GB, disarankan 20 GB+        |
| CPU            | Minimal 1 core, disarankan 2 core+      |
| Video Memory   | 128 MB                                  |

3. Mount ISO T4n OS di **Settings → Storage → Controller: IDE → Empty**
4. Jalankan VM dan instal T4n OS seperti biasa

### VirtualBox Guest Additions

Guest Additions menyediakan integrasi antara host dan guest:

- Resolusi layar otomatis (seamless resize)
- Sinkronisasi clipboard
- Drag and drop file
- Shared folders
- Pointer mouse terintegrasi (tanpa perlu klik untuk masuk/keluar VM)
- Sinkronisasi waktu host-guest

#### Instalasi

```bash
sudo xbps-install -S virtualbox-ose-guest-utils virtualbox-ose-guest-dkms
```

Untuk sesi grafis (X11/Wayland):

```bash
sudo xbps-install -S virtualbox-ose-guest-utils-x11
```

Muat modul kernel guest:

```bash
sudo modprobe vboxguest
sudo modprobe vboxsf
sudo modprobe vboxvideo
```

Untuk memuat otomatis saat boot, buat `/etc/modules-load.d/vbox-guest.conf`:

```
vboxguest
vboxsf
vboxvideo
```

Tambahkan pengguna ke grup `vboxsf` untuk akses shared folders:

```bash
sudo usermod -aG vboxsf <nama_user>
```

## Konfigurasi

### Resolusi Layar Otomatis

Pastikan `virtualbox-ose-guest-utils-x11` sudah terinstal. Jika resolusi tidak menyesuaikan secara otomatis, tambahkan ke `/etc/X11/xorg.conf.d/10-vbox.conf`:

```
Section "Device"
    Identifier "VirtualBox GPU"
    Driver     "modesetting"
EndSection
```

### Shared Folders

1. Di VirtualBox host, buka **Settings → Shared Folders → Add**
2. Tentukan path folder di host dan nama share (contoh: `hostfolder`)
3. Centang **Auto-mount** jika ingin otomatis mount

Mount manual di dalam guest:

```bash
sudo mkdir -p /mnt/shared
sudo mount -t vboxsf hostfolder /mnt/shared
```

Mount otomatis via `/etc/fstab`:

```
hostfolder    /mnt/shared    vboxsf    defaults,uid=1000,gid=1000    0    0
```

Ganti `uid` dan `gid` dengan ID pengguna kamu (`id -u` dan `id -g`).

### Shared Clipboard

Di VirtualBox host: **Settings → General → Advanced → Shared Clipboard** → pilih **Bidirectional**.

Pastikan Guest Additions sudah terinstal dan modul `vboxguest` sudah dimuat.

### Drag and Drop

Di VirtualBox host: **Settings → General → Advanced → Drag'n'Drop** → pilih **Bidirectional**.

## Jaringan

VirtualBox menyediakan beberapa mode jaringan:

| Mode              | Keterangan                                                          |
|-------------------|---------------------------------------------------------------------|
| **NAT**           | VM bisa akses internet, tidak bisa diakses dari luar (default)      |
| **Bridged**       | VM seperti perangkat fisik di jaringan lokal                        |
| **Host-only**     | VM hanya bisa berkomunikasi dengan host, tidak ke internet          |
| **Internal**      | VM hanya bisa berkomunikasi dengan VM lain dalam jaringan internal  |
| **NAT Network**   | Seperti NAT tapi beberapa VM bisa saling berkomunikasi              |

Ganti mode di: **Settings → Network → Adapter 1 → Attached to**

## Snapshot

Snapshot menyimpan state VM pada titik waktu tertentu.

```
Machine → Take Snapshot       # Buat snapshot
Machine → Restore Snapshot    # Restore ke snapshot
Machine → Delete Snapshot     # Hapus snapshot
```

Via command line:

```bash
# Buat snapshot
VBoxManage snapshot "NamaVM" take "NamaSnapshot"

# Lihat daftar snapshot
VBoxManage snapshot "NamaVM" list

# Restore snapshot
VBoxManage snapshot "NamaVM" restore "NamaSnapshot"

# Hapus snapshot
VBoxManage snapshot "NamaVM" delete "NamaSnapshot"
```

## VBoxManage (CLI)

VBoxManage adalah tool command-line untuk mengelola VirtualBox tanpa GUI:

```bash
# Lihat daftar VM
VBoxManage list vms

# Lihat VM yang sedang berjalan
VBoxManage list runningvms

# Jalankan VM (headless)
VBoxManage startvm "NamaVM" --type headless

# Hentikan VM
VBoxManage controlvm "NamaVM" poweroff

# Shutdown graceful
VBoxManage controlvm "NamaVM" acpipowerbutton

# Lihat info VM
VBoxManage showvminfo "NamaVM"

# Ubah RAM VM
VBoxManage modifyvm "NamaVM" --memory 4096

# Ubah jumlah CPU
VBoxManage modifyvm "NamaVM" --cpus 4

# Buat disk image
VBoxManage createhd --filename disk.vdi --size 20480 --format VDI
```

## Troubleshooting

### Modul Kernel Tidak Termuat

```bash
# Cek status modul
lsmod | grep vbox

# Kompilasi ulang modul DKMS
sudo dkms autoinstall

# Atau instal ulang
sudo xbps-reconfigure -f virtualbox-ose-dkms
```

### Kernel Module Version Mismatch

Setelah update kernel, modul VirtualBox perlu dikompilasi ulang:

```bash
sudo xbps-install -S linux-headers
sudo dkms autoinstall
```

### VM Tidak Bisa Booting (EFI)

Aktifkan EFI di pengaturan VM: **Settings → System → Motherboard → Enable EFI**.

### Performa Lambat

- Aktifkan **VT-x/AMD-V** di **Settings → System → Acceleration**
- Aktifkan **Nested Paging**
- Tambahkan Video Memory hingga **128 MB**
- Instal Guest Additions untuk driver yang lebih optimal

### USB Tidak Terdeteksi di Guest

Instal VirtualBox Extension Pack (tersedia di [virtualbox.org](https://www.virtualbox.org/wiki/Downloads)) untuk dukungan USB 2.0/3.0.

Tambahkan pengguna ke grup `vboxusers` di host:

```bash
sudo usermod -aG vboxusers <nama_user>
```

### Shared Folder Tidak Bisa Diakses

```bash
# Pastikan modul vboxsf termuat
lsmod | grep vboxsf
sudo modprobe vboxsf

# Pastikan user di grup vboxsf
groups <nama_user>
sudo usermod -aG vboxsf <nama_user>
```
