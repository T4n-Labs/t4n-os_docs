# Panduan Chroot dan Container - T4n OS

*Chroot* dan *Container* dapat digunakan untuk berbagai tujuan, antara lain:
*   Menjalankan perangkat lunak berbasis `glibc` di sistem `musl` (atau sebaliknya).
*   Menjalankan aplikasi dalam lingkungan yang lebih terkontrol atau terisolasi (*sandboxed*).
*   Membuat *rootfs* untuk proses instalasi sistem baru (*bootstrapping*).

---

## 1. Pembuatan Chroot

### **Metode A: Menggunakan `xvoidstrap` (Rekomendasi)**
Cara termudah adalah menggunakan perintah `xvoidstrap` dari paket `xtools`.

1.  Buat direktori tujuan:
    ```bash
    sudo mkdir <direktori_chroot>
    ```
2.  Lakukan instalasi sistem dasar:
    
```bash
    sudo XBPS_ARCH=<arsitektur> xvoidstrap <direktori_chroot> base-container <paket_lainnya>
```
    *Catatan: `<paket_lainnya>` bersifat opsional jika Anda ingin menginstal paket tambahan sejak awal.*

### **Metode B: Cara Manual**
Jika Anda ingin melakukannya secara manual menggunakan `xbps-install`:

1.  Siapkan kunci repositori:
    
```bash
    sudo mkdir -p "<direktori_chroot>/var/db/xbps/keys"
    sudo cp -a /var/db/xbps/keys/* "<direktori_chroot>/var/db/xbps/keys"
```
2.  Instal paket `base-container`:
    
```bash
    sudo XBPS_ARCH=<arsitektur> xbps-install -S -r <direktori_chroot> -R <url_repository> base-container
```

## 2. Penggunaan Chroot

### **Metode A: Menggunakan `xchroot`**
Gunakan `xchroot` (dari `xtools`) untuk masuk ke lingkungan chroot secara otomatis tanpa perlu melakukan mounting manual.
```bash
sudo xchroot <direktori_chroot>
```

### **Metode B: Cara Manual**
Jika masuk secara manual, Anda harus memuat beberapa direktori sistem host ke dalam chroot:

1.  Salin konfigurasi jaringan (opsional):
    ```bash
    sudo cp /etc/resolv.conf <direktori_chroot>/etc/
    ```
2.  Mount direktori sistem:
    
```bash
    sudo mount -t proc none <direktori_chroot>/proc
    sudo mount -t sysfs none <direktori_chroot>/sys
    sudo mount --rbind /dev <direktori_chroot>/dev
    sudo mount --rbind /run <direktori_chroot>/run
```
3.  Masuk ke chroot:
    
```bash
    sudo chroot <direktori_chroot> /bin/bash
```
    *Penting: Selalu lakukan `umount` setelah selesai. Jika direktori chroot dihapus tanpa unmount, sistem host Anda bisa mengalami kerusakan.*

## 3. Alternatif Lain

### **Bubblewrap (Sandboxing)**
`bwrap` (dari paket `bubblewrap`) memungkinkan isolasi yang lebih kuat dan **tidak membutuhkan akses root**.

Contoh menjalankan perintah di dalam sandbox:
```bash
bwrap --bind <direktori_chroot> / \
      --dev /dev --proc /proc \
      --bind /sys /sys --bind /run /run \
      --ro-bind /etc/resolv.conf /etc/resolv.conf \
      sh
```

### **Flatpak**
Solusi paling praktis untuk menjalankan aplikasi grafis atau *proprietary* lintas pustaka (`glibc`/`musl`) secara aman dan terisolasi.

### **Application Containers (Docker/Podman)**
T4n OS menyediakan *image* OCI resmi yang dapat digunakan dengan `docker` atau `podman`. Ini adalah solusi yang lebih matang dan terintegrasi tanpa perlu membuat direktori chroot secara manual.
