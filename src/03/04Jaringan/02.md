# Firewall

T4n OS menyediakan dua pilihan firewall: **iptables** (sudah terinstal secara default) dan **nftables** (pengganti modern iptables).

---

## iptables

`iptables` sudah terinstal di sistem dasar dan menyediakan `iptables` (IPv4) serta `ip6tables` (IPv6).

File ruleset disimpan di:

- `/etc/iptables/iptables.rules` — untuk IPv4
- `/etc/iptables/ip6tables.rules` — untuk IPv6

Dua contoh ruleset tersedia di `/etc/iptables/` sebagai referensi:

| File                    | Keterangan                      |
|-------------------------|---------------------------------|
| `empty.rules`           | Ruleset kosong                  |
| `simple_firewall.rules` | Contoh firewall dasar           |

Salin salah satu sebagai titik awal:

```bash
sudo cp /etc/iptables/simple_firewall.rules /etc/iptables/iptables.rules
sudo cp /etc/iptables/simple_firewall.rules /etc/iptables/ip6tables.rules
```

### Menerapkan Rules saat Boot

**Opsi 1 — Stage 1 (disarankan):** Instal `runit-iptables`:

```bash
sudo xbps-install -S runit-iptables
```

Paket ini menambahkan core-service yang otomatis memuat ruleset saat boot.

**Opsi 2 — Stage 2:** Tambahkan ke `/etc/rc.local`:

```bash
if [ -e /etc/iptables/iptables.rules ]; then
  iptables-restore /etc/iptables/iptables.rules
fi

if [ -e /etc/iptables/ip6tables.rules ]; then
  ip6tables-restore /etc/iptables/ip6tables.rules
fi
```

Verifikasi setelah reboot:

```bash
sudo iptables -L
sudo ip6tables -L
```

### Menerapkan Rules saat Runtime

Aktifkan layanan iptables:

```bash
ln -s /etc/sv/iptables  /var/service/
ln -s /etc/sv/ip6tables /var/service/
```

| Perintah               | Fungsi                  |
|------------------------|-------------------------|
| `sv up iptables`       | Muat ruleset IPv4       |
| `sv down iptables`     | Flush ruleset IPv4      |
| `sv up ip6tables`      | Muat ruleset IPv6       |
| `sv down ip6tables`    | Flush ruleset IPv6      |

## nftables

`nftables` adalah pengganti modern dari `iptables`, `ip6tables`, `arptables`, dan `ebtables` dalam satu tool terpadu.

```bash
sudo xbps-install -S nftables
```

File konfigurasi utama: `/etc/nftables.conf`

### Migrasi dari iptables

`nftables` menyertakan tool konversi bawaan:

```bash
# Konversi satu rule
iptables-translate -A INPUT -p tcp --dport 22 -j ACCEPT

# Konversi seluruh ruleset dari file
iptables-restore-translate -f /etc/iptables/iptables.rules
```

### Menerapkan Rules saat Boot

Instal `runit-nftables`:

```bash
sudo xbps-install -S runit-nftables
```

Core-service ini otomatis memuat ruleset dari `/etc/nftables.conf` saat boot.

### Menerapkan Rules saat Runtime

Aktifkan layanan:

```bash
ln -s /etc/sv/nftables /var/service/
```

| Perintah            | Fungsi             |
|---------------------|--------------------|
| `sv up nftables`    | Muat ruleset       |
| `sv down nftables`  | Flush ruleset      |

---

## Perbandingan iptables vs nftables

| Aspek         | iptables                    | nftables                        |
|---------------|-----------------------------|---------------------------------|
| Status        | Default, sudah terinstal    | Perlu diinstal manual           |
| Cakupan       | IPv4 & IPv6 terpisah        | IPv4, IPv6, ARP, bridge — satu tool |
| Sintaks       | Tradisional                 | Lebih modern dan konsisten      |
| Migrasi       | —                           | Ada tool konversi otomatis      |
