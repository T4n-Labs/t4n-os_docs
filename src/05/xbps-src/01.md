# 7. Troubleshooting xbps-src

Kumpulan error yang sering muncul dan solusinya.

---

## ❌ Error Umum dan Solusi

### 1. `ERROR: pkg myapp: failed to extract`

**Penyebab:** Checksum salah atau distfiles corrupt.

**Solusi:**
```bash
# Hapus cache source
rm -rf masterdir/builddir/myapp*
# Re-fetch
./xbps-src fetch myapp
# Bandingkan checksum
xbps-uhelper fetch <distfile_URL>
sha256sum <file>
# Perbaiki checksum di template
```

---

### 2. `ERROR: configure: error: ... missing library`

**Penyebab:** Dependensi kurang.

**Solusi:** Tambahkan ke `makedepends` atau `hostmakedepends`.

---

### 3. `ERROR: xbps-uchroot: Operation not permitted`

**Penyebab:** Pengguna tidak punya akses ke chroot.

**Solusi:**
```bash
# Pastikan termasuk grup xbuilder
sudo usermod -a -G xbuilder $USER
# Logout/login
```

Atau ganti metode chroot di `etc/conf`:
```
XBPS_CHROOT_CMD=uunshare
```

---

### 4. `!!! ERROR: couldn't find `make` in PATH`

**Penyebab:** `build_style` membutuhkan `make`, tapi tidak diinstall.

**Solusi:** Tambahkan `hostmakedepends="make"` (meskipun biasanya sudah ada di base-chroot).

---

### 5. `ERROR: myapp-1.0_1: file `/usr/lib/libfoo.so` already installed by foo-1.2_1`

**Penyebab:** Konflik file dengan paket lain.

**Solusi:** Tambahkan `conflicts` di template atau gunakan `replaces`.

---

### 6. `ERROR: failed to install package myapp from repository`

**Penyebab:** Paket gagal diinstall setelah build.

**Solusi:** Periksa log di `hostdir/logs/myapp-*.log`. Kemungkinan:
- `post_install()` tidak lengkap (file hilang)
- Dependensi tidak terpenuhi saat runtime
- Permission file salah

---

### 7. `xbps-src: command not found` (padahal sudah clone)

**Penyebab:** Belum `cd void-packages`.

**Solusi:** Pastikan Anda di direktori `void-packages/`.

---

## 🔍 Debugging Lebih Dalam

### Mode verbose:

```bash
./xbps-src -v pkg myapp
```

### Mode shell (masuk ke chroot):

```bash
./xbps-src chroot
# Sekarang di dalam environment chroot
cd builddir/myapp-version
# Coba compile manual
```

### Lihat log lengkap:

```bash
cat hostdir/logs/myapp-*.log
```

### Mensimulasikan tanpa build:

```bash
./xbps-src extract myapp   # hanya extract source
./xbps-src configure myapp # hanya configure
./xbps-src build myapp     # hanya build
```

---

## 🧹 Reset Bersih

Jika semua error, reset ke kondisi awal:

```bash
cd void-packages
rm -rf masterdir hostdir
./xbps-src binary-bootstrap
```

Atau reset penuh (termasuk perubahan template):

```bash
git reset --hard origin/master
git clean -fdx
./xbps-src binary-bootstrap
```

---

## 📝 Logging

Untuk mengirim log ke maintainer atau forum, gunakan:

```bash
./xbps-src pkg myapp 2>&1 | tee build.log
```

Simpan `build.log` dan bagikan.
